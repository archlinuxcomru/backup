tar -xvvzf
Достаточно 
tar xvvf
В этом случае будут распаковываться tar-архивы с любым форматом сжатия, а не только gzip.
А вот для других типов архивов tar не подойдёт, тут уже понадобятся unzip/unrar/7z и т.д.

разархивирует рядом в папочку
На самом деле нет.
tar, как и другие консольные архиваторы, не распаковывает “в папочку”, а вываливает всё как есть. Если появляется каталог, значит он уже был внутри архива, и из него распаковался. Если его там нет – сами понимаете, что будет.
Подразумевается, что консольная команда будет использоваться из консоли или из скрипта, и если требуется создать каталог для распаковки, он будет создан другой командой.

Cmndr_AlexGreen
Но поскольку я применяю команду не через консол к архиву а по средством Thunar"а то возможно и папку рядышком с архивом создает фменеджер?
Нет. Никаких папок файл-менеджер не может создавать, если он не умеет сам работать с этим архивом. Я же сказал – папка распаковывается ИЗ АРХИВА. Если в архиве её не будет, она не распакуется.
О чём мы спорим? Просто загляните в архив и посмотрите, есть там папка или нет. Если нечем, откройте консоль и распакуйте той же командой. Или просмотрите содержимое архива, для этого tar нужен с ключом t вместо x.

Насколько я помню, thunar не умеет и не должен уметь работать с архивами, для этого есть специальные графические оболочки для архиваторов. В пакете p7zip, который вы установили, есть графическая оболочка 7zG и 7zFM. Для гнома есть оболочка file-roller. Для xfce есть аналогичная оболочка xarchiver. В KDE те же функции исполняет ark.

Из всех пакетов архиваторов что я поставил не один не запустился просто по команде и не открыл свое окно для работы((видно они просто консольные?)
Верно! Они консольные, но именно их обычно используют графические оболочки, и без соответствующего консольного архиватора графическая оболочка не сможет работать с определённым форматом архивов.

как я понял тар сам по себе распаковывает пакеты заархивированные зипами, а к примеру для рара нужен другой архиватор
Нет, tar распаковывает только свои собственные tar-архивы. Они могут быть поверх сжаты gzip (.gz), bzip2 (.bz2), xz и т.д., но внутри должен быть tar.
Для зипа нужен unzip, для рара unrar.
Для распаковки зипа с русскими DOS-именами файлов нужен патченный unzip.
