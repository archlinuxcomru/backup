Хотел переустановить xorg, но pacman выдал такое сообщение - error while loading shared libraries: libgcc_s.so.1: cannot open shared object file: No such file or directory.
Непонятно, зачем переустанавливал xorg.
И что в итоге всего этого получилось, тоже не понятно.
А потому можно попробовать действовать прямо в лоб, то есть раз ругается на отсутствие libgcc_s.so.1, удовлетворить это желание.
Возможны 2 варианта
1 - просото подсунуть эту библиотеку в нужное место
2 - переустановить пакет, которому эта библиотека принадлежит.
А дальше будет видно.
EDIT 1 - сначала, конечно, неплохо бы проверить наличие этой библиотеки. Запусти - find / -name 'libgcc_s.so.1' 2> /dev/null
EDIT 2 - у меня показало в двух местах
/usr/lib/libgcc_s.so.1 - принадлежит gcc-libs 7.2.0-3
/usr/lib32/libgcc_s.so.1 - принадлежит lib32-gcc-libs 7.2.0-1
Ошибки не исчезают с опытом - они просто умнеют
