Имеется nvidia gtx650 с проприетарным драйвером, монитор, подключенный по dvi, и арч с xfce (или xubuntu 14.04). При изменении яркости и контрастности в color correction меняется яркость и контрастность всего, кроме курсора. Идеи есть?

Порядок действий для решения (не обязательно повторять в точности, суть действий - отключение в настройках иксов HWCursor и включение SWCursor):
  1. открыть nvidia-settings
  2. на вкладке X Server Display Configuration нажать Save to X Configuration File, затем Show Preview
  3. полученный текст скопировать, в блок Section "Screen" добавить
    4.     Option         "SWCursor" "on"
    Option         "HWCursor" "off"
  4. сохранить всё в файл /etc/X11/xorg.conf.d/10-nvidia.conf
  5. перезагрузиться и заново изменить яркость и контрастность в настройках, если они сбросились
  6. для сохранения настроек (актуально только для Арча) добавить в автозагрузку команду 
    nvidia-settings -l


Извините за качество, но скриншот, по понятным причинам, тут не поможет.

Тут товарищ говорит:

One other fun thing about using X gamma settings to adjust brightness. The first night I used it, I noticed at some point that my cursor looked very different -- it had become blindingly white. It turns out that the cursor is implemented at a lower level and doesn't go through the X gamma system. So turning the brightness down via gamma curves doesn't affect the cursor, which remains always at full brightness.

А что, если попробовать опцию "SWCursor" в xorg.conf ?
