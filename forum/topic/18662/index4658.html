nafanja, если тебе необходимо использовать флешку только в качестве пароля для входа в систему и в качестве пароля для sudo, то можно не приобретать eToken, а использовать любую имеющуюся флешку, подготовленную специальным образом. eToken, как правило, используется для других целей.
Данная флешка может обеспечить следующие меры безопасности для входа в систему и использования sudo:
- обязательное наличие как вставленной флешки так и ввод пароля user
- не нужно вводить пароль user при вставленной флешке, а при отсутствии флэшки действует общепринятый режим работы.
Если тебе нужно именно это, то могу описать подробно.

UPD - у меня флэшка задействована только для использования sudo, чтобы не вводить пароль. Для входа в систему настраивать не стал, но, конечно, пробовал. Работает нормально.
Привожу пример использования для sudo
1. при наличии флешки (пароль вводить не требуется)
sudo mc
* pam_usb v0.5.0
* Authentication request for user "vasek" (sudo)
* Device "usb_16" is connected (good).
* Performing one time pad verification...
* Access granted.

2. при отсутствии флэшки (требуется ввод пароля - общепринятый режим работы)
sudo mc
* pam_usb v0.5.0
* Authentication request for user "vasek" (sudo)
* Device "usb_16" is not connected.
* Access denied.
[sudo] пароль для vasek:

Опиши подробно если не трудно
1. Ставим из aur пакет pam_usb (yaourt -S pam_usb )
2. Подключаем к ноуту флешку, которую хотим использовать в качестве ключа (с флешкой ничего не будет, можно ее будет использовать и дальше, как накопитель). Желательно все другие USB убрать.
3. Создание ключа - sudo pamusb-conf –add-device <name_usb> - (где <name_usb> любое имя флешки).
В конце спросит - Save to /etc/pamusb.conf ? ….. [Y/n] ….отвечаем Y
4. Определим (укажем) пользователя, который будет этим пользоваться
sudo pamusb-conf –add-user <user> (<user> - имя юзера)
спросит … Save to /etc/pamusb.conf ? ….[Y/n] …..отвечаем Y
5. Проверяем наше творение...
pamusb-check <user> –debug
Если все нормально, то выдаст * Access granted.
6. Разбираемся с логином и паролем при загрузке (если есть необходимость) ...... описано для загрузки startx, без DM, с DM - описал в конце
sudo nano /etc/pam.d/login
и после строки (должна быть 1-ая)
auth required pam_securetty.so
вставляем строку
auth required pam_usb.so или auth sufficient pam_usb.so
required - аутентификация и по флешке и по паролю
sufficient - аутентификация только по флешке …. то есть пароль вводить не нужно
7. sudo без пароля, по флешке (если флешки не будет, то по паролю)
в файл /etc/pam.d/sudo добавить на верх строку auth sufficient pam_usb.so
Дополнительно добавить в /etc/sudoers строку в конец файла (это чтобы пароль спрашивался каждый раз при работе без флешки)
Defaults timestamp_timeout=0
UPD - для пользующихся DM (например, gdm) - в файл /etc/pam.d/gdm-password добавить на верх строку auth sufficient pam_usb.so если входить только по флешке или auth required pam_usb.so если входить и по флешке и по паролю.
Все, отмонтируем и вытаскиваем флешку и перегружаемся

UPD - забыл написать, на флешке появится директория .pamusb …... не удали...)

EDIT 1 - заметил старый баг - --add-device, --add-user, --debug ........ две черточки, а не одна

EDIT 2 - внес в описание дополнение - описано для загрузки startx, без DM, с DM - описал в конце
