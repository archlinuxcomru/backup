Уже пару дней пользуюсь sudo и всё хорошо, но покоя не дают две вещи:

1 Как сделать так, чтобы после запуска команды с помощью sudo, для других действий, требующих права root'a, sudo вводить не надо было, в течении определённого времени? Поясню на примерах.
Сейчас у меня для запуска команды нужно ввести sudo, после чего в течении двух минут не нужно вводить пароль, но sudo всё равно требуется ставить перед каждым действием. В Ubuntu, помню, после введения sudo тебе давались права root'a на время, то есть ввёл и в течении энного времени исполняешь всё от root'a. У меня есть такая строка в файле настроек:
Defaults timestamp_timeout=2,rootpw
Как сделать так, чтобы в течении указанного времени sudo вообще не надо было вводить? Искал, давались параметры -i, -s. Но без толку.

2 По непонятым причинам некорректно работает sudo на флешке. В чём суть:
Пытаюсь сразу-же на флешку записать список пакетов.
pacman -Sp nitrogen > nitrogen
Отказано в доступе.
Но запись файлов, удаление, копирование - всё отлично. Если сделать su - тоже всё отлично, и pacman пишет. Только с sudo и только pacman -Sp не хотят работать. Флешку монтирую
mount /dev/sdc1
(Сначала указывал и тип ФС, и место монтирования, но потом mount сама запомнила как-то.)

Если нужно именно список пакетов на флэшку, то, например:
yaourt -Qge > /media/KINGSTON/pkglist
поставить своё значение вместо KINGSTON.
Если писать как yaourt -Qge > /dev/sdf1 - то, соответственно, пошлёт.
/dev/sdfX - блочное устройство, /media/ФЛЭШКА - каталог. Проще говоря, для записи нужно указать каталог, а не девайс.

ps. Только к sudo этот момент ни каким боком.

Guzerus, ты, кажется, путаешь кислое с горячим.
Если команда требует sudo - тебе, по-любому, придётся вводить от юзера sudo …, если ты вводишь sudo -i - ты получаешь root shell и в нём уже работаешь без sudo.
Тебе, во что бы то ни стало нужно выполнение этих команд без ввода sudo перед ними или просто пароль влом вводить?
Если без sudo - используй yaourt, но для этого, чтобы ещё и без пароля, нужно внести соответствующюю запись в /etc/sudoers
Вообще, почитай-ка лучше man sudo и т.п. по теме.
Оговорка: либо ты можешь использовать alias'ы в ~/.bashrc + ещё что-нибудь намутить в /etc/sudoers, но что именно делать, не скажу, просто не заморачивался на эту тему. Мне кажется, это какой-то кривой путь, ради невведения четырёх знаков s u d o перед командой, там, где это требуется.
