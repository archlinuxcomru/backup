После обновления cups отвалился принтер HP Laserjet 1020. Проверка:
sudo systemctl status cups
показала, что юнит не загружен. Решение простое [Updated, в соответствии с Wiki]:
1. Открываем каталог /etc/systemd/system/printer.target.wants и наблюдаем битую ссылку cups.service.
2. Удаляем битую ссылку:
sudo systemctl disable cups
3. Открываем каталог /etc/systemd/system/multi-user.target.wants и видим битую ссылку cups.path.
4. Удаляем битую ссылку:
sudo systemctl disable cups.path
5. Запускаем cups:
sudo systemctl start org.cups.cupsd.service
и добавляем его в автозагрузку:
sudo systemctl enable org.cups.cupsd.service
Всё работает.
P.S. Спасибо vdemin и eikoninaru! Подсказанный ими способ лучше: при запуске org.cups.cupsd.service вместо удалённых ссылок создаются новые:
Created symlink from /etc/systemd/system/printer.target.wants/org.cups.cupsd.service to /usr/lib/systemd/system/org.cups.cupsd.service.
Created symlink from /etc/systemd/system/sockets.target.wants/org.cups.cupsd.socket to /usr/lib/systemd/system/org.cups.cupsd.socket.
Created symlink from /etc/systemd/system/multi-user.target.wants/org.cups.cupsd.path to /usr/lib/systemd/system/org.cups.cupsd.path.
