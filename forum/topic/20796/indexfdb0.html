Query the files database. This operation allows you to look for packages owning certain files or
display files owned by certain packages. Only packages that are part of your sync databases are
searched. See File Options below.
….
FILE OPTIONS (APPLY TO -F)
-y, –refresh
Download fresh package databases from the server. Use twice to force a refresh even if databases
are up to date.

-l, –list
List the files owned by the queried package.

-x, –regex
Interpret each query as a regular expression.

-q, –quiet
Show less information for certain file operations. This is useful when pacman’s output is

–machinereadable

repository\0pkgname\0pkgver\0path\n with \0 being the NULL character and \n a linefeed.

По сути дела, в части COMRESET - при инициализации диска происходит обмен последовательными сигналами между хостом и устройством (так называемая OOB Sequence) - хост отправляет сигнал COMRESET, устройство при получении сигнала отправляет сигнал COMINIT, далее идет обмен сигналом COMWAKE и заканчивается отправкой сигнала SYNC - если все прошло успешно, соединение готово.
В данном случае получили сообщение COMRESET failed - конечно, 1-ое наиболее вероятное, это плохой кабель, … но возможны и другие причины, нужно смотреть сообщения libata error ... тем более, что есть сообщение и другого рода
INFO: task swapper/0:1 blocked for more then 122 seconds
возможно это все и взаимосвязано, а возможно и нет ..
