Все! Наконец-то получилось! Ошибки были из-за неправильных прав после копирования. Нужно было копировать командой:
cp -ra *

CP(1) User Commands CP(1)

NAME
cp - copy files and directories

SYNOPSIS
cp … SOURCE DEST
cp … SOURCE… DIRECTORY
cp … -t DIRECTORY SOURCE…

DESCRIPTION
Copy SOURCE to DEST, or multiple SOURCE(s) to DIRECTORY.

Mandatory arguments to long options are mandatory for short options too.

-a, –archive
same as -dR –preserve=all
R - лишняя в команде ;)

mknod команда для создания байт(блочно)-ориентированных устройств, это в основном файлы устройств , каналы и прочее. вам нет нужды колдовать с бубном вместо ядра,имхо. Да и вопрос поставлен кривовато имхо, попробую ответить на половину =). Помоему перед чирутом достаточно “перевесить” эти устройства и информацию о процессах.
mkdir /mnt/new/system
sudo mount /dev/disk/s/moey/novoy/sistemoy /mnt/new/system
sudo mount --bind /dev /mnt/new/system/dev
sudo mount --bind /proc /mnt/new/system/proc
sudo chroot /mnt/new/system
