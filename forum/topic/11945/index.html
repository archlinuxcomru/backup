При подключении к нашему домашнему WiFi падает роутер (сеть остаётся, но не работает), подключаюсь через netctl, сеть защищённая. С бесплатным WiFi рядом (FON Free SSID, без пароля) всё работает нормально. Проблема началась с момента установки KDE. Содержимое файла с профилем нашей сети:
Description='My home WiFi'
Interface=wlo1
Connection=wireless
Security=wpa
IP=dhcp
ESSID='Snark'
Key='********'
#Я не идиот, и пароль в сообщении заменил звёздочками
Подключаюсь так:
netctl start Snark
Вывод iwconfig:
eno1      no wireless extensions.
wlo1      IEEE 802.11bgn  ESSID:"FON_FREE_ssid"
          Mode:Managed  Frequency:2.412 GHz  Access Point: 00:18:84:EE:4E:24
          Bit Rate=28.9 Mb/s   Tx-Power=16 dBm
          Retry  long limit:7   RTS thr:off   Fragment thr:off
          Power Management:off
          Link Quality=37/70  Signal level=-73 dBm
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:2  Invalid misc:194   Missed beacon:0
lo        no wireless extensions.
Вывод lspci | grep net -i:
03:00.0 Ethernet controller: Realtek Semiconductor Co., Ltd. RTL8111/8168 PCI Express Gigabit Ethernet controller (rev 07)
04:00.0 Network controller: Qualcomm Atheros AR9485 Wireless Network Adapter (rev 01)
Файл с профилем открытой сети:
Description='Free wifi'
Interface=wlo1
Connection=wireless
Security=none
ESSID=FON_FREE_ssid
IP=dhcp
