спасибо за скриншот, очень читабельно
ты путаешь глобальную адресацию и локальную

1) твой роутер коннектится к инету и получает для себя глобальный ip
2) роутер организует NAT для локальной сети и раздает пакеты в локальную сеть
3) роутер испольует адрес 192.168.1.1 со стороны локальной сети, поэтому шлюз по умолчанию для ноута должен быть 192.168.1.1
4) роутер умеет раздавать в локальную сеть айпишники и прочие параметры по DHCP
5) настройки DHCP в сторону лоакальной сети должны выглядеть как-то так: нижний адрес 192.168.1.2, верхний адрес 192.168.1.254, шлюз по умолчанию 192.168.100.1

после этого любой интерфейс на ноуте, должен получить для себя ip-адрес 192.168.1.X, адрес шлюза 192.168.1.1 и dns-серверы
никаких других адресов в локальной сети быть не должно

найди в роутере настройки DHCP

В общем, простыми словами
1) включить NAT на роутере
2) включить DHCP-сервер на роутере
3) настроить DHCP-сервер на роутере, указав к раздаче адреса из подсети 192.168.1.0, шлюз по умолчанию 192.168.1.1, и dns-серверы
4) на ноуте запросить адреса у роутера посредством dhclient
5) посмотреть, какие адреса дал роутер ноуту командами ifconfig, route -n, cat /etc/resolv.conf
6) пропинговать с ноута шлюз 192.168.1.1
7) попробовать выйти в инет
