Скорее всего, Вы поменяли по старому рецепту первую строку в /usr/bin/hp-setup . Это уже не требуется, потому что начиная с 3.15.2 hplip поддерживает python 3.

Нашел статью на которую опирался (http://archlinux.org.ru/forum/topic/14509/?page=2). При попытки решить использовал пост:
сделал так:

1) переустановил: python sudo pacman -S python
2) удалил hplip: pacman -R jplip
2) поставил основным python2: sudo cp /usr/bin/python2 /usr/bin/python
3) скачал с сайта http://hplipopensource.com исходники hp-lip версией 3.14, установил как scratch (пункт 14)
4) после этого в папке /usr/lib/python2.7/site-packages появился hpmudext
5) установил последний hplip: sudo pacman -S --force hplip
6) запустил настройку принтера: hp-setup -i
7) и все заработало
Вообщем у меня подвидом 3 лежал 2 питон :D. Просто переустановил и все зашло (проверил доки печатает и тестовую от hp тоже расспечатал). Но в процессе не смог воспользоваться графикой (хотя мало кому вообще гуи нужен но все-таки интересно):
