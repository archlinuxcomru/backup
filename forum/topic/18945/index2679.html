Если на предвидится UEFI, то можно поизвращаться:
Форматируем SSD в btrfs,
Создаем два поддтома: /root и /home
На HDD создаём каталоги Музыка, Видео, Документы и т.д, затем делаем симлинки на эти каталоги в хомяк
Своп не нужен
Итого: на SDD корень и хомяк, ФС btfs сама позаботится о распределении места на диске между поддтомами. В хомяке на SDD "быстрый" кэш приложений: браузер, почта и т.д. "Тяжелые" данные - музыка, доки, фильмы - на HDD.

swap раздела ноутбук будет выходить из гибернации?
Нет.
Если нужна гибернация, то нужен и swap, не меньше размера ОЗУ. в Вашем случае 6+ГБ

А для чего делать симлинки, насколько я помню это как ярлыки в окнах?
Это ссылки. То есть данные лежат на одном месте, а к ним ведёт ссылка. Тупо для удобства. То есть Вы в домашнем разделе (SSD) нажимаете на папку Музыка и "проваливаетесь" на HDD.

Никогда раньше не использовал btrfs, возможны какие-либо проблемы с этой файловой системой?
Проблемы могут быть с любой ФС, поэтому не забывайте про резервное копирование важных файлов! Систему всегда можно переустановить (в крайнем случае), а вот потерянные данные вернуть тяжелее. Но у меня за пару лет пользования btrfs проблем не возникало. Кстати, в Suse btrfs - это ФС по умолчанию, так что вполне стабильная и рабочая ФС. Из плюсов - снапшоты, RAID-массив средствами самой ФС. Можно дефргаментацию в фоне.
Если заинтересовало, пишите. Помогу разобраться.

Как я понял, схема разметки дисков будет такая
sda это SSD
sdb это HDD

lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0  83.9G  0 disk
├─sda1   8:01   0     83.9G  0 part /
sdb      8:16   0 698.7G  0 disk
├─sdb1   8:17   0    9G  0 part #SWAP
├─sdb2   8:18   0    500G  0 part /home
├─sdb3   8:19   0 110.5G  0 part /home #Раздел на котором хранятся данные
├─sdb4   8:20   0 10G  0 part /var
Решил, что лучше использовать SSD диск только под систему попробую файловую систему btrfs, так как HDD 750Gb
Swap раздел вынес на HDD, отдал под него 9G (файловая система btrfs не поддерживает swap файлы, вычитал на ArchWiki)
Выделил отдельный раздел для /var на 10G (решил воспользоваться советом и ArchWiki)
Разделу home отдаю всё остальное пространство на HDD, после установки системы перенесу все файлы с раздела sdb3 на sdb2, удаляю раздел sdb3, а освободившееся место отдам разделу sdb2 (Возможно оставлю пустым для раздела с бэкапами)

Оцените пожалуйста данную схему, всё ли нормально будет с её работой?
