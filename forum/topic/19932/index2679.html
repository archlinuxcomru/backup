Что бы не захламлять тему о дефектных обновлениях, продолжение от https://archlinux.org.ru/forum/post/229739/

redix схема ( /usr/share/glib-2.0/schemas/net.launchpad.plank.gschema.xml ) что идет в комплекте пакета планк валидная.
Просто для dock.settings не указали path (путь) видимо для того чтобы можно было несколько настроек дока хранить (dock1 dock2 и т.д.)

Можно исправить указав path
например для дефолтного dock1
файл /usr/share/glib-2.0/schemas/net.launchpad.plank.gschema.xml линия 36 добавить path="/net/launchpad/plank/docks/dock1/"
<schema id="net.launchpad.plank.dock.settings" path="/net/launchpad/plank/docks/dock1/" gettext-domain="plank">

И перекомпилировать схемы
sudo glib-compile-schemas  /usr/share/glib-2.0/schemas

После можно запускать dconf-editor должна быть схема /net/launchpad/plank/docks/dock1/



Не забудьте сохранить изменения в dconf-editor
Для проверки можно глянуть текущие параметры
gsettings list-recursively |grep net

Может еще от чего настройки не сохранялись, схема по идеи для этого не нужна:
выбран не тот номер дока
права на файл проверить
