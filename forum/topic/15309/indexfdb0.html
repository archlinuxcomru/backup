systemd-analyze blame никак не отражает время загрузки гнома. Он показывает лишь время старта gdm.service а вся загрузка гнома происходит уже после запуска процесса.
Для примера, полный старт с гномом на машинке с ссд занимает реально около 7 секунд:
Startup finished in 2.682s (kernel) + 951ms (userspace) = 3.633s
Старт менее мощной машинки с гномом - 32 секунды, при показаниях:
Startup finished in 3.551s (kernel) + 12.060s (userspace) = 15.612s
Та же слабая машинка с отключенным гдм, бут в консоль:
Startup finished in 3.533s (kernel) + 8.373s (userspace) = 11.906s
После чего, с момента ручного запуска systemctl start gdm до появления логинскрина проходит 16.5 сек (секундомер).

Раньше был бутчарт, там можно было увидеть реальное время загрузки гнома, а теперь его выпилили...
