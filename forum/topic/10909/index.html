Здравствуйте. Я в Arch еще новичок, вот первый раз установил arch по английскому гиду. Все вроде бы хорошо, да только после установки нет сети.
Я использую интернет с логином и паролем, но эту задачу выполняет роутер, логинится и т.п. Мне же остается просто поднять сеть, но проблема: нет сетевого интерфейса eth*. Команда ip a выводит только интерфейс lo. Погуглил, решил поставить официальный драйвер realtek. Скачал с оффсайта исходники, поправил autorun.sh, который там был, в итоге этот драйвер становиться не хочет, не может обнаружить команду clean. Так же до этого он не обнаруживал rmmod и /lib/modules/ядро/build. Путь на rmmod я поправил в autorun, символическую ссылку на usr/src/ядро сделал, все равно драйвер не становится. В autorun.sh видел много слов ubuntu и debian - испугался. Посоветуйте, пожалуйста, как решить эту проблему. Реалтек встроенный в мамку. Самое интересное, что при установке интернет есть, и настраивать его не надо. По появлению рута можно пинговать сразу сайт, все работает. Как установлю, так пипец - интерфейса нет. Устанавливал все по официальному англоязычному гиду.

https://wiki.archlinux.org/index.php/Network_Configuration
Note: Systemd v197 introduced Predictable Network Interface Names, which automatically assigns static names to network devices. Interfaces are now prefixed with en (ethernet), wl (WLAN), or ww (WWAN) followed by an automatically generated identifier, creating an entry such as enp0s25. This behavior may be disabled by adding a symlink:
# ln -s /dev/null /etc/udev/rules.d/80-net-name-slot.rules
Users upgrading from an earlier systemd version will have a blank rules file created automatically.
