Уже второй день не могу решить проблему.

Как устанавливал:
sudo su
pacman -Syu
pacman -S qemu virt-manager dnsmasq dmidecode
systemctl enable libvirtd.service
nano /etc/libvirt/libvirtd.conf
-----------------libvirtd.conf-----------------
#unix_sock_group = "libvirt"       -> unix_sock_group = "libvirt"
#unix_sock_rw_perms = "0770"  -> unix_sock_rw_perms = "0770"
---------------------------------------------------
usermod -a -G libvirt [пользователь]
usermod -a -G kvm [пользователь]
usermod -a -G input [пользователь]
systemctl restart libvirtd.service
На этом этапе всё хорошо, в BIOS режиме могу устанавливать виртуальные машины и пользоваться всем функционалом.

Проблема начинается на этапе добавления UEFI.
Устанавливал так:
sudo su
pacman -Syu
pacman -S edk2-ovmf
nano /etc/libvirt/qemu.conf
-----------------qemu.conf-----------------
#nvram = [   -> nvram = ["/usr/share/ovmf/x64/OVMF_CODE.fd:/usr/share/ovmf/x64/OVMF_VARS.fd"]
--------------------------------------------------
systemctl restart libvirtd.service
