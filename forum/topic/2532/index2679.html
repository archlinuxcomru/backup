Для ntp достаточно настроить /etc/ntp.conf – у меня так:
 # /etc/ntp.conf
# Select three geolocalized NTP public servers; see http://www.pool.ntp.org/
server ntps1-0.cs.tu-berlin.de
server ntps1-1.cs.tu-berlin.de
server ntp.mobiletime.ru
# Only allow read-only access from localhost
restrict default nomodify nopeer
restrict 127.0.0.1
restrict ::1
# Location of drift and log files
driftfile /var/lib/ntp/ntp.drift
logfile /var/log/ntp.log
(здесь как раз и задается положение driftfile)
и добавить ntpd в демоны:
DAEMONS=(... ntpd)
Больше ничего не надо.
Однако, по-моему, не дружит с hwclock!

Не всё так просто.
Грубая коррекция времени по крону может иметь нехорошие побочные эффекты – реакция некоторых программ на скачок времени, особенно назад, может быть непредсказуемой. К примеру, mplayer от скачка назад намертво виснет.

Как сделано у меня.
Файл /etc/rc.local (не забыть сделать исполняемым):
#!/bin/bash
#
# /etc/rc.local: Local multi-user startup script.
#
ntpd -qg >/dev/null 2>&1 &
В /etc/rc.conf в строку DAEMONS добавлен hwclock.
При каждой загрузке системные часы синхронизируются с серверами в интернете, при каждой выгрузке аппаратные часы синхронизируются с системными. Работает.
