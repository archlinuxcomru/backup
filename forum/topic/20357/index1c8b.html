Предположу, как наиболее вероятное, что причина в нерабочем framebuffer … но не факт.

Насколько я понимаю, о видеодрайвере пока говорить рано - в X-ы не грузимся, в archiso для взаимодействия с системой нужна текстовая консоль … а вот чтобы запустить консоль в режиме UEFI, нужен framebuffer EFI. Что включено в 32-битное ядро, не знаю - нужно смотреть, но как минимум должно быть CONFIG_FB_EFI=y, CONFIG_EFI_STUB=y и др. - точно не знаю.
Возможно это все и включено, но тогда остается вопрос поддержки ядром данного оборудования - то есть нужно смотреть.

И далее, предположим, что запустили archiso и даже установили систему, но нет 100% уверенности, что в ядре есть полная поддержка устройств, входящих в этот планшет. Сначала бы неплохо в этом убедиться, чтобы не ломать зря копья - и, имхо, для начала все-таки неплохо бы проверить это, … а уж потом смотреть, если смысл заниматься установкой или нет. А проверить нужно то оборудование, которое планируется использовать.

с ядром до 4.15
Попробуйте качнуть
http://mirrors.advancedhosters.com/archlinux/iso/archboot/2016.12/archlinux-2016.12-1-archboot.iso

положить рядом с archlinux-2020.11.01-i686.iso ( на флешке в папку images)
в конфиг груба (menu.cfg) добавить
##старое ядро
dir=/images
insmod regexp               # для шаблонов * в именах файлов
probe -s root_uuid -u $root # получаем UUID для подстановки в img_dev=

for iso in $dir/archlinux-*.iso ; do      # ищем образ(ы) Arch по шаблону
 if [ ! -f "$iso" ] ; then continue; fi        # только если образ существует
 regexp -s build 'archlinux-(.+)-'archboot "$iso"  # получаем из имени образа дату сборки

  menuentry "Arch Linux ISO $build archboot" --source="\
loopback loop $iso
probe -s isolabel -l loop
linux (loop)/boot/vmlinuz_i686 archisolabel=\$isolabel img_dev=/dev/disk/by-uuid/$root_uuid img_loop=$iso earlymodules=loop
initrd (loop)/boot/initramfs_i686.img"

done
## Перезагрузка компьютера

И попробовать при загрузке в меню груба выбрать archlinux-2016

Грузиться может долго с пустым экраном , подождите пару минут

Стало интересно, установил ли кто то на это чудо Linux - потратил на это некоторое время.
Статьи имеются, но ничего толкового нет, например, в русской аудитории - рассуждений и рассказов много, например, здесь, (другие ссылки не привожу), но как то вот веры особо в это нет ... в смысле, что им удалось это выполнить
В зарубежной аудитории - там написано поменьше, но попались два высказывания спецов, к которым обратились с вопросом, привожу в оригинале
- дипломатичный ответ спеца не спецу … чтобы не расстраивать
Bad news: due to the BIOS which is old and not updated anymore, you can't install any other operating system than Windows. That computer is Windows-only compatible unfortunately. I know, this is frustrating and it is not the reply you wanted to read, sometimes this kind of things happens (I had the same issue with a desktop computer, I know how it feels). Next time, I would advise you to check if you can install Linux before buying your computer. Linux is installable in 90% of the time, but this time you are out of luck!
- более жесткий ответ … и как резюме, что если и возможна загрузка с USB, то не хватит навыков для этого
It is not a question of whether or not this processor can support Android or Linux (it certainly can), it is a question of whether or not the BIOS will support booting from any source other than the Windows eMMC (and specifically from a USB device). If it does, you should be able to boot from, for example, a Linux installation image. To be honest and forthcoming, I am thinking that, if you have to ask this question, you may not have the skills to pull this off.
И в дополнение, как и писал выше, много заметок, что не все устройства будут работать.
И резюмируя, конечно, теоретически установить возможно, но нужно много знать и хорошо разбираться … а для начала нужно определиться, что из нужных устройств будет работать, а что не будет и стоит ли этим заниматься … и если уж и браться за установку, то начинать не с ArchLinux.

PS - попалось еще одно высказывание: данное устройство не поддерживает legacy boot mode, но теоретически можно разработать свой CSM модуль … даже стало интересно - а исполнил ли кто-нибудь такое практически? … думаю, вряд ли.
