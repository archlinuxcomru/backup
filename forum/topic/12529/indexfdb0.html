Потому что это глупо! Файловые системы
mount | grep -E "^sys |^dev "
sys on /sys type sysfs (rw,nosuid,nodev,noexec,relatime)
dev on /dev type devtmpfs (rw,nosuid,relatime,size=1019336k,nr_inodes=254834,mode=755)
не обычные файловые системы на диске, а являются абстракцией интерфейсов взаимодействия железа с ядром. То есть, даже если предположить, что программист использует операторы read & write для обмена информацией с устройством через интерфейс, например, /dev/unit, то все данные реально не проходят через этот псевдофайл. На самом деле для этого действия ядро вызывает драйвер устройства, который знает, как обработать данную абстракцию на реальном железе и через реальную шину обмена данными и шину управления. Ядро, которое вы хотите обмануть, расшарив данную папку по сети, просто не найдет необходимого драйвера для обработки операций ввода-вывода.
