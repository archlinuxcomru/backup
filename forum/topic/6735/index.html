http://archlinux.org.ru/node/225
Это нормально - это он создает файл необходимого размера, чтобы качать в него содержимое (причем в фоне все работает дальше, виснет только GUI часть). В результате файл получается нефрагментированный на диске.

Изменить это поведение можно через файл:
.config/transmission/settings.json :

“preallocation”: 1, - поставить 0

Инфа по теме: http://trac.transmissionbt.com/wiki/Edi … s#Advanced
Я у себя на рейзере выставил “preallocation”: 2 - лучше чуть-чуть подождать, чем потом терять до половины скорости при копировании этого файла. Почему-то файлы вытянутые через торренты страдают таким багом.
