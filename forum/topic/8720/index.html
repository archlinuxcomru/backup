Comment by: srabd on 2009 02 19 [21:02:54]
Ok, this new PKGBUILD (5.0.11337.1968-2) doesn't include some libraries to allow google-earth use Arch ones, so there are new dependencies:
[*] lib32-curl                                                                                                                             
[*] lib32-gcc-libs                                                                                                                         
[*] lib32-mesa                                                                                                                             
[*] lib32-openssl (This one was included in 5.0.11337.1968-1 because of related issue)                                                     
[*] lib32-qt                                                                                                                               
[*] lib32-zlib                                                                                                                             
'libX11.so.6' problem was not a 'bin32-google-earth' PKGBUILD issue, and it seems to be solved right now.                                  
Added also 'lib32-freetype2' (not 'lib32-freetype' which doesn't exist) as dependency.                                                     
For luuuciano:                                                                                                                             
If google-earth find the file 'libcrypto.so.0.9.8' provided by itself it won't be able to run, so you must "hide" that file. You can rename or remove it, but, why store a file you don't need? This PKGBUILD doesn't provide the file.                                                                                                                                 
Comment by: donvla on 2009 02 20 [14:48:26]
why these deps:                            
aurpage comments googleearth googleearth.desktop googleearth-mimetypes.xml PKGBUILD lib32-curl
aurpage comments googleearth googleearth.desktop googleearth-mimetypes.xml PKGBUILD lib32-sqlite3
no need for them here.                                                                           
Comment by: srabd on 2009 02 21 [10:34:19]
The installer provides 'libcurl.so.4', but this PKGBUILD removes all binary libraries provided by the installer (including 'libcurl.so.4') in order to use the Arch ones. Arch provides 'libcurl.so.4' with 'lib32-curl' package, so it's a dependency.                                                                                                                 
When I try to run 'googleearth' without 'lib32-curl' installed I get "error while loading shared libraries: libcurl.so.4" and without 'lib32-sqlite3' I get "error while loading shared libraries: libsqlite3.so.0".
