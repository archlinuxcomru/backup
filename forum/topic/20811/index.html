Ребят, воюю с этим принтером уже лет 8. Уже морально вымотало. Стабильно раз в 1-2 месяца какие-то проблемы с ним. Первые годы гуглил, решал, потом уже просто устал, стал действовать по такому мануалу:

sudo systemctl stop cups.service (org.cups.cupsd.service)
sudo systemctl disable cups.service (org.cups.cupsd.service)
spa -Rsn hplip hplip-plugin cups (или spa -Rsndd)
sudo rm -r /usr/share/hplip
sudo rm -r /var/lib/hp
# может еще что-то...
# reboot
установка hplip-plugin из aur
установка cups
sudo systemctl start cups.service (org.cups.cupsd.service)
sudo systemctl enable cups.service (org.cups.cupsd.service)
sudo hp-setup -i

либо полностью восстанавливать систему.

Сейчас впервые оба варианта не спасают. Система обновлена, драйвер последней версии с аура.
Печать тестовой страницы не идет с сообщением "приостановлено пользователем" и "Rendering completed".

systemctl status cups.service
io/hpmud/musb.c 1454: unable to write data hp:/usb/HP_LaserJet_Professional_P_1102w?serial=000000000W425066SI1c: 45 second io timeout

p.s. После того как в 2006 году перешел с дебиана на арч, - этот принтер моя самая большая проблема и геморрой, я большего горя не встречал.

Опыта работы с этим принтером не имею, но навскиду рекомендую 3 момента (без информации что то сказать не возможно)
1. Поискать решение на BBS - топики, имеющих отношение к данному принтеру, там имеются, возможно и найдешь что то дельное .
2. Если принтер определяется как виртуальный CD-ROM, то неплохо бы отключить эту функцию (заразу …). --- это похоже лишнее ...
3. Возможно проблема связана с Binary Plug-In (типа firmware)
Some HP printers require proprietary software technologies to allow full access to printer features and performance. Unfortunately, these technologies cannot be open sourced, but to resolve this HP uses a binary plug-in for these printers. This plug-in works in conjunction with our Linux Open Source Printing Software to improve the printing experience for HP’s Linux Printing Customers. Additionally, it requires the user to read and agree to a license agreement at the time of driver installation. However, to limit confusion, there is a single plug-in file (for each HPLIP release) for all plug-in enabled devices.
Ошибки не исчезают с опытом - они просто умнеют

Можно попробовать получить больше информации из логов - по дефолту стоит опция отладки warn, можно попробовать установить debug (или изменить в конфиге или запустить cupsctl --debug-logging … после проведения отладки вернуть на место - или в конфиге или запустить cupsctl --no-debug-logging)

PS 1 - В части HPLIP Binary Plug-In (hplip-plugin) - если ставишь из AUR, рекомендую ставить из китайской репы - все-таки репа лучше AUR
pacman -Ss hplip-plugin
archlinuxcn/hplip-plugin 1:3.21.4-1
    Binary plugin for HPs hplip printer driver library
