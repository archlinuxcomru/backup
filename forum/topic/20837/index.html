Какая программа для управления громкостью на USB?
DAC может иметь "аппаратный" регулятор громкости - тогда громкость регулируется аппаратно через алсу любым миксером.
А может не иметь, тогда громкость регулируется только программно пульсой или приложением выводящим звук

Сначала гляньте что за устройство и какой драйвер используется, возможно требуется фирмарь еще или дополнительные параметры драйвера
https://wiki.archlinux.org/title/Advanced_Linux_Sound_Architecture#ALSA_firmware

Неплохо бы для начала выложить хоть какую-нибудь информацию, чтобы иметь представление об устройстве, например,
lspci -knn | grep Audio -A 3  # .... забыл, что это USB - тогда другой вывод, дающий представление об устройстве
cat /proc/asound/cards*
cat /proc/asound/card*/codec#0 | head  # .... лучше, наверное, так - grep -i codec /proc/asound/card* -R
lsmod | grep snd
может кто и подскажет или затребует дополнительную информацию.

если установлен pipewire, то можно поиграться с этими настройками:
                # Use ALSA-Card-Profile devices. They use UCM or
                # the profile configuration to configure the device
                # and mixer settings.
                api.alsa.use-acp = false

                # Use UCM instead of profile when available. Can be
                # disabled to skip trying to use the UCM profile.
                api.alsa.use-ucm = true
также можно попробовать сохратить настройки звука через alsactl store.
одно время даже имел файл в домашней папке и загружался с командой
alsactl –file ~/asound.state restore

Для данного DAC, скорее всего, используется модуль snd_usb_audio, который возможно не поддерживает аппаратную регулировку громкости данного устройства.
Насколько мне известно в этих случаях применяют программное управление звуком (есть такой термин, как software-volume-regulation), которое можно осуществить и путем создания/редактирования фйлов/конфигов, например, /etc/asound.conf и др. - попадались раньше топики на эту тему (звук имеется, но не регурируется) ... теоретически в эту тему не углублялся в виду отстуствия данного девайса ... рекомендую погуглить по фразе - This sound device does not have any controls

PS - может что есть интересное в dmesg?

EDIT 1 - в AUR есть пакет alsa-control ...
paru -Ss softvol
aur/alsa-control 0.2.0-1 [+2 ~0.40]
    Easy to use ALSA default soundcard and volume configurator using Dmix and Softvol
из описания на GitHub ....
Since ALSA provides dmix for soundcards that don't support multiplexing and softvol for those that can't control their volume, it is not necessary to run pulseaudio for that. This application creates .asoundrc default configurations for this purpose, while the GUI may work as a replacement for pavucontrol
и не нужно самому редактировать asound.conf с задействованием softvol - вроде бы все готово и есть даже GUI - попробуй

EDIT 2 - хотя, наверное, лучше не использовать левые пакеты, а настроить самому, как писал выше, используя softvol в конфиге alsa - DOC1 ... DOC2
softvol - плагин ALSA, который позволяет пользователю добавлять новый регулятор громкости и управлять громкостью звука или его частями с помощью программного обеспечения. Это часто необходимо, если звуковая карта не может управлять громкостью аппаратно.
... хотя, конечно, нет 100% уверености, что это может помочь ...
