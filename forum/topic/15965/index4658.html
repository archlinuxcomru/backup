По-моему, vasek ещё на предыдущей странице написал, что сообщение "Загружается начальный виртуальный диск" выдаёт не ядро, не initramfs, его выдаёт автогенерированный конфиг груба.

Если замена параметра quiet на debug ничего не добавляет к выводу, то скорей всего, зависание происходит ДО запуска ядра.

Загрузчик читает в память файл с образом ядра (команда linux, перед которой в конфиге стоит сообщение "Загружается Linux ..."), потом файл с образом initramfs (команда initrd, перед которой выдаётся "Загружается начальный виртуальный диск ..." ), и только после выполнения всех команд груб должен передать управление загруженному ядру.

Можно точно установить, на каком именно этапе происходит зависание.
Для этого надо либо открыть консоль груба, и выполнить команды загрузки по одной, либо открыть пункт меню на редактирование, и дописать в конец (после всех строк) команду вроде 
echo "Starting kernel now"
Если вы НЕ увидите эту надпись при загрузке после "Загружается начальный виртуальный диск", то загрузчик не действительно почему-то не может прочитать initramfs (либо микрокод, попробуйте убрать его из команды initrd).
Если зависание произойдёт уже после этой строки – значит либо ядро не стартует, либо груб не может его запустить.
