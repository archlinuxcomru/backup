общесистемная фича, не зависящая от DE
Если общесистемно от cups-pdf
то https://wiki.archlinux.org/index.php/CUPS
проверяете службу
systemctl status org.cups.cupsd.service
если не запущена( active (running)) запускаете
systemctl start org.cups.cupsd.service

В браузере открываете страничку http://localhost:631
и добавляете принтер → добавить принтер → CUPS-PDF

В КДЕ должны быть свои инструменты
Добавьте в название темы KDE может кдешники подскажут получше
вроде system-config-printer и print-manager (/usr/bin/kde-add-printer --add-printer)

Добавил! Причём попробовал и через CUPS в браузере, и через системные параметры KDE.
В обоих случаях следует выбирать CUPS-PDF (Virtual PDF Printer) и далее Generic CUPS PDF Printer. Их там будет два варианта, с меньшим количеством настроек и с большим.
Спасибо!

P.S.: открыл для себя удобную настройку (здесь на форуме). Чтобы при изменении в принтерах система не спрашивала без конца пароль, можно добавить своего пользователя в массив @SYSTEM, который определяется в файле /etc/cups/cups-files.conf, строка SystemGroup. Как раз на этот массив и ссылается настройка в /etc/cups/cupsd.conf:
<Policy default>
... Require user @OWNER @SYSTEM
...
После этого рестартнуть CUPS.
