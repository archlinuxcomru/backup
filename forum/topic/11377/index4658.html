systemctl enable cups.service правильно?
Стоп, Вы что, устанавливаете принтер с незапущенной системой управления печатью? %(

и как теперь перезапускать cups?
systemctl restart cups.service

есть хитрость?
Никакой хитрости:
-rw-r--r-- 1 root root 14534 мар 11 12:08 /etc/cups/ppd/HP_LaserJet_P1005.ppd

говорит какого-то плагина нет
Понятное дело, без проприетарного плагина этот принтер не будет печатать даже со свободными дровами. Отдельно hplip-plugin есть в ауре. Можете качать с офсайта НР, но у меня почему-то постоянно не совпадала md5sum.

Давайте сначала. Я бы для гарантии удалил hplip, cups, почистил хвосты. Короче:
systemctl stop cups.service
systemctl disable cups.service
pacman -Rsn hplip cups
rm -r /usr/share/hplip
rm -r /var/lib/hp
# может еще что-то...
pacman -Syu cups hplip
systemctl start cups.service
systemctl enable cups.service
ln -sf /usr/bin/python2.7 /usr/bin/python
hp-setup -i
ln -sf /usr/bin/python3 /usr/bin/python
Разумеется, всё от рута. Не помню, нужна ли перезагрузка...

На всякий случай, если будут траблы с автоматической загрузкой прошивки после включения девайса:
/etc/udev/rules.d/86-hpmud-hp_laserjet_p1005.rules
——————————————————————————————————————————————————
# Load hp firmware for this printer.
ACTION!="add", GOTO="hpmud_rules_end"
# hp_laserjet_p1005
SUBSYSTEM=="usb", ENV{DEVTYPE}=="usb_device", ATTRS{idVendor}=="03f0", ATTRS{idProduct}=="3d17", RUN+="/usr/bin/systemd-cat -t hp-firmware -p5 /usr/bin/python2 /usr/bin/hp-firmware -n -y3 -s$env{BUSNUM}:$env{DEVNUM}"
LABEL="hpmud_rules_end"
# udevadm control --reload-rules
