Где-то месяц назад столкнулся с таким явлением, что имя моей машины в самбовской сети дублируется, в нижнем регистре, выглядит это приблизительно вот так:


DEBIAN - шлюз + мастер-браузер самбы.
KUZJA - убунтовский десктоп.
TATNJA и VASILY - вин7
Ну и VITOK в верхнем и нижнем регистрах, это моя арчевская машина.

smb.conf
[global]
	map to guest = Bad User
	log file = /var/log/samba/%m.log
	max log size = 50
	load printers = No
	dns proxy = No
	wins support = Yes
	usershare allow guests = Yes
	usershare max shares = 100
	usershare owner only = No
	usershare path = /var/lib/samba/usershare
	idmap config * : backend = tdb

из под убунты так-же видно дубликат имени vitok, на виндовых машинах дубликата не видно.

Во, еще заметил такую штуку, когда нахожусь в рабочей группе WORKGROUP (testparm почему-то не показал этой части конфига) то через убунту в WORKGROUP вижу имя vitok в обоих регистрах, когда на арче меняю группу на любую другую. то с убунты вижу в верхнем регистре VITOK в “новой” измененной группе, а vitok в нижнем регистре вижу в WORKGROUP.
Когда непосредственно с арча иду по очереди в эти группы, то vitok в нижнем регистре, не вижу, с арча его видно только в разделе “сеть” наутилуса. Виндовые машины vitok в нижнем регистре не видят ни при каких условиях.

В общем решено. Поскольку мне avahi-daemon нужен только для поиска сетевых принтеров, а именно mdns и dns-sd запросов, без публикации машины и ресурсов в сети, решением было выставить в avahi-daemon.conf
disable-publishing=yes
