У ext{234} файловых систем есть параметр "max-mount-count". Если текущее количество монтирований достигло этого значения, то автоматически будет запущена проверка fsck. Текущие значения можно посмотреть так:
sudo tune2fs -l /dev/sda1 | grep -i "mount count"
Установить другое значение max-mount-count или вообще запретить проверку (установив в 0) можно с помощью ключа -с

По второму вопросу с “BUG: unable to handle…” трудно что-нибудь вразумительное ответить на основании одной ошибки. Это могут быть проблемы с карточкой, проблемы с памятью и т.п. На основании встреченных в свое время подобных проблем, для начала советую:
1. С загрузочного образа запустить тест памяти. Если будут ошибки - уже понятно.
2. Если тест прошел без ошибок, все равно открыть корпус и переткнуть все модули памяти (вытащить, если есть возможность, почистить контакты, вставить аккуратно на место). То же самое можно и с другими картами, устанавливаемыми в разъемы мат.платы.
3. Если после этого ошибка не исчезнет, попробовать еще раз поднять эту тему с новыми логами и рассказом, что уже было сделано. Надо попробовать собрать логи за несколько дней. Будет ли повторяться адрес памяти, фигурирующий в сообщении? Будет ли подобная ошибка повторяться для одного и того же модуля?
