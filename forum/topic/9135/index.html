22.6 “This binary does not support kernel L2TP”

When you use l2tpd you will see the message “This binary does not support kernel L2TP” in /var/log/messages. This does not indicate an error. It is just an informational message. The original creators of l2tpd intended to implement support for both user mode and kernel mode L2TP. But they never got around to do kernel mode L2TP.

отсюда: http://www.jacco2.dds.nl/networking/freeswan-l2tp.html вторая ссылка на google.com/linux ;-)

Да, через ppp, конфиги из /etc/ppp/peers не используются.
3 конфига используются:
/etc/xl2tpd/xl2tpd.conf

access control = yes


lns = tp.ul.corbina.net
redial = yes
redial timeout = 1
require chap = yes
require authentication = no
name = sonikkk
ppp debug = yes
pppoptfile = /etc/ppp/options.xl2tpd
require pap = no
autodial = yes
/etc/ppp/options.xl2tpd
name sonikkk
remotename l2tp
ipparam corbina
connect /bin/true
nodeflate
nobsdcomp
persist
maxfail 0
nopcomp
noaccomp
defaultroute
/etc/ppp/chap-secrets
