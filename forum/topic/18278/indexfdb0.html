Добрый день.
Столкнулся с такой же ситуацией: помогите разобраться, чтобы nemo сам искал шары.
Сервисы nmbd.service smbd.service и avahi-daemon.service запущены
Стоит Cinnamon и nemo (с пакетом gvfs-smb) не ищет шары windows.
При этом компьютеры Win прекрасно ищут и заходят ко мне.
Сеть -> Сеть Windows -> и все, в командной строке только smb:///
При этом smbtree прекрасно видит компы в сети и можно подключиться через ФМ, если набрать в строке:
smb://Имя_компьютера/Шара правда запрашивая пароль пользователя.
***
мой smb.conf:
[global]
netbios name = Virty
workgroup = WORKGROUP
server string =
security = user
map to guest = bad user
log file = /var/log/samba/log.%m
max log size = 10
interfaces = 192.168.10.1/24 192.168.10.194/24 попробовал и с этим
dns proxy = no
dos charset = cp866
unix charset = UTF-8
force user = virt
force group = users
server min protocol = SMB2 попробовал и с этим

[tmp] - для примера
comment = Temporary file space
path = /tmp
browseable = yes
read only = no
public = yes
guest ok = yes
