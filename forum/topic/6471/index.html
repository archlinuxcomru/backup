eth0="dhcp"
INTERFACES=(eth0)
#gateway="default gw 192.168.0.1"
ROUTES=(!gateway)
DAEMONS=(syslog-ng hal network netfs crond acpid cpudyn cpufreq laptop-mode alsa @adsl slim @deluged)

1. Судя по логам, не поднимается eth0. Клиент dhcp не видит broadcast сообщений, из-за чего отваливается по тайм ауту.
1.1. Проверьте настройки файервола.
1.2. Обратитесь за помощью к локальному администратору dhcp сервера.
2. Зато успешно стартует adsl, для которого поднятие eth0 не обязательно, поэтому интернет всё-таки есть.
3. Демон adsl лучше поместить между network и netfs, убрав перед ним @
Разберемся, голубчик!
