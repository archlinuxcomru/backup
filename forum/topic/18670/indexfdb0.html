проблема решилась простым копированием ВСЕХ ttf-шрифтов в системный каталог - /usr/share/fonts/TTF
Дак они по дефолту должны там и находится (система размещает их там автоматом).
Общепринято - пути шрифтов, изначально известные Fontconfig, находятся в: /usr/share/fonts/,  ~/.local/share/fonts. Fontconfig будет сканировать эти каталоги рекурсивно. Чтобы увидеть список известных Fontconfig шрифтов, используется команд: fc-list : file
Кроме того, как писал выше, всегда можно посмотреть, куда лезет приложение - например, за шрифтами firefox лезет в следуюшие директории/файлы
/etc/fonts/fonts.conf
/etc/fonts/conf.d/
/usr/share/fonts/
/usr/lib/firefox/fonts
/home/vasek/.local/share/fonts/

UPD - ls /usr/share/fonts
100dpi 75dpi OTF TTF cantarell cyrillic encodings gsfonts misc util

fc-list : file | grep TTF
/usr/share/fonts/TTF/DejaVuSerif-BoldItalic.ttf:
/usr/share/fonts/TTF/LiberationMono-BoldItalic.ttf:
......................
