Столкнулся с этим ещё в ноябре
Сколько ни гуглил, ничего полезного не нашёл. Единственное - эта гадость вылезает не только в арче, но и в других системах - есть баг на лончпаде и в федоровской багзилле. Решения проблемы нет. Нет даже чёткого понимания, из-за чего это может появляться.
Самое полезное, что удалось найти - комментарий на реддите
It shouldn't be an issue, these errors are caused by a new kernel patch introduced to help track down namespace errors in applications. (At least, that's my understanding, YMMV.)
These used to be ignored, but as ancient, "pre-namespace" x86 code is being depreciated, kernel developers are adding in new features to make logging and bug fixing more efficient and appropriate to modern coding. You'll probably see more of these "errors" in the short-term, but they'll go a long way to helping application developers avoid conflicts with other applications and system software.

P.S> наблюдается на ядрах 4.14.*
