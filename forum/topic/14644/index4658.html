Если используешь параметр acpi_backlight=vendor , то попробуй пропиши дополнительно параметр video.use_native_backlight=1 (по умолчанию этот параметр = -1).
У меня нормально работают оба интерфейса и acpi_video0 и intel_backlight, но отличия имеются
- обычная загрузка, без параметров,
$ cat /sys/class/backlight/intel_backlight/brightness
181
- загрузка с параметрами acpi_backlight=vendor video.use_native_backlight=1
$ cat /sys/class/backlight/intel_backlight/brightness
192

$ cat /sys/class/backlight/intel_backlight/brightness

Some system's ACPI video backlight control interface is broken and the native backlight control interface should be used by default. This patch sets the use_native_backlight parameter to true for those systems so that video backlight control interface will not be created. To be specific, the ThinkPad T430s/X230/X1 Carbon, Lenovo Yoga 13, Dell Inspiron 7520, Acer Aspire 5733Z and HP ProBook 4340s are added here, if they appear in some other DMI table before, they are removed from there.
Note that the user specified kernel cmdline option will always have the highest priority, i.e. if use_native_backlight=0 is specified and the system is in the DMI table, the video module will not skip registering backlight interface for it.
А лучше всегда смотреть исходники — загляни в …./drvirs/acpi/video.c
В дополнение к сказанному, можешь посмотреть, что модуль video имеет 3 параметра
$ modinfo -p video
brightness_switch_enabled: (bool)
allow_duplicates: (bool)
use_native_backlight: (int)
значение которых (всех сразу) можно посмотреть так
$ cat /sys/module/video/parameters/*
N
Y
-1
Варирование этими параметрами вносит определенные изменения в управление подсветкой.
PS...........имеется еще один параметр, влияющий на подсветку в vendor drivers (в моем случае это intel)
$ modinfo i915 | grep brightness
parm: invert_brightness:Invert backlight brightness (-1 force normal, 0 machine defaults, 1 force inversion), please report PCI device ID, subsystem vendor and subsystem device ID to [email protected], if your machine needs it. It will then be included in an upcoming module version. (int)
