Пробовал и так:

и так:

Может, дело в этом:
[    5.292592] xc2028 7-0061: Could not load firmware xc3028-v27.fw.

А если убрать все modprobe.d то получается такое:

дек 20 07:03:50 localhost kernel: tuner 7-0061: Tuner -1 found with type(s) Radio TV.
дек 20 07:03:50 localhost kernel: tuner 7-0061: tuner type not set

UPD: Вопрос решен так: http://blog.beford.org/2010/11/27/prolink-pixelview-mpeg-8000gt-en-archlinux/
cd /lib/firmware/
wget http://beford.org/linux/xc3028-v27.fw

nano /etc/modprobe.d/modprobe.conf :

options cx88xx card=66 tuner=71
options snd-hda-intel index=0
options cx88_alsa index=1

Теперь бы звук сделать :)

UPD2: Со звуком тоже появились подвижки, хотя до сих пор не удается юзать звук через альзу:
$cat /etc/modprobe.d/modprobe.conf
options cx88xx card=66 tuner=71 alsa=1
options snd-hda-intel index=0
options cx88_alsa index=1
install snd /usr/bin/modprobe --ignore-install snd && { /usr/bin/modprobe --quiet snd-ioctl32 ; : ; }
install snd /usr/bin/modprobe --ignore-install snd && { /usr/bin/modprobe -Qb snd-seq ; }
install snd-pcm /usr/bin/modprobe --ignore-install snd-pcm && { /usr/bin/modprobe --quiet snd-pcm-oss ; : ; }
install snd-mixer /usr/bin/modprobe --ignore-install snd-mixer && { /usr/bin/modprobe --quiet snd-mixer-oss ; : ; }
Теперь можно смотреть с тюнера с mplayer (с mplayer, правда, начинает сильно тормозить) и mpv:
для mplayer, строка такая:
mplayer -tv device=/dev/video0:input=0:normid=13:chanlist=russia:alsa:adevice=/dev/audio1:immediatemode=0:audiorate=48000:amode=1:adevice=hw.1,0 tv://R-2
Кстати с mplayer узнал одну хитрость: есть режим, где плеер ожидат в терминале консольную команду, отсюда:
mplayer tv:// -slave -quiet -tv driver=v4l2:chanlist=russia:normid=4
Здесь chanlist=russia - российская сетка частот, normid=4 - выбор системы цветности (4=PAL) большинства передаваемых в вашем регионе каналов. При этом запустится окно mplayer с 1 каналом в сетке и консоль будет ожидать ввода управляющих команд.
