Вибер, ватцап, сигнал, трима требуют сканирования телефоном с приложением экранного баркода, что неприемлемо.

Jami пиринговая, p2p, предполагается прямое соединение собеседников (контакт - через сигнальный сервер) - не не гонку трафика через всех участников (при уникальности аудио-видео потока это представляется бессмысленным, разве что сигнальный распределенный сервер через всех участников). Так когда-то, на момент взлета, скайп работал. Сейчас так может Трима.

ТЛГ в свое время сильно расстроил (как и Сигнал) - невзирая на запрет доступа к Контактам регулярно присылает сообщения вида "такой-то стал использовать ТЛГ". Понятно, что в ноуте такого нет, но в моих глазах это признак недобросовестности разработчиков.

Но очень не хочется разводить у себя зоопарк клиентов, в т.ч. в телефоне. Я соггласен включить - поиспользовать - выключить. Телеграмм после использования выключается только через диспетчер процессов или kill. Конечно можно запускать всех их через скрипт, который после завершения kill-ит все хвосты. Но...

Продолжу про Jami. Он запускается так -
Exec=jami-gnome --restore-last-window-state

оставляет после себя кучу дочерних процессов вида
gnome-jami
WebKitNetworkProcess
WebKitWebProcess

Пытаюсь решить задачу их автоприбития после выхода - фигушки.

Казалось бы привычное вида -
ps ax | grep WebKitWebProcess | awk '{ print $1 }' && ps ax | grep WebKitNetworkProcess | awk '{ print $1 }' && ps ax | grep dring | awk '{ print $1 }'

показывает ПИДы родительских процессов и скормив это kill -9 должно решать задачу. Однако нет, процессы не убиваются - в процессе выполнения команды они успевают переродиться с новыми ПИДами.

Вопрос - как прибить? - решилось так -

kill -9 `ps ax | grep jami-gnome | awk '{ print $1 }' && ps ax | grep dring | awk '{ print $1 }' && ps ax | grep WebKitWebProcess | awk '{ print $1 }' && ps ax | grep WebKitNetworkProcess | awk '{ print $1 }' `

Запуск в терминале вида -
jami-gnome --restore-last-window-state && kill -9 `ps ax | grep jami-gnome | awk '{ print $1 }' && ps ax | grep dring | awk '{ print $1 }' && ps ax | grep WebKitWebProcess | awk '{ print $1 }' && ps ax | grep WebKitNetworkProcess | awk '{ print $1 }' `

по нажатии в Меню команды Выход задучу решает. Однако трюк вида в файле .desktop
Exec=jami-gnome --restore-last-window-state && kill -9 `ps ax | grep jami-gnome | awk '{ print $1 }' && ps ax | grep dring | awk '{ print $1 }' && ps ax | grep WebKitWebProcess | awk '{ print $1 }' && ps ax | grep WebKitNetworkProcess | awk '{ print $1 }' `

результата не дает.
