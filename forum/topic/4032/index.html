Спасибо за ссылку. Там написано, кстати, не только как заткнуть kernel-messages вообще, но и как вернуть старое поведение – то есть чтобы loglevel понижался после старта /etc/rc.sysinit . Только там небольшая ошибка, не хватает последней строчки add_hook . Согласно комментариям в /etc/rc.d/functions , правила такие: 
# Define a hook function in a functions.d file using:
#  function_name() {
#    ...
#  }
#  add_hook hook_name function_name
Поэтому, чтобы вернуть старое поведение, нужно сделать так: 
echo 'sysinit_start () {
        dmesg -n 3
}
add_hook sysinit_start sysinit_start' > /etc/rc.d/functions.d/dmesg
