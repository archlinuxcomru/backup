Обновился, получаю то же самое
tilix: error while loading shared libraries: libphobos2-ldc.so.75: cannot open shared object file: No such file or directory
Могу порекомендовать, пока не исправят (чувствую, что это будет повторятся постоянно) следующий временный путь
Создаем симлинк - sudo ln -s /usr/lib/libphobos2-ldc.so.76 /usr/lib/libphobos2-ldc.so.75
Проверяем всех ли библиотек хватает
$ ldd /usr/bin/tilix | grep 'not found'
libdruntime-ldc.so.75 => not found
libdruntime-ldc.so.75 => not found
libdruntime-ldc.so.75 => not found
Оказывается, что нехватает еще одной. Создаем еще один симлинк - sudo ln -s /usr/lib/libdruntime-ldc.so.76 /usr/lib/libdruntime-ldc.so.75
Запускаем tilix - все нормально
UPD - Нужно следить за обновлением tilix и libphobos и не забывать удалять созданные симлинки - /usr/lib/libphobos2-ldc.so.75 и /usr/lib/libphobos2-ldc.so.75
Ошибки не исчезают с опытом - они просто умнеют

vasek
Прежде чем переходить на другой, нужно понять, чем они отличаются.
Имхо, скорее всего ничем. Просто один собирается из исходников на локальной машине с теми опциями, которые указаны в makepkg.conf, а второй уже собран со стандартными.
Т.е., если кто-то у себя использует -O3 или что-то типа -march=native (или какие-то другие флаги), то естественно лучше собирать самому. Если устраивает стандартное -O2, то и уже собранный вполне подойдёт.
