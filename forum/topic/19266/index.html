Периодически слетают пароли от google аккаунтов в chromium. Посоветовали отказаться от gnome-keyring и установить программы для сохранения паролей:
Удалил пакеты: gnome-keyring seahorse.
Пароли от аккаунтов google все равно сохраняются. Насколько я понимаю в папке:
/home/sergey/.config/chromium/
Возникла другая проблема, когда я подключаю облачные хранилища через webdav, файловый менеджер thunar. Пароли не сохраняются после перезагрузки хотя стоит галочка запомнить навсегда. Насколько я понимаю они должны хранится в связке ключей:
~/.local/share/keyrings/
которые я удалил.
Есть ли какое-то решение проблемы для chromium и webdav через thunar без использования gnome-keyring? Заранее благодарен за ответ.

Ну, для файлманагеров удобнее через davfs2
К примеру, для яндексдиска и box :
добавить в /etc/davfs2/secrets
https://webdav.yandex.ru:443 логин пароль
https://dav.box.com/dav логин пароль
Создать в /home/user папки yandex и box, добавить в /etc/fstab (монтирование по клику):
https://webdav.yandex.ru /home/valera/yandex davfs uid=valera,file_mode=640,dir_mode=755,users,_netdev,noauto 0 0
https://dav.box.com/dav/ /home/valera/box davfs uid=valera,file_mode=640,dir_mode=755,users,_netdev,noauto 0 0
----
С правами не заморачивался, для себя просто в usr/share/polkit-1/actions/org.freedesktop.udisks2.policy в секциях с:
..filesystem-mount
..filesystem-mount-system
..filesystem-mount-system
..filesystem-fstab
..filesystem-unmount-others
заменил <allow_active>auth_admin_keep</allow_active> на <allow_active>yes</allow_active>
--
Правда , от обновления udisks2 это приходится прописывать снова.
