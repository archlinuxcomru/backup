[РЕШЕНО] После обновления отвалилась русская локаль

Вчера обновил систему. Пакеты glib2, avahi, gcc-libs, boost-libs, coreutils, gcc, openjdk6, icedtea-web, udev, initscripts, inputproto, kernel26, libpurple, net-tools, nvidia, pidgin, qt.
Опасался, что в связи с изменениями в initscripts будут проблемы с интернетом или вообще не запустится система. Отвалилась локализация. Этот участок rc.conf я не менял:
LOCALE=“en_US.UTF-8”
DAEMON_LOCALE=“no”
HARDWARECLOCK=“UTC”
TIMEZONE=“Europe/Moscow”
KEYMAP=“ru”
CONSOLEFONT=“ruscii_8x16”
CONSOLEMAP=
USECOLOR=“yes”
VERBOSE=“3”
В locale.gen раскомментирована одна строка: ru_RU.UTF-8 UTF-8.
Команда locale -a выдает:
locale: Cannot set LC_CTYPE to default locale: No such file or directory
locale: Cannot set LC_MESSAGES to default locale: No such file or directory
locale: Cannot set LC_COLLATE to default locale: No such file or directory
C
POSIX
ru_RU.utf8
Впервые столкнулся с проблемой, потому не знаю, с какого края ее решать.
Спасибо.
