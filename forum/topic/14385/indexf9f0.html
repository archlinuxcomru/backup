This is intentional, see https://blogs.gnome.org/mclasen/2019/05/25/pango-future-directions/
https://bugs.archlinux.org/task/63297

Stop using freetype
Freetypes FT_Face object has locking semantics that are broken and hard to work with; they are constantly getting in the way as we are juggling hb_fonts, FT_Face and cairo scaled font objects.

We’ve concluded that the best way forward is to stop using freetype for font loading or accessing font and glyph metrics. We can use Harfbuzz for all of these (a small gap will be closed soon).

Using Harfbuzz for font loading means that we will lose support for bitmap and type1 fonts. We think this is an acceptable trade-off, but others might disagree. Note that Harfbuzz does support loading bitmap-only OpenType fonts.

Меняйте шрифты

и по теме еще https://gitlab.gnome.org/GNOME/pango/issues/386

В связи с тем, что во многих местах прошла, мягко говоря, не совсем верная информация... По шрифтам:
формат файлов PCF для растровых шрифтов больше не поддерживается pango/harfbuzzz, но поддерживается другой формат OTB (OpenType Bitmap). Всеми любимый Terminus в новом формате уже существует . Кроме того похоже, любой OTB можно сформировать из PCF с помощью fontforge. По крайней мере, мой любимый Unifont я именно так и "переформатировал".
