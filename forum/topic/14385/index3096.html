Забей, на работу системы это не влияет. Если эти файлы будут накапливаться, их смело можно удалять.
Эти файлы (.serverauth.nnn) создаются во время запуска X-сервера и удаляются при выходе из системы. Но в случае нюансов с завершением X-ов, эти файлы не удаляются
Each time you start an X windows session, that file gets created and it is deleted when X session exits normally. If startx exits ungracefully - for instance, if the shutdown sequence sends it a kill signal (the script doesn't catch signals) - the file will never be removed.
Как мог заметить, сейчас имеются нюансы и с shutdown.
Имхо, лучше ничего не трогать и не менять. Подождем изменений.

EDIT 1 - проверил, у меня эти файлы не накапливаются - каждый раз создается новый, старый удаляется. Значит все нормально.
EDIT 2 - да раньше этого файла не было, видимо что то изменили - что конкретно, не вникал, будем считать, что так и нужно.
Ошибки не исчезают с опытом - они просто умнеют
