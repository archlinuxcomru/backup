“Я не логинюсь через su, только sudo и реже sudo -s, еще реже sudo -i (для этого случая я так не логинился), пользуюсь эмулем urxvt - да”

ответа на п1. было достаточно: при sudo, бай-дефолт переменные окружения не сбрасываются, сиречь, после перезапуска cron подцепляет DISPLAY переменную, которой просто на момент старта не существовало (как и не существовало запущенных иксов, в общем, команда set без аргументов тебе в подарок)
