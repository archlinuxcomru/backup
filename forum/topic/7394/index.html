Добрый день. Похожей темы не нашел, поэтому создал новую. Вопрос в следующем:
есть ноутбук Acer Aspire 6935G. lspci -k:
00:1b.0 Audio device: Intel Corporation 82801I (ICH9 Family) HD Audio Controller (rev 03)
Subsystem: Acer Incorporated Device 0146
Kernel driver in use: HDA Intel
Kernel modules: snd-hda-intel
Установлена alsa. Звук есть, а вот микрофон работать не хочет. Кроме того, при попытке увеличить mic boost начинают проявляться шумы. В Скайпе в Тестовом сервисе при воспроизведении слышен один посторонний шум.
Интересен тот факт, что микрофон работал в Ubuntu 9.10

запускаешь alsamixer
Front Mic = 0
нажимаешь F4
Front Mic Boost = 0
Input source - Int Mic
Digital = 50%
Capture = 50%
Capture - нажимаешь пробел, появляется красная надпись CAPTURE
пробуешь микрофон, удобнее через стандартную утилиту Звукозапись, настраиваешь уровни Capture и Digital по своему усмотрению
закончив, нажимаешь Esc
sudo alsactl store - сохраняешь текущие настройки
в автозапуск нужно засунуть alsactl restore, для возобновления настроек при перезагрузке

Не могу найти сам решение. В общем дело такое - при записи звука с микрофона звук получается очень искаженным, хрипящим, с артефактами. В звукозаписи, в аудасити, в скайпе… Бывает проскальзывает удачная запись, но в 90% попыток что то записать выходит полная лажа.

ICH9 Family HD Audio Controller, чип ALC888, дрова последние с сайта реалтек (альса 1.0.22.1 с патчами реалтека, которая всегда кстати отлично работала).

В микшере все настроено кажись правильно, .asoundrc чист как слеза младенца.  В modprobe.conf прописаны алиасы нужные 

   # ALSA portion
          alias char-major-116 snd
          alias snd-card-0 snd-hda-intel 
        # OSS/Free portion
          alias char-major-14 soundcore
          alias sound-slot-0 snd-card-0
        # card #1
          alias sound-service-0-0 snd-mixer-oss
          alias sound-service-0-1 snd-seq-oss
          alias sound-service-0-3 snd-pcm-oss
          alias sound-service-0-8 snd-seq-oss
          alias sound-service-0-12 snd-pcm-oss
