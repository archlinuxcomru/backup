Что-то пошло не так и теперь gnome-boxes и gimp, при попытке открыть файл просят пароль. То есть окно polkit вылазит(Gnome 3.22). Раньше не было такого.

Вот журнал пишет после отмены ввода пароля. Всё вроде работает, но запрос теперь раздражает.
ноя 23 01:07:09 arch-linux pkexec[823]: pam_unix(polkit-1:session): session opened for user root by (uid=1000)
ноя 23 01:07:09 arch-linux pkexec[823]: deadmetaler: Executing command [USER=root] [TTY=unknown] [CWD=/home/deadmetaler] [COMMAND=/usr/lib/gvfs/gvfsd-admin --spawner :1.5 /org/gtk/gvfs/exec_spaw/0 --address unix:path=/run/user/1000/bus]
ноя 23 01:07:09 arch-linux kernel: capability: warning: `gvfsd-admin' uses 32-bit capabilities (legacy support in use)
ноя 23 01:07:13 arch-linux polkitd[369]: Operator of unix-session:c1 FAILED to authenticate to gain authorization for action org.gtk.vfs.file-operations for unix-process:784:1997 [gimp-2.9] (owned by unix-user:deadmetaler)

Файл /usr/share/polkit-1/rules.d/org.gtk.vfs.file-operations.rules присутствует. Принадлежит gvfs. С таким содержанием:
polkit.addRule(function(action, subject) {
        if ((action.id == "org.gtk.vfs.file-operations-helper") &&
            subject.local &&
            subject.active &&
            subject.isInGroup ("wheel")) {
            return polkit.Result.YES;
        }
});

Я сопоставил журнал и конфиг, заменил во второй строке org.gtk.vfs.file-operations-helper на org.gtk.vfs.file-operations и сообщения пропали. Но это не решение.
Gvfs переставлял, что не так?
