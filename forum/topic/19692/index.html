Переустановка ничего не даёт. Установил пока xfce-theme-cobibird, но всёравно это не то.

Вот дословный перевод на wiki, но я что то не соображу что делать:
Ваш PKGBUILD плохой! Это фактически не помещает файлы в пакет. Файлы будут установлены в вашей системе в тот момент, когда вы увидите запрос пароля от ниндзя. Теперь в вашей системе есть файлы, которые pacman не может удалить.
Вот другой PKGBUILD, который работает так, как должен:
http://ix.io/27wz
Хитрость в этом PKGBUILD - переменная окружения "DESTDIR = $ pkgdir". Это заставляет ниндзя помещать файлы туда, где их ожидает makepkg.

А не проще темы не устанавливать, а скачать их и поместить в .themes. а для запуска/выбора тем установить пакет lxappearance - далее все очень просто - с помощью lxappearance выбираешь нужную тему из списка (будет показан при запуске lxappearance), надоело, сменил - все просто и не нужно темы устанавливать, тем более связываться с AUR.

В твоем случае - заходишь в AUR, скачиваешь xfce-theme-greybird-3.22.11.tar.gz (перед Latest Comments), распаковываешь, создаешь в .themes две директории Greybird-dark и Greybird-light
В Greybird-dark копируешь две директории gtk-2.0 и gtk-3.0 из Greybird-3.22.11/dark
В Greybird-light копируешь две директории gtk-2.0 и gtk-3.0 из Greybird-3.22.11/light
В принципе, думаю, этого должно быть достаточно - попробуй.
Ошибки не исчезают с опытом - они просто умнеют

sudo yum install rubygem-sass glib2-devel gdk-pixbuf2-devel -y
git clone https://github.com/shimmerproject/Greybird.git && cd Greybird
./autogen.sh
make -j4
sudo make install
