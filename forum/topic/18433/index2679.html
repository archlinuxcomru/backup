Чтобы не увидеть ошибку "Error 403 - Access forbidden!" при попытке открыть phpmyadmin
отредактируйте файл '/etc/httpd/conf/extra/phpmyadmin.conf' как показано ниже:

Мой файл phpmyadmin.conf
Alias /phpmyadmin "/usr/share/webapps/phpMyAdmin"
     <Directory "/usr/share/webapps/phpMyAdmin">
          AllowOverride All
          Options FollowSymlinks
          Order allow,deny
          Allow from all
     </Directory>
Но это не помогает, в чем может быть проблема?

Проблему с доступом решил, исправил /etc/httpd/conf/extra/phpmyadmin.conf на

Alias /phpmyadmin "/usr/share/webapps/phpMyAdmin"
<Directory "/usr/share/webapps/phpMyAdmin">
    DirectoryIndex index.php
    AllowOverride All
    Options FollowSymlinks
    Require all granted
</Directory>
Перезагрузил apache

sudo systemctl restart httpd
Перешел по адресу http://localhost/phpmyadmin, выдало ошибку

Error during session start; please check your PHP and/or webserver log file and configure your PHP installation properly. Also ensure that cookies are enabled in your browser. session_start(): open_basedir restriction in effect. File(/tmp) is not within the allowed path(s): (/home/skyrewir/Sites:/usr/share/webapps/:/etc/webapps) session_start(): Failed to initialize storage module: files (path: )
Исправляется

/etc/php/php.ini

open_basedir = /home/skyrewir/Sites:/usr/share/webapps/:/etc/webapps[b]:/tmp[/b]
Перезагрузил apache

sudo systemctl restart httpd
