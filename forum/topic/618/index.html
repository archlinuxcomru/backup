Нужно иногда подключаться по ssh. Из за этого при загрузке хочу отключать X, который запускается по умолчанию через gdm. Для отключения X решил использовать runlevel 3. Для установки runlevel 3 жму e на пункте меню GRUB2, далее в конце строки linux ставлю 3, жму f10 после сообщения udev выводится init runlevel 3, в конечном итоге X не отключаются и появляется gdm. Проверял с runlevel 0 - комп успешно отключается. Вопрос, что не так делаю и как можно ещё иногда отключать запуск по умолчанию X, чтобы работать по ssh?

Чтобы работать по SSH, не обязательно отключать X

Если речь идёт о systemd, то для загрузки без X в параметры ядра можно добавить systemd.unit=multi-user.target
Если об обычном init, достаточно цифры 3
(systemd пока тоже поддерживает цифры для совместимости)

Если gdm не отключается на уровне 3, значит вы зачем-то добавили его в список демонов. Уберите его оттуда.

Если gdm не отключается на уровне 3, значит вы зачем-то добавили его в список демонов. Уберите его оттуда.
Но gdm в демонах нужен для автоматического запуска X.

Решил отключать X т.к. через gnome 3 настроил переход в спящий режим через 10 мин простоя. Соответственно gnome 3 переведёт в сон комп и по ssh уже не подключишься, поэтому решил отключать X при загрузке, чтобы не запускался gnome 3.
