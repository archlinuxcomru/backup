Возник такой вопрос
стоит extra/dhcp 4.1.0.p1-1
имеется под него конфиг
при таком запуске руками
# dhcpd -cf /etc/dhcpd.conf eth2
рабоатает как надо
в rc.conf имеется следующее
#Static IP example
#eth0="eth0 192.168.0.2 netmask 255.255.255.0 broadcast 192.168.0.255"
eth0="dhcp"
wpa_supplicant -Dwired -ieth1 -c/etc/wpa_supplicant.conf -B
eth1="dhcp"
eth2="eth2 192.168.1.1 netmask 255.255.255.0 broadcast 192.168.1.255"
INTERFACES=(!eth0 eth1 eht2)
# Routes to start at boot-up (in this order)
# Declare each route then list in ROUTES
#   - prefix an entry in ROUTES with a ! to disable it
#
gateway="default gw 192.168.0.1"
при запуске службой, то есть вот так выдается такое:
]# /etc/rc.d/dhcpd start                                                          
ctrl_iface exists and seems to be in use - cannot override it                                    
Delete '/var/run/wpa_supplicant/eth1' manually if it is not used anymore                         
Failed to initialize control interface '/var/run/wpa_supplicant'.                                
You may have another wpa_supplicant process already running or the file was                      
left by an unclean termination of wpa_supplicant in which case you will need                     
to manually remove this file before starting wpa_supplicant again.                               
:: Starting DHCP Server                                                                   [FAIL]
понятно что почему-то dhcpd пытается сесть на интерфейс eth1. Гугли ответ не нашел почему так. Как объяснить dhcpd в rc.conf что ему надло садится на интерфейс eth2?

Решено… странно но решено :)
в rc.conf
# -----------------------------------------------------------------------
# DAEMONS
# -----------------------------------------------------------------------
#
# Daemons to start at boot-up (in this order)
#   - prefix a daemon with a ! to disable it
#   - prefix a daemon with a @ to start it up in the background
#
#dhcpd_enable="YES"
dhcpd_ifaces="eth2"
DAEMONS=(syslog-ng dbus hal network netfs @crond alsa cups dhcpd  sshd fam !kdm)
(Это из мануала по фряхе)

 /etc/rc.d/dhcpd restart
