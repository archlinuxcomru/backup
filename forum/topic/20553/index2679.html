1. showkey -k работает только в консоле
2. Если нет вывода от xev (и других команд), нужно смотреть логи, если клавиша действительно не определяется, то должно быть сообщение в dmesg.
3. Обычно, если мультимединые клавиши в linux не работают, то настроить их поблематично ... но всегда можно назначить нужную команду, необходимую для данной операции, на определенную комбинацию клавиш (даже на ту же самую, Fn+F10, лишь бы она определялась ядром).

при просмотре dmesg -w нет событий связанных с этой кнопкой
Запусти 2 терминала:
- в одном запусти режим online - journalctl -f
- в другом команду
xev | awk -F'[ )]+' '/^KeyPress/ { a[NR+2] } NR in a { printf "%-3s %s\n", $5, $8 }'
и нажимай на комбинации клавиш: Fn+F2, Fn+F3 и далее Fn+F10.
Если клавиша не определяется (имеет проблемы), то в логах будет что то типа
Unknown key pressed (translated set 2, code 0xbd on isa0060/serio0). Use 'setkeycodes e03d <keycode>' to make it known.

Можно для надежности запустить sudo evtest , выбрать клавиатуру и понажимать клавиши ...
