Текущая версия, которая помечена как пререлиз (в AUR доступна версия с указанного коммита) также предоставляет упощенный вариант виджета, предоставляющий панельку со списком рабочих столов (PTM Desktop panel вроде зовется). Вид отдельных полей настраивается, доступны теги - общее число рабочих столов, номер данного рабочего стола, метка (показывается, если рабочий стол активен, настраивается отдельно), имя данного рабочего стола. Доступно выделение цветом активного рабочего стола. И по клику переключается на выбранный рабочий стол.

Оба виджета также теперь позволяют отключить у себя фон насовсем (a-la conky).

Полноценный релиз 1.11.0 (мб будет отмечен, как 2.0, если придется много переписывать для адаптации) будет ориентировочно в сентябре (по крайней мере, не раньше появления KF5 и особенно python-bindings) в репах.

Проект переписан полностью на C++. В связи с этим, он переименован в awesome-widgets.
Версия 2.0.0 будет выложена как только будут готовы release notes и скопированы имеющиеся переводы из старого проекта. Для пользующих просьба обратить внимание на эти самые release notes, т.к. я немного сломал api =) (ну то есть ваши старые настройки точно не будут работать с новым виджетом)

В принципе, настройка упростилась - теперь там одно текстовое поле с кучей доступных тегов. С учетом кастомного html-форматирования можно получить что то весьма похожее на коньки не прибегая к самим конькам.

Важное отличие - теги теперь обрамляются с двух сторон знаком $

к релизу (2.1.0) готовится:
- ряд улучшений/изменений в бекенде, из важных - улучшение в работе Desktop Panel, в топку DBus, теперь он прибит гвоздями к KWin (вернее, к стандартам X через либы kwin)
- графический тултип-превьюшка для Desktop Panel (рисует границы окон на выбранном рабочем столе)
- убраны кастомные команды. Совсем. Вместо них более гибкие кастомные скрипты. Можно прикрутить определенные действия. Был фич реквест
- графические бары для ряда тегов. Был фич реквест.

Core часть уже готово, осталось довести до юзабельного состояния. К релизу подготовлю подробную хаутушку по последним пунктам в говнобложике.

Планы на будущее:
- портировать на kf5. Сейчас никак, связался с разрабами через форум - молчат пока, в список рассылки письмо не заапрувили. Или ждать пока запилят нужные методы в либы.
- улучшить модульность, в идеале: 1 тег - 1 модуль.
- сделать версию на чистом Qt для пользователей LXQt (или с использованием API LXQt).

Версия 2.2.0 помечена как бета, версии 2.1.* будут содержать только обновления, связанные с dataengine и фиксы критических мест.

2.2.0:
* портирование на ветку Plasma5
+ окна теперь рисуются по настоящему превьюшкой вместе с рабочим столом и их положением (может быть неправильный порядок окон!)
+ добавлена возможность показывать список окон
+ добавлены уведомления на разные системные события - изменение статуса адаптора питания, высокая загрузка CPU/RAM, использование свопа, мало места на диске, изменилось активное сетевое устройство
+ добавлена возможность скастовать тег (в настройках и из контекстного меню)
- убрана возможность сокрытия панелей (в wish-list)

2.1.1:
* обновление под новый стандарт системного монитора

общее:
+ добавлен тег netdev
+ добавлена поддержка smartctl
* оптимизация работы extsysmon, в частности убраны вызовы всех внешних команд, каких можно
* изменены настройки пакетного менеджера (в сторону большей интуитивной понятности)

К выходным будет в aur

Недавно создал тег 3.0.1. Смена мажорной версии вызвана значительными изменениями в бекенде, которые аффектят другие проекты (если они есть). Для простых пользователей обновление должно пройти гладко. Наиболее значимые изменения с версии 2.2.0:

3.0.1:
корректирующий релиз для старых версий qt

3.0.0:
+ поддержка лямбда выражений на qml (js-like яп)
+ новые теги
+ фильтры из коробки при получении списка обновлений (больше не надо использовать grep!)
+ опциональные иконки погоды (раньше были utf8 символы)
+ худо бедная поддержка графиков внутри виджета на основе бекенда GraphicalItem
+ импорт/экспорт настроек
* багфиксы
* улучшения в UI
* оптимизации, включая свою (а не плазменную) мультипоточность
* как и всех взрослых проектов у нас теперь есть CONTRIBUTING.md. Также код подведен под него.

2.4.0:
+ погодный информер
+ новые теги
* багфиксы
* значительные оптимизации
* апдейт под последние версии плазмы

2.3.3:
* нормальные шрифты на панели, а не мыло!

2.3.2:
* возможность показа скорости загрузки/отдачи в мб/с
* улучшения в расчете заряда батареи для ноутбуков с несколькими батареями (теперь используются емкости)

2.3.1:
* улучшена поддержка тегов, связанных с lmsensonrs (температура, скорость кулера). В частности, раньше могла быть ситуация с пропущенными значениями

2.3.0:
+ поддержка фильтров в кастомных скриптах (замена символов или их последовательностей на другие)
+ новые теги
+ добавлены таймауты интернет запросов, реализованные дополнительным костылем
* улучшения в UI

2.2.2:
+ поддержка котировок с использованием сервиса yahoo finance
+ кеширование ключей, которое нужно, чтобы пронумерованные значения не менялись от перезагрузок
* улучшение работы с обновлением размера виджета, в частности появилась возможность не полагаться на автоматику, а самому указать желаемый размер
* багфиксы
