Благодаря обновлению Linux ядра до версии 3.0, перемещенному в , пакет kernel26 переименован в linux. Аналогичные изменения сделаны для имен файлов ядра и образа инициализации:

  • vmlinuz26 теперь vmlinuz-linux
  • kernel26.img теперь initramfs-linux.img
  • kernel26-fallback.img теперь initramfs-linux-fallback.img

    Для предотвращения ситуации «не было печали», были созданы символические ссылки указывающие на старые имена файлов. Мы рекомендуем обновить файлы конфигурации загрузчика, после чего удалить символические ссылки.

    Оригинал новости.
Разберемся, голубчик!

Кстати, будьте внимательны: обновился и grub … и неверно определил/прописал корень в menu.lst:
kernel /vmlinuz-linux root=/dev/sda3 ro
У меня root=/dev/sda2 - вовремя заподозревал (ранее он там был прописан как root=/dev/disk/by-uuid/xxxxx), проверил и исправил.

Как обновить и удалить символьные ссылки?
Как-как.., руками. 
# sed -i 's/vmlinuz26/vmlinuz-linux/' /boot/grub/menu.lst
# sed -i 's/kernel26.img/initramfs-linux.img/' /boot/grub/menu.lst
# sed -i 's/kernel26-fallback.img/initramfs-linux-fallback.img /boot/grub/menu.lst

Прямо в оригинальный конфиг прописал? Или имеется ввиду menu.lst.pacnew?
Да без разницы. Я прописал в новый, сделал бэкап старого, на всякий случай, затем переименовал:
# mv /boot/grub/menu.lst.pacnew /boot/grub/menu.lst
# (0) Arch Linux
title  Arch Linux 
root   (hd0,0)
kernel /vmlinuz-linux root=/dev/sda2 ro quiet vga=873
initrd /initramfs-linux.img
# (1) Arch Linux (Fallback)
title  Arch Linux Fallback
root   (hd0,0)
kernel /vmlinuz-linux root=/dev/sda2 ro 3 vga=873
initrd /initramfs-linux-fallback.img
# (2) Arch Linux (Fallback)
title  Arch Linux Singlemode
root   (hd0,0)
kernel /vmlinuz-linux root=/dev/sda2 ro 1 vga=873
initrd /initramfs-linux-fallback.img
Использование sed - конечно, на примере старого, в menu.lst.pacnew даже Fallback'а не было почему-то. Но без разницы, исправить-ли старый или добавить недостающее в новый.
