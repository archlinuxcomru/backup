эффект тот-же самый.
В журнале те же ошибки связанные с "Failed to set invocation ID for unit"
По логам видно, что система из suspend выходит с ошибкой.
А в части
Failed to set invocation ID for unit
это говорит о том, что некоторые службы не возобновляют свою работу после suspend - и таких служб у тебя несколько.
Проблема активно обсуждается на BBS и на GitHub (привел основные ссылки) уже в течение нескольких месяцев и наблюдается во многих системах Linux …. насколько я понял проблема наблюдается только на AMD и связана с функцией rdrand - в подробности не вдавался.
Рекомендую следить за развитем событий на BBS.
А не пробовал ядро linux-lts? - если нет, то установи и проверь. Кому то помог откат, кто то пробовал патч (правда помогло или нет - не отследил)

PS - но в логе кроме этого еще есть один момент - при загрузке наблюдается Call Trace, чем обусловлен, не понял
Ошибки не исчезают с опытом - они просто умнеют
