manukian, подскажите, пожалуйста, как вы с помощью iw подключаетесь к интернету?

поднимаю карту, в данно случаи у меня wlo1(определил при помощи iw dev)
ip link set wlo1 up
Сканирую четь:
iw dev wlo1 scan | less
ну и подключаюсь:
wpa_supplicant -B -i wlo1 -c /etc/wpa_supplicant/wpa_supplicant.conf

wpa_supplicant.conf генерирую при помощи wpa_passphrase ваш_ssid ваш_пароль > /etc/wpa_supplicant/wpa_supplicant.conf

А почему
manukian
подключаюсь нормально к интернету через утилиту iw
С помощью iw вы делаете одну ненужную операцию, поскольку имя сети и пароль уже известны и сканировать ни чего не нужно. iw не умеет подключаться к wpa сетям, только к wep. Можно подключиться и без iw:

>topsecret wpa_passphrase ваш_ssid ваш_password && wpa_supplicant -B -Dwext -iwlo1 -ctopsecret

"Генерировать" тоже ничего не требуется, на самом деле :)

Для подключения к вайфаю достаточно активировать два юнита, допустим [email protected] и [email protected] и создать примерно такой минимальный конфиг, соответственно, /etc/wpa_supplicant/wpa_supplicant-wlan0.conf
network={
        ssid="name"
        scan_ssid=1
        psk="password"
        key_mgmt=WPA-PSK
        group=CCMP TKIP
}

К тому же, для wpa_supplicant есть и свой гуи (для нелюбителей NM), wpa_supplicant_gui, имеется в официальном репозитории. Для него никаких параметров сетей в конфиг руками вообще писать не надо, достаточно двух строчек:
ctrl_interface=DIR=/run/wpa_supplicant GROUP=wpa
update_config=1
остальное будет дописывать сам, после выбора очередной сети и ввода пароля.

Что касается параметра -Dwext в командной строке (обычно в юните) wpa_supplicant, она требуется только для очень некоторых либо старых, либо черезчур огороженных карточек, не имеющих современного драйвера для линукса.
