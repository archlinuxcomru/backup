Настраивал сабжевую карточку. Успешно.

В арчвики сказано, что для корректной работы драйвера rt2860 необходимо вырубить следующие модули:
rt2800pci rt61pci rt2x00pci rt2800usb rt2800lib rt2x00usb rt2x00lib

Т.о. я прописал следующее в /etc/rc.conf в строчку MODULES:
MODULES=(!rt2800pci !rt61pci !rt2x00pci !rt2800usb !rt2800lib !rt2x00usb !rt2x00lib)

Это важно, ибо стандартные драйвера заглушают устройство сразу и не дают правильным драйверам работать с ним.

А в остальном все просто: скомпилировал RT3062PCI/mPCI/CB/PCIe(RT3060/RT3062/RT3562/RT3592) с официального сайта ралинка.
Перед компиляцией откорректировал файл os/linux/config.mk из скаченного архива:
HAS_WPA_SUPPLICANT=y
HAS_NATIVE_WPA_SUPPLICANT_SUPPORT=y

Затем собрал и установил:
make
sudo make install

Перезагрузился, заглянул в pci-устройства и обнаружил нужные драйверы в нужном месте:
$ lspci -v
05:00.0 Network controller: RaLink Device 3060
	Subsystem: D-Link System Inc Device 3c04
	Flags: bus master, slow devsel, latency 64, IRQ 16
	Memory at febf0000 (32-bit, non-prefetchable) [size=64K]
	Capabilities: <access denied>
	Kernel driver in use: rt2860
	Kernel modules: rt3562sta, rt2800pci


Кстати, драйвер rt2860 с AURа у меня тоже не заработал. На досуге нужно будет собрать рабочий пакет..
