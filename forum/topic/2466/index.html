"Правильное" выключение KDE через терминал

На случай, если кому-нибудь пригодится, хотя вопрос корректного выключения KDE через консоль, как показывает гугл и поиск по нашему форуму, освещался не раз. Ну, может пару раз;)

Идея: быстрый доступ к командам выключения и перезагрузки компьютера из “Меню запуска приложений”, вызываемого средним кликом (при соответствующих настройках поведения мыши в данной комнате/виджете)

Исполнение: kmenuedit > Новое подменю > “Выход” >
новый пункт > “перезагрузить” > команда > qdbus org.kde.ksmserver /KSMServer org.kde.KSMServerInterface.logout 0 1 0
новый пункт > “выключить” > команда > qdbus org.kde.ksmserver /KSMServer org.kde.KSMServerInterface.logout 0 2 0

Значение имеют три цифры (опции) в конце команды.
В первом случае без подтверждения (0) выполняются перезагрузка (1), на данный момент завершены все активные сессии (0) (если я правильно перевёл про сессии ;).
Во втором: выключение на тех-же условиях.

Подробнее об опциях на англ. применительно к dcop, но актуально и для D-Bus.
И ещё, почитать: Control KDE applications from the commandline with D-Bus
