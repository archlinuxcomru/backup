Отваливается сетевой интерфейс после перезагрузки системы

Установлил Арч. Де xfce4. Системаза пустилась и всё вроде бы прекрасно, если бы не одно НО.
Нет сети!
Настраиваю сеть посредством netctl так:
p /etc/netctl/examples/ethernet-dhcp /etc/netctl/my-network
Отредактировал my-network
Активировал демона systemctl enable netctl.service и запустил службу systemctl start netctl.service
Не работает вообще. Этот netctl вообще какой-то убитый. Проверяю статус до перезагрузки и после, и вижу:
[[email protected] hoz]# systemctl status netctl.service ● netctl.service - (Re)store the netctl profile state
   Loaded: loaded (/usr/lib/systemd/system/netctl.service; enabled; vendor prese
   Active: active (exited) since Wed 2016-09-07 19:19:43 MSK; 29min ago
  Process: 248 ExecStart=/usr/bin/netctl restore (code=exited, status=1/FAILURE)
 Main PID: 248 (code=exited, status=1/FAILURE)
    Tasks: 0 (limit: 4915)
   CGroup: /system.slice/netctl.service

Sep 07 19:19:43 hozpc systemd[1]: Starting (Re)store the netctl profile state...
Sep 07 19:19:43 hozpc systemd[1]: Started (Re)store the netctl profile state.
lines 1-10/10 (END)
Раз уже netctl не работает вообще пришлось искать другие варианты решения данной проблемы.
Вышло настроить сеть вот так:
ip addr add 192.168.1.2/24 broadcast 192.168.1.255 dev enp2s0
ip route add default via 192.168.1.1
Есс-но прописал DNS в файле /etc/reslov.conf
Всё работает до перезагрузки. Перегружаюсь и сетевой интерфейс отваливается.
Таким образом, приходится выполнять эти 2 команды после каждой перезагрузки системы! В чём подвох?
