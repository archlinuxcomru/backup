общем набросал простой скрипт и повесил на хоткей, вроде пашет
недостаток, не читает большие объёмы, нужно разбивать на меньшие части, но т.к. это нафик нужно то и не нужно)
для остановки чтения нужно просто выделить мало текста и проиграть его
#!/bin/sh
killall aplay
notify-send -i audio-volume-medium "$(xsel -o|head -n5)"
xsel -o|sed 's/  *$//'|RHVoice-client -s anna -r -0.2 -p -0.1 |aplay -q

если редко используется(как у меня) то такая версия, дабы RHVoice-service не висел постоянно
#!/bin/sh
killall aplay
killall RHVoice-service
RHVoice-service&
notify-send -i audio-volume-medium "$(xsel -o|head -n5)"
sleep 0.2
xsel -o|sed 's/  *$//'|RHVoice-client -s anna -r -0.2 -p -0.1 |aplay -q
killall RHVoice-service

а есть чего хорошего для того чтоб прикрутить его к firefox например?
В Firefox 49 в режиме "Вид для чтения" появилась возможность воспроизводить текст голосовым движком через Speech-dispatcher. Для этого последний должен быть настроен на работу с RHVoice.
Я свой скрипт сделал еще когда этой функции не было. Он рассчитан на резидентно работающий RHVoice-service. Работает со скриптом на Python'е, предварительно обрабатывающим текст (правильное чтение дат, чисел и т.п.). Взять можно тут: bash-скрипт и python-скрипт. Этот же скрипт в несколько видоизмененном виде имеется на GitHub в проекте RHVoice-dictionary (maintainer vantu5z) - там самая последняя версия.
