Здравствуйте, уважаемые знатоки Arch Linux. На днях купил новый ноутбук, Lenovo Ideapad 3 с процессором AMD Athlon Silver 3050U со встроенной графикой AMD Radeon Vega 3. До этого никогда с AMD не сталкивался, всегда пользовался Intel. Проблема появляется на этапе загрузки установочного носителя (флешка), при загрузке не работает подсветка дисплея, видно процесс но все в темноте. Поэтому плохо видно что он выводит на экран. Возможно ли что то прописать при загрузки что бы нормально установить Arch Linux.

Вряд ли это софтовая проблема. Сбросьте BIOS, обновите, если есть обновление. Посмотрите, какие там есть настройки видеокарты. Попробуйте какой-нибудь другой LIVE-образ загрузить, например, ALT Linux. Он максимально вылизанный и отлаженный.

Сам не использую AMD, но если не ошибаюсь, то VEGA использует amdgpu, а там иногда бывает проблема с Display Core (DC) и рекомендую его отключить
When CONFIG_DRM_AMD_DC is enabled, DC will be initialized by default for supported ASICs. To force disable, set amdgpu.dc=0 on kernel command line. Likewise, to force enable on unsupported ASICs, set amdgpu.dc=1.
точнее, попробуй для начала загрузиться с параметром amdgpu.dc=0
Ошибки не исчезают с опытом - они просто умнеют
