Давно терепел внутреннее недовольство по этому поводу. С тех пор, как отвалился милый моему сердцу Touchegg. Что отмечу - довольно давно отмечено, что штатная гномова регулировалка скорости тачпада работает в сеансе вайланда и не работает в сеансе иксовом, разница в скорости раза в два. Ну, думал, после метания в Сусю и обратно будет чистая установка, с нуля буду ставить все по минимуму. До установки xf86-input-synaptics тачпад работал, но нажатия двух-трех пальцев не переопределялись, т.е. подсовывание конфига Хорг не срабатывало. Поставил, прописал - теперь православные два пальца как средняя кнопка. Теперь доставил xorg-xinput, но вот в выводе не вижу ни слова про скорость -
xinput list-props 9
Device 'Apple Inc. Apple Internal Keyboard / Trackpad':
	Device Enabled (139):	1
	Coordinate Transformation Matrix (141):	1.000000, 0.000000, 0.000000, 0.000000, 1.000000, 0.000000, 0.000000, 0.000000, 1.000000
	libinput Send Events Modes Available (259):	1, 0
	libinput Send Events Mode Enabled (260):	0, 0
	libinput Send Events Mode Enabled Default (261):	0, 0
	Device Node (262):	"/dev/input/event6"
	Device Product ID (263):	1452, 657
Предполагаю, что это один из параметров в строке Coordinate Transformation Matrix. Никто не сталкивался?

ответ был прост -
1. командуем synclient и читаем примененные параметры
2. командой synclient параметр=значение подбираем интересные
3. подобранные параметры записываем в конфиг.

И ведь помню же, что раньшеэто делал, да забыл за давностью.
