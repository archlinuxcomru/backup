Могу предложить следующий вариант
Поддержка файловой системой файлов более 2Гб (large_file) у нас имеется
$ sudo tune2fs -l /dev/sda3 | grep large
Filesystem features: …..... large_file
Согласно man tune2fs
-O [^]feature[,...]
Set or clear the indicated filesystem features (options) in the filesystem. More than one filesystem feature can be cleared or set by separating features with commas. Filesystem features prefixed with a caret character ('^') will be cleared in the filesystem's superblock; filesystem features without a prefix character or prefixed with a plus character ('+') will be added to the filesystem. For a detailed description of the file system features, please see the man page ext4(5).
И можно включить это свойство (поддержку файлов размером более 2Г)
$ sudo tune2fs -O +large_file /dev/sda....
или отключить это свойство - $ sudo tune2fs -O ^large_file /dev/sda... # можно временно, а потом убрать
Но никогда этого не делал. Есть нюансы по применению, встречался совет - после этой процедуры следует запустить команду e2fsck -f, а еще лучше все эти свойства задавать программой mke2fs -O при создании файловой системы.
И еще один нюанс — можно ли это делать на смонтированной файловой системе или лучше на отмонтированной??? - лучше поискать.....PS...на мой взгляд можно это провернуть и из действующей системы..
Пробуй.............на свой страх и риск.............потом раскажешь.....
Ошибки не исчезают с опытом - они просто умнеют
