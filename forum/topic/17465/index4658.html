Просьба, при наличии времени, вкратце описать, что делал.
Ну у меня сложнее, поскольку ноут обновляется через флешку (там где он находится нет интернета), но общие рекомендации пока вижу следующими:

В конкретном случае я не использовал mirrorlist, а зеркало прописал в конфиг пакмана:
[core]
SigLevel = PackageRequired
Server = https://archlinux32.mirror.roelf.org/i686/$repo

[extra]
SigLevel = PackageRequired
Server = https://archlinux32.mirror.roelf.org/i686/$repo

[community]
SigLevel = PackageRequired
Server = https://archlinux32.mirror.roelf.org/i686/$repo
Кому нужно как обычно - зеркала тут.
Далее создаем и устанавливаем archlinux32-keyring:
$ git clone https://github.com/archlinux32/packages
$ cd packages/core/archlinux32-keyring
$ makepkg -i
По-хорошему нужно (уже не нужно) пересобрать/переустановить pacman, добавив этот пакет в его зависимость:
depends+=(archlinux32-keyring)
Генерируем ключи:
# pacman-key --init
# pacman-key --populate archlinux32
# pacman-key --refresh-keys
Собственно обновляемся:
# pacman -Syyu
Кажись все.
-----
EDIT.
archlinux32-keyring сейчас есть в core (32), так что в сборке не нуждается. Можно просто скачать-установить, а можно как рекомендуют на странице загрузки:
pacman -Syyw archlinux32-keyring
pacman -U /var/cache/pacman/pkg/archlinux32-keyring-*.pkg.tar.xz
Кстати, подчеркивают понижение версий при обновлении:
pacman -Suu
