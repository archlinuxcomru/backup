Смотрю отметился на разных форумах, но … советов нет. Разумнее анализировать самому …
В части
fancat
сообщений от ядра при подключении наушников нет.
В логах journal (включая dmesg) сообщений о подключении наушников и не будет.
Это нужно смотреть по другому, способов несколько, например, используя pacmd смотрим active port и плюс уровень звука
Без наушников
pacmd list-sinks | egrep 'active|volume'
	volume: front-left: 32766 /  50% / -18.06 dB,   front-right: 32766 /  50% / -18.06 dB
	base volume: 65536 / 100% / 0.00 dB
	volume steps: 65537
	active port: <analog-output-speaker>
С наушниками
pacmd list-sinks | egrep 'active|volume'
	volume: front-left: 58409 /  89% / -3.00 dB,   front-right: 58409 /  89% / -3.00 dB
	base volume: 65536 / 100% / 0.00 dB
	volume steps: 65537
	active port: <analog-output-headphones>

Но интреснее другое, из вывода видим, что данная звуковуха имеет alternative analog output (Alt Analog)
fancat
card 0: PCH [HDA Intel PCH], device 2: VT2020 Alt Analog [VT2020 Alt Analog]
а насколько мне известно, нормальной поддержки автоопределения Alt Analog нет и требуется дополнительная настройка - как и что делать описано, ищи.

EDIT 1 - как пишут, при правильной настройке в выводе aplay -L должно быть типа такого:
alt:CARD=PCH,DEV=0
HDA Intel PCH, ALC1220 Alt Analog
Front headphones
... но могу и ошибаться, это чисто теоретические познания - практически же с данными дивайсами дела не имел.
Ошибки не исчезают с опытом - они просто умнеют
