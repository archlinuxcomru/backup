Так как помощи от "демагогов" не дождаться (жду ещё jim945, так как он обещал воскресенье-понедельник посмотреть), нашел сам частичное решение, посетив несколько десятков зарубежных форумов. Итак,
9. Если возникла ошибка после выполнения п.8
rTorrent должен иметь доступ на чтение/выполнение к директории tmp. ruTorrent не будет работать. (/tmp/)
Нужно создать папку tmp в папке rutorrent по адресу, куда и был скачан-установлен rutorrent - /usr/share/webapps/rutorrent/
sudo mkdir /usr/share/webapps/rutorrent/tmp
Затем, даём этой папке права нашего веб-сервера (http):
sudo chown -cR http:http /usr/share/webapps/rutorrent/tmp
После этого изменить путь к временной папке (tmp) в файле конфигурации rutorrent по адресу /srv/http/rutorrent/conf/config.php
Открыть в редакторе
sudo vim /srv/http/rutorrent/conf/config.php
найти строку (у меня она последняя)
$tempDirectory = null;
заменить её на
$tempDirectory = '/usr/share/webapps/rutorrent/tmp';

Итак. Для запуска вебморды rutorrent использовалось 2 статьи.
nginx и rutorrent.
Файл /etc/php/php.ini
Добавляем в open_basedir параметры указанные в обеих статьях.
Раскоментируем необходимые extension. Фиг знает какие конкретно нужны в данном случае. У меня раскомменчена добрая половина)))

Файл /etc/php/php-fpm.conf
Пишем следующую строку.
security.limit_extensions = .php .html .htm
Как и указано в статье nginx.

/etc/nginx/nginx.conf У меня получился такой кусок.
    server {
 listen       80;
 server_name  localhost;

  root   /usr/share/webapps/rutorrent;

location ~ \.(php|html|htm)$ {
 fastcgi_pass   unix:/run/php-fpm/php-fpm.sock;
 fastcgi_index  index.php;
 include        fastcgi.conf;
}

location / {
  index  index.html index.htm index.php;
}

 location /RPC2 {
  include scgi_params;
  scgi_pass localhost:5000;
}
}
Все запчасти из Вики)))
# systemctl start php-fpm nginx
В браузере localhost
