Доброго времени! Потребовалось накатить очередной арчик: скачал образ, скопировал на мультибут флешку, загрузился. Создал устройство LUKS, отдав ему весь диск (без таблицы разделов -- полнодисковое), открыл, получил новое устройство в /dev/mapper, форматировал его в btrfs. И дальше начались чудеса:
Если сказать
mount /dev/mapper/crypt /mnt
Оно замечательно монтируется. Внутри создаётся subvolume (нужно будет делать снепшоты). Отмонтируется тоже дивно.
mount /dev/mapper/crypt -o realtime,ssd,space_cache,compress=lzo,subvol=@root /mnt
Плюёт в консоль стандартное сообщение "open_ctree failed", после чего также видно стандартную отговорку mount: "wrong fs type... or other error."

Если снова смонтировать, не указывая никаких опций -- всё чудно.
Если смонтировать указав те же самые опции, которые видно в выводе команды "mount" -- опять ругается.
Я не понимаю, что делать... вот это видел. Но проблема в том, что здесь ФС только что была отформатирована и в принципе-то монтируется...

arch iso грузится вот так:
## Загрузка ArchLinux
dir=/.boot/images
arch=x86_64
insmod regexp               # для шаблонов * в именах файлов
probe -s root_uuid -u $root # получаем UUID для подстановки в img_dev=

for iso in $dir/archlinux-*-$arch.iso ; do      # ищем образ(ы) Arch по шаблону
 if [ ! -f "$iso" ] ; then continue; fi        # только если образ существует
 regexp -s build 'archlinux-(.+)-'$arch "$iso"  # получаем из имени образа дату сборки

  menuentry "Arch Linux ISO $build $arch" --source="\
loopback loop $iso
probe -s isolabel -l loop
linux (loop)/arch/boot/$arch/vmlinuz archisolabel=\$isolabel img_dev=/dev/disk/by-uuid/$root_uuid img_loop=$iso earlymodules=loop
initrd (loop)/arch/boot/$arch/archiso.img"

done

P.S. Буквально пару месяцев назад на аналогичной конфигурации таких проблем не было.
Сотрудник техподдержки признался: он хорошо зарабатвыает просто потому, что умеет гуглить быстрее клиентов.
