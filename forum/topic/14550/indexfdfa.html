Не помню, спрашивали ли уже, листать лень, поэтому ещё вопрос: что выдаёт 
cat /etc/environment
Ладно.., вне зависимости от, можно попробовать в терминале "поломанного" юзера выполнить 
~$ export LANG=ru_RU.UTF-8
с последующей проверкой на приложениях уже без бубна, просто запуская их обычным способом - это разовая инъекция. Можно и в ~/.bashrc это воткнуть с последующим ~$ source ~/.bashrc - и проверить.
Если сработает - смотреть дальше. По идее, для русской локали и содержание /etc/environment должно быть соответствующим:
# This file is parsed by pam_env module
#
# Syntax: simple "KEY=VAL" pairs on separate lines
#
LANG=ru_RU.utf8
Если соответствует, то тем более - причина в конфигах конкретного пользователя, а ещё горячее - в настройках Xfce данного пользователя.
