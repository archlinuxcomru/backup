Доброго времени суток!

При обновлении/установке/удалении вот такая ошибка выпадает:
ошибка: не удалось подготовить запрос
:: rp-pppoe: требует ppp=2.4.6
Например:
$ sudo pacman -Syu
[sudo] password for :
:: Синхронизируются базы данных пакетов...
 core не устарел
 extra не устарел
 community не устарел
:: Запускается полное обновление системы...
разрешение зависимостей...
проверка на взаимную несовместимость...
ошибка: не удалось подготовить запрос (не удалось удовлетворить зависимости)
:: rp-pppoe: требует ppp=2.4.6
Причём:
$ sudo pacman -Ss ppp
core/perl 5.20.0-5 (base) [установлен]
    A highly capable, feature-rich programming language
core/ppp 2.4.7-1 [установлен: 2.4.6-2]
    A daemon which implements the Point-to-Point Protocol for dial-up networking
...
extra/rp-pppoe 3.11-5 [установлен]
    Roaring Penguin's Point-to-Point Protocol over Ethernet client
...
$

Всё ещё хуже.
Сегодня утром я тоже напоролся на это, и выяснил, что нужный пакет есть, висит (утром висел) в тестинге.
И вот почему:

Maintainers: Orphan
Короче, у пакета rp-pppoe не осталось мейнтейнеров в Арч.

О причинах несложно догадаться:
1) Поддержка PPPoE включена в ядре Арч
2) Команда pppoe-discovery включена в пакет ppp
3) Плагин rp-pppoe.so для pppd, использующий "ядерную" поддержку PPPoE, тоже входит в ppp, только там он чуть более старой версии, чем в rp-pppoe.
В результате, пакет rp-pppoe в Арче не особо нужен, и никто не хочет его поддерживать.
