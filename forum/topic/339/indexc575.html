Ну для меня это не религиозный вопрос, а практический.

Я уже выше писал.
для конфигов (фстаб, конфиг загрузчика и т.п.) только UUID
для консоли (ручной ввод в командах) LABEL или /dev/sdXX с предварительным blkid.

Поэтому если уж создавать голосовалку, то нужно разделять для чего именно использовать UUID и LABEL.

by-id опять забыли.
Почему забыли? Я никогда не забывал. Вот, пожалуйста, это с моей домашней машины :)
Обратите внимание на /dev/wd* и /dev/boot* – это симлинки, которые я организовал по этому самому ID, то бишь серийникам винчестера и загрузочной флешки.
$ cat /proc/cmdline 
BOOT_IMAGE=/vmlinuz-linux init=/bin/systemd root=/dev/wd1 loglevel=6 resume=/dev/wd2
$ head -n9 /etc/fstab
# 
# /etc/fstab: static file system information
#
# <file system>        <dir>         <type>    <options>          <dump> <pass>
/dev/wd1               /             ext4      defaults            0      1
/dev/boot1             /boot         ext2      defaults            0      1
/dev/wd2               swap          swap      defaults            0      0
tmpfs                  /tmp          tmpfs     nodev,nosuid        0      0
/dev/wd6               /home         ext4      defaults            0      1
$ cat /etc/udev/rules.d/61-nickname-disk.rules 
KERNEL!="sd*", GOTO="nick_end"
ACTION=="add", GOTO="check_id"
ACTION=="change", GOTO="check_id"
GOTO="nick_end"
LABEL="check_id"
ENV{ID_SERIAL}=="WDC_WD3000JB-00KFA0_WD-WCAMR3013826", ENV{NICK}="wd", GOTO="set_name"
ENV{ID_SERIAL_SHORT}=="0780C310F02BB9E6", ENV{NICK}="boot", GOTO="set_name"
GOTO="nick_end"
LABEL="set_name"
ENV{DEVTYPE}=="partition", SYMLINK+="$env{NICK}%n", GOTO="nick_end"
ENV{DEVTYPE}=="disk", SYMLINK+="$env{NICK}"
LABEL="nick_end"
