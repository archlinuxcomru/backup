Привожу цитату из одной старой статейки (подкоректированную на нормальный язык)
Несмотря на кажущуюся простоту доступа к общей папке Windows из Linux некоторые пользователи Linux, могут столкнуться с проблемой, которая, на первый взгляд, может оказаться неразрешимой. При этом пользователям Windows папка общего доступа открывается без проблем, а при обращении пользователям Linux к этой общей папке  достучаться невозможно. Проблема в большей части случаев обусловлена "системным непониманием" между Linux и Windows
Это я к тому, что поддержу indeviral - сначала хорошо бы проверить все на стороне Windows

Cooler2 попробуйте еще глянуть что там за сервер

смотрим айпишники и состояние портов( вместо 192.168.1.0/24 ваша сеть) сканирование может некоторое время занять
sudo nmap 192.168.1.0/24 -p 139,445,137 -oG '/tmp/sambaskan'
смотрим инфу по найденному самба серверу с открытым портом какие протоколы самба доступны, если первого нет то и обзор шары вряд ли будет. По версии протокола можно и выставить для самбаклиента версию
sudo nmap -p 445 --script smb-protocols  192.168.1.104 -oG '/tmp/sambainfo'
или инфу о шарах если сервер позволяет
sudo nmap --script smb-enum-shares  192.168.1.104 -oG '/tmp/sambafolder'
смотрите по скриптам https://nmap.org/nsedoc/
можно и 
enum4linux -a -u barmaley 192.168.1.104
или sparta поставить

А для монтирования можете smb4k использовать он самбаклиент тоже использует но в отличие от gvfs нормально монтирует
sudo pacman -S smb4k

smb4k - просто зависает при подключении надолго, и ничего не происходит
Определить сначала по которым протоколам работает сервер с шарой
sudo nmap -p 445 --script smb-protocols 10.10.2.112
потом проверить что самбаклиент подключается с такой версией нормально
smbclient //192.168.1.104/Screenshots -m SMB3
Enter WORKGROUP\oleg's password:
Try "help" to get a list of possible commands.
smb: \> ls
перечень файлов должен выводиться
smb: \>exit
вместо SMB3 поддерживаемый на сервере протокол
NT1
SMB2
SMB3
SMB2_02
SMB3_02
SMB3_11

Добились нормального подключения клиента - пропишите в параметры версию протокола.
/etc/samba/smb.conf
[global]
workgroup = WORKGROUP
client min protocol = CORE
#client min protocol = SMB3_11
#client min protocol = SMB2
client max protocol = SMB3_11
#client max protocol = SMB2

Перезагрузите smb4k и попробуйте снова. Открыть диалог подключения (ctrl+O) →расположение указываете //192.168.1.104/Screenshots для сервера без анонимного(гостевой) доступа в имя юзера что то вставить а пароль пустой и ок
Некотрые сервера не позволяют анонимный доступ а настроены на гостевой то есть в какое то имя надо передать, ссылку vasek давал сверху
