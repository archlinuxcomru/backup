smbclient -L 10.10.2.112 -N - срабатывает нормально, показывая открытые ресурсы.
smbtree -b - N выдает ошибки
smbXcli_negprot_smb1_done: No compatible protocol selected by server.

https://ibb.co/zN2PVvV

Если в проводниках вбить smb://10.10.2.112/Общий обмен
то выходит запрос
https://ibb.co/p1g9Xfs

Стоит файрвол ufw с правилами:
https://ibb.co/yS5mzf3
Да и с файрволом и без него результат тот же - не пускает.

Конфиг Samba:
[global]
	dns proxy = No
	dos charset = cp866
	guest account = guest
	log file = /usr/local/samba/var/log.%m
	map to guest = Bad Password
	max log size = 50
	security = USER
	server role = standalone server
	server string = Samba server
	unix charset = UTF8
	unix extensions = No
	usershare allow guests = Yes
	idmap config * : backend = tdb
	guest ok = Yes
	wide links = Yes
[homes]
	browseable = No
	comment = Home Directories
	read only = No
[printers]
	browseable = No
	comment = All Printers
	guest ok = No
	path = /usr/spool/samba
	printable = Yes
[public]
	guest ok = No
	path = /srv/public
	read only = No
	valid users = @rvd
[public_share]
	comment = Anonymous File Server Share
	locking = No
	path = /home/rvd/share
	read only = No

Пользователь rvd и guest состоят в группах smbgrp.

Запущены и работают сервисы: smb и nmb. Avahi-daemon.
