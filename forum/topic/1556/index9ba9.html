Мой возьми, для примера:
. $HOME/.bashrc
[[ -z $DISPLAY && $(tty) = /dev/tty11 ]] && exec startx &> /dev/null < /dev/null
он работает вкупе с такими параметрами /etc/inittab:
# Boot to X11
id:5:initdefault:
[...]
# Example lines for starting a login manager
x:5:once:/sbin/agetty -a USER_NAME -8 -s 38400 tty11 linux
как понимаешь, в этой конфигурации не используется KDM и запуск происходит через ~/.xinitrc:
#!/bin/sh
#
# ~/.xinitrc
#
# Executed by startx (run your window manager from here)
#export BROWSER=firefox
#export EDITOR=nano
#D-bus
if which dbus-launch >/dev/null && test -z "$DBUS_SESSION_BUS_ADDRESS";
then
       eval `dbus-launch --sh-syntax --exit-with-session`
fi
exec ck-launch-session startkde
Но я уже давно пользуюсь входом через KDM и мой ~/.xinitrc мог “устареть”, тем не менее, можно проверить твои незапускаемые приложения, используя вышеприведённые конфиги и запуск через startx
А там видно будет, что дальше делать, если будет, конечно.
