что делать с пайтон скриптом
Перл скрипт все же а не питоновский
На всякий случай отключить автостарт дисплейного менеджера
например 
sudo systemctl disable lxdm
пропатчить библиотеки (для 495.46)
perl -pwE 's/\x0f\x84\x5b\x01\x00\x00\x4c\x8d/\xe9\x5c\x01\x00\x00\x00\x4c\x8d/g' /usr/lib/libnvidia-glcore.so.495.46 > /tmp/libnvidia-glcore.so.495.46.new
perl -pwE 's/\x0f\x84\x5b\x01\x00\x00\x4c\x8d/\xe9\x5c\x01\x00\x00\x00\x4c\x8d/g' /usr/lib32/libnvidia-glcore.so.495.46 > /tmp/lib32-libnvidia-glcore.so.495.46.new
sudo mv /tmp/libnvidia-glcore.so.495.46.new /usr/lib/libnvidia-glcore.so.495.46
sudo mv /tmp/lib32-libnvidia-glcore.so.495.46.new /usr/lib32/libnvidia-glcore.so.495.46
перезагрузиться залогиниться
и запустить дисплейный менеджер
например
sudo systemctl start lxdm
если все нормально проверить на спам и включить автостарт DM
#тест опенгл
glxheads
glthreads32
#тест на спам
sudo dbus-monitor --system
#включение DM
sudo systemctl enable lxdm

если не зароботали опенгл программы то откатить изменения переставив nvidia-utils
sudo pacman -Syu nvidia-utils lib32-nvidia-utils

У меня сработало спама нет

Все же рекомендую сменить dbus.service
https://archlinux.org.ru/forum/post/252226/
