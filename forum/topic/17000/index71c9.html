У меня тоже не получалось запустить спектр, потом на гитхабе в инструкциях нашел это:

As some configuration in Archlinux has changed recently edit your local .spectrwm.conf file to set bar_font similar like this:

bar_font = xos4 Terminus:pixelsize=14:antialias=true

(Here assuming that font Terminus is installed. Other fonts that are listed with fc-list should also work)

For more details see also issue #193

после редактирования конфига все заработало.

Пилить в принципе очень мало, а, главное все просто и понятно.
В основном все уже стоит по дефолту. Если исключить цвет/толщину бордюра и другие мелочи, которые на любителя, то основные изменения в основном следующие
 bar_font               = xft:Terminus (TTF):size=14:weight=medium:antialias=true:hinting=true
 bar_action           = ~/.local/bin/baraction_dzen.sh
 bar_delay            = 3
 bar_format	    = +I +S +5<+A
плюс автозапуск
# Launch applications in a workspace of choice
 autorun		= ws[1]:tilix
 autorun		= ws[2]:pcmanfm
плюс hotkey для запуска программ и управления, типа
program[tilix]	        = tilix
bind[tilix]              	= MOD+Shift+T
program[pcmanfm]	= pcmanfm
bind[pcmanfm] 	     = MOD+Shift+P
program[xbacklight1]    = xbacklight -inc 5
bind[xbacklight1]       = MOD+Y
program[xbacklight2]    = xbacklight -dec 5
bind[xbacklight2]       = MOD+Shift+Y
И самое сложное это status bar (baraction_dzen.sh) - что выводить и как выводить
bar_action           = ~/.local/bin/baraction_dzen.sh

EDIT 1 - В части указания шрифтов - у меня работает любая из этих комбинаций
bar_font    = xft:Terminus (TTF):size=14:weight=medium:antialias=true:hinting=true
bar_font    = terminus:bold:size=16
bar_font    = terminusRe33:size=12:antialise=true:hinting=true
bar_font    = "xos4 Terminus-14"
в части использования шрифтов все прописано в man (там же прописано, что есть нюансы со шрифтами в dmenu).
Ошибки не исчезают с опытом - они просто умнеют
