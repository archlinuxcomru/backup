Вообще-то, там речь о /etc/X11/xinit/.Xresources.
Еще раз: скрипт /etc/X11/xinit/xinitrc, в котором обрабатывается команда
xrdb -merge /etc/X11/xinit/.Xresources
будет вызван только в случае отсутствия ~/.xinitrc.
Если .xinitrc присутствует в домашнем каталоге пользователя, startx и xinit выполнят его. Иначе startx выполнит по умолчанию /etc/X11/xinit/xinitrc.
... Поэтому, чтобы запустить другой оконный менеджер или окружение рабочего стола, сначала создайте копию по умолчанию xinitrc в вашем домашнем каталоге:
$ cp /etc/X11/xinit/xinitrc ~/.xinitrc
Wiki
Что не понятно? Может быть один /etc/X11/xinit/.Xresources для всех пользователей, но использовать его содержимое каждый пользователь должен индивидуально в своем ~/.xinitrc. В противном случае должен быть один общий /etc/X11/xinit/xinitrc для всех пользователей. Это кому-то надо?
