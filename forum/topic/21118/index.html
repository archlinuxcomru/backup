Здравствуйте! Проблема следующая - хочу использовать amdgpu, тк подозреваю radeon в генерации артефактов на видео после выхода из гибернации
Делал по инструкции с английской вики.
Симптомы - система использует radeon несмотря на мои попытки заставить ее использовать amdgpu. Если внаглую убрать radeon из mkinitcpio и в modprobe.d его заблэклистить, иксы вообще не стартуют, пока руками не сделаешь modprobe radeon
Что сделано:
/etc/modprobe.d/amdgpu.conf

options amdgpu si_support=1
options amdgpu cik_support=1
/etc/modprobe.d/radeon.conf

options radeon si_support=0
options radeon cik_support=0
/etc/mkinitcpio.conf

MODULES="amdgpu radeon"
zcat /proc/config.gz | grep AMDGPU

CONFIG_DRM_AMDGPU=m
CONFIG_DRM_AMDGPU_SI=y
CONFIG_DRM_AMDGPU_CIK=y
CONFIG_DRM_AMDGPU_USERPTR=y
dmesg | grep amdgpu

[   17.608782] [drm] amdgpu kernel modesetting enabled.
[   17.608876] amdgpu: CRAT table not found
[   17.608879] amdgpu: Virtual CRAT table created for CPU
[   17.608888] amdgpu: Topology: Add CPU node
lspci -k

03:00.0 VGA compatible controller: Advanced Micro Devices, Inc. [AMD/ATI] Turks PRO [Radeon HD 6570/7570/8550 / R5 230]
        Subsystem: Micro-Star International Co., Ltd. [MSI] Device 2501
        Kernel driver in use: radeon
        Kernel modules: radeon, amdgpu
Установлен amdgpu-pro-libgl из AUR
Если принудительно заставить иксы использовать amdgpu, ругается на то, что версия DRM 2.50.0, но драйверу нужна 3.x.x
