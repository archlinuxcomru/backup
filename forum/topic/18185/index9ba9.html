и на более сложное с Grub2
Там по началу сложно только grub.cfg править. Можно скрипт автоопределения использовать для простоты.Определяет распространенные линдистры
https://github.com/Jimmy-Z/grub-iso-boot

Тогда в простейшем случае создание мультибут флешки для bios:
В gparted форматируем флешку в fat32 и ставим флаг boot

Потом пару команд
для примера флешка sdc
sudo umount /dev/sdc1
sudo mount /dev/sdc1 /mnt
sudo mkdir /mnt/boot
sudo mkdir /mnt/boot/iso
sudo grub-install --target=i386-pc --recheck --boot-directory=/mnt/boot /dev/sdc
cd /mnt/boot/grub
sudo wget https://github.com/Jimmy-Z/grub-iso-boot/raw/master/grub.cfg
cd
sudo umount /dev/sdc1

Затем копируем исошки дистров в папку /boot/iso на флешке с помощью вашего файлменеджера
ждем пока скопируется и можно проверять

Проверить сам губ на флешке можно
qemu-system-x86_64  /dev/sdc
qemu должен быть установлен

multibootusb вместе с syslinux устанавливает загрузчик Grub2.
Можно воспользоваться им (выбираем в загрузочном меню Grub2)

1. Создаешь в корне папку ISO.
2. Кидаешь туда ubuntu-16.04.3-desktop-amd64.iso (или другой).
3. Редактируешь multibootusb\grub\grub.cfg:
menuentry "ubuntu-16.04.3-desktop-amd64" {
          set isofile="/ISO/ubuntu-16.04.3-desktop-amd64.iso"
          loopback loop $isofile
	  linux	(loop)/casper/vmlinuz.efi boot=casper iso-scan/filename=$isofile noprompt noeject
	  initrd (loop)/casper/initrd.lz
}
У меня работает...
