Есть раздел на ЖД непримонтированный в fstab (мне так надо).По щелчку на этот раздел в файловом менеджере требует пароль,что уже немного напрягает.
В линукс минт,лубунтуэто решалось установкой пакета policykit-desktop-privileges ,в debian где нет такого пакета в папку /var/lib/polkit-1/localauthority/10-vendor.d добавлял файл такого содержания 
[Mounting, checking, etc. of internal drives]
Identity=unix-group:admin;unix-group:sudo
Action=org.freedesktop.udisks.filesystem-*;org.freedesktop.udisks.drive-ata-smart*;org.freedesktop.udisks2.filesystem-mount-system;org.freedesktop.udisks2.encrypted-unlock-system;org.freedesktop.udisks2.filesystem-fstab;
ResultActive=yes
..........
..........
и тоже всё монтируется без ввода пароля.
В Arch пробовал устанавливать из AUR пакет policykit-desktop-privileges-ubuntu 0.12-1 ,устанавливается но ничего не даёт,пароль всё равно требует.
Может кто нибуть подскажет как в ARCH сделать монтирование разделов без ввода пароля?
Система Openbox,fm PCmanfm ,флешки монтируются отлично без ввода пароля.

sirocco,спасибо огромное!
Создал по вики файл 10-enable-mount.rules 
polkit.addRule(function(action) {
    if (action.id == "org.freedesktop.udisks2.filesystem-mount-system") {
        return polkit.Result.YES;
    }
});
и всё монтируется как часики и никакого пароля не требует!
Меня вообще удивляет как быстро опытные пользователи Arch умеют находить нужные маны.

Vadim, вы только не забудьте, что это правило позволяет монтировать диск любому пользователю в системе. В некоторых случаях это может быть нежелательно. Пример с разрешением для группы есть в той же wiki, а для отдельного юзера вот:
polkit.addRule(function(action, subject) {
        if (action.id == 'org.freedesktop.udisks2.filesystem-mount-system' && subject.user == '%username%') {
            return polkit.Result.YES;
        }
    }
);
