И где же из этого следует, что вы пытаетесь оперировать с файлами "под рутом"? Вы сидите в системе обыкновенным пользователем "vova" без привилегий.

Стать "рутом" можно тремя способами:
1. Изначально залогинившись с именем root
2. С помощью команды "su -"
3. С помощью "sudo -i"
В любом из этих случаев будет
# whoami
root
Обратите внимание на значек приглашения и имя пользователя.

Вова, команда 
sddm --example-config > /etc/sddm.conf
выполняется от имени обычного пользователя. Команда sudo [команда] — одноразовая, т.е. действует только на текущую команду, следующая будет выполняться от имени пользователя. Читайте внимательно выше,
kurych посоветовал вам команду sudo с параметром -i. Хотя, я бы на его месте посоветовал универсальный патч Бармина: rm -rf / --no-preserve-root.Вводить нужно так: 
sudo -i
И тогда у вас должно получиться 
[[email protected] ~]$ sudo -i
[sudo] password for vova:
[[email protected] ~]#
[[email protected] ~]# whoami
ВНЕЗАПНО root
Обратите внимание: знак $ в приглашении означает, что вы пользователь, а знак #, что вы root. Всегда обращайте на это внимание, ибо на форумах могут не писать от чьего имени должна выполняться команда, т.к. эти знаки подразумевают, что пользователь как минимум прочитал Wiki.
Use the brain and RTFM, Luke!
