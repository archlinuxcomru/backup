Про syslog-ng:
==> syslog-ng now reads from /run/systemd/journal/syslog in the
default config. If you still use sysvinit/initscripts, this
must be /dev/log in /etc/syslog-ng/syslog-ng.conf
upgraded syslog-ng (3.3.6-1 -> 3.3.6-3)
Если Вы не перешли на systemd, то верните обратно "/dev/log“ вместо нового пути ”/run/systemd/journal/syslog" в файле конфигурации.
Монтирование, скорее всего, поломалось в связи с заменой console-kit. Точно трудно сказать… Поведайте для начала, какой DM используете.
