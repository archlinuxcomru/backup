Приветствую всех. Появилась проблема с пульсовым эквалайзером - при подключении модуля, pulseaudio падает:
[[email protected] ~]$ pactl load-module module-equalizer-sink
Ошибка подключения: Соединение завершено

Система: Arch Linux, pulseaudio 3.0-3, аудиокарта Asus Xonar DX1, WM - WMFS, запускаемый с помощью slim.
В .xinit pulse не упоминается. Пользователь santic расположен в группе audio.
При старте системы и иксов pulseaudio автоматом запускается, в домашнем каталоге santic папки .pulse нет.
Самое интересное: если создать в домашнем каталоге santic папку .pulse и перезапустить пульс - пульса не поднимается:

[[email protected] ~]$  pulseaudio --start
N: [pulseaudio] main.c: User-configured server at {f785b2f96d974a698d8a92f6d1ca8059}unix:/run/user/3/pulse/native, which appears to be local. Probing deeper.
E: [pulseaudio] main.c: Запуск демона не удался.
[[email protected] ~]$

Без папки ~/.pulse pulseaudio-equalizer-gtk не будет запускаться.

Содержание /etc/pulse/client.conf
; default-sink =
; default-source =
; default-server =
; default-dbus-server =
autospawn=no
; autospawn = yes
; daemon-binary = /usr/bin/pulseaudio
; extra-arguments = --log-target=syslog
; cookie-file =
; enable-shm = yes
; shm-size-bytes = 0 # setting this 0 will use the system-default, usually 64 MiB
; auto-connect-localhost = no
; auto-connect-display = no

Содержание /etc/pulse/default.pa
http://pastebin.com/F6AjW7Dy

Собственно не могу понять, почему pulse запускается автоматом и не позволяет локальные настройки для конкретного юзера?
И почему падает pulseaudio при подключении модуля?
Всем спасибо за внимание!
