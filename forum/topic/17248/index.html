У меня подобное наблюдалось если перед выключением/ребутом не остановить transmission-daemon (запускается с помощью юнита при логине). Тогда может думать от пары секунд до минуты.
Просто прописал остановку демона в скрипты, которыми выключаю/перезагружаю комп
systemctl --user stop transmission.service && systemctl poweroff               # systemctl reboot
Теперь если и появляется такое, то на 1-2 секунды - видимо ещё какие-то пользовательские сервисы тормозят, у меня их ещё парочка есть.

P.S> сейчас чуть изменил
systemctl --user stop {transmission,cache,psd,mpd} && systemctl poweroff
поглядим, что это даст

Тоже вылазит не часто, но иногда ещё вылазит с С4.

Сделал так как советуют:
Install watchdog

# pacman -S watchdog

And then start the service at boot:

# systemctl enable watchdog.service

Start the service to don't see the message any more

# systemctl start watchdog.service

Пока всё нормально, больше не вылазит "A stop job is running for session c2 of user ..."
