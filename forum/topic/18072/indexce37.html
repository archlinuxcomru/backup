Xatk утилита для назначения хоткеев на окна приложения
Позволяет назначить хоткей на запуск приложения и переключения на его окно. При нажатии на хоткей запускается приложение, если оно уже запущено то переключается на окно этого приложения, если окон несколько перебираются окна этого приложения ( можно также переключиться на конкретное).
Если хоткей не назначен, он присваивается автоматически.
Подсказка какой хоткей выводится в заголовок окна.

pps
- сейчас скрипт в Ледяном выглядит так:

#!/bin/sh
#knn 190714+krasnyh
fileName="/tmp/trans.txt"

t=`xsel -o | trans -b -i`
echo -e "$t" > $fileName

yad --title="Перевод" --window-icon="logo-32" --text-info --filename=$fileName --text-align=left --wrap --geometry=500x300+50+370

rm $fileName

exit 0

'trans' и 'xsel' вроде ставится пакетом. Плюс назначение скрипта на хоткей в конфигах WM~ов.
( сначала для Ледяного был включен вариант для терминала sakura, затем перешли на yad-вывод)

Не знаю вкусно полезно ли, но давно раздражало, что в лог пакмана добавили секунды и таймзону, имхо, только засоряющие журнал. Не придумал ничего более цивилизованного, чем добавить хук:
/etc/pacman.d/hooks/20-date_clean.hook
-----
[Trigger]
Operation = Install
Operation = Upgrade
Operation = Remove
Type = Package
Target = *
[Action]
Description = Seconds, timezone remove...
When = PostTransaction
Exec = /usr/bin/sed -i 's/\(^\[.*\)T\(.*\):..+.*\(\]\)\( \[.*\)/\1 \2\3\4/' /var/log/pacman.log
Результат:
$ tail -n5 /var/log/pacman.log
