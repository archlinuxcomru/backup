Вчера ставил Арч и столкнулся с такой же проблемой. Речь ведь о кабеле? У меня была старая флешка с Арчем. Загрузился. Комп на кабеле подключен к роутеру, который раздаёт DHCP. Все прекрасно, инет есть. Поставил базу, перегружаюсь - инета нет. Посмотрел по 
ip a
Интерфейс опущен. Поднимаю 
ip link set enp4s0 up
Поднялся, но ничего не пингую. Ладно, думаю, назначу вручную настройки в диапазоне роутера 
ip a add 192.168.1.101/24 enp4s0
ip route add default via 192.168.1.1
в /etc/resolv.conf вписывал 
nameserver 8.8.8.8
и адрес роутера, и адрес DNS провайдера - всё безрезультатно, точнее интерфейс поднят, ip-адреса пингую, а ya.ru не пингую. Потом нашел такое решение.
cp /etc/netctl/examples/ethernet-dhcp /etc/netctl/ed
nano /etc/netctl/ed
В строку Interface=eth0 впмсал свой интерфейс - enp4s0
ip link set enp4s0 down
netctl start ed
Вуаля! Всё заработало))
