Да и не всякая винда грузиться с GPT.
  • Форточка грузится с GPT только в режиме UEFI, и только если она была установлена на GPT в режиме UEFI.
  • Соответственно, с MBR форточка грузится только в режиме BIOS, и только если она была установлена на MBR в режиме BIOS.
  • 32-битная XP вообще не может грузиться с GPT и не устанавливается на него.

линуксовый загрузчик сможет загрузить с GPT диска винду без UFI, понятно что виндовый не сможет, а вот к примеру grub2 сможет ?
В дополнение к моему предыдущему ответу:

Форточка именно потому и не может загрузиться в "неправильной" для неё конфигурации, что у неё свой загрузчик, причём свой для каждого варианта – либо загрузчик для BIOS+MBR, либо для UEFI+GPT. Тот или другой загрузчик устанавливается и настраивается в процессе установки самой форточки.

Линуксовый загрузчик не может загрузить форточку, он может лишь запустить штатный форточный загрузчик, который, в свою очередь, прочтёт из форточного реестра список необходимых для загрузки драйверов, загрузит ядро форточки со всеми этими драйверами, и запустит форточку.

Warning: If you are dual-booting Windows in the same drive, remember that Windows cannot boot from GPT disk in BIOS mode. If you have already installed Windows in MBR drive which boots via BIOS, do not convert the drive to GPT as Windows will fail to boot, irrespective of the bootloader used to chainload Windows. You need to install Windows in UEFI mode and use one of the UEFI Bootloaders to chainload Windows if you are booting from GPT drive. This is a limitation in Windows.
