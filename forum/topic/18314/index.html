[РЕШЕНО] Не получается загрузить скетч на плату Arduino uno

SkyRewir
# 4 года, 5 месяцев назад (отредактировано 4 года, 5 месяцев назад)
Темы: 19
Сообщения: 117
Участник с: 18 января 2018
Здравствуй, Форум!
Помогите помогите пожалуйста решить проблему с платой arduino uno.
1.Подключил плату по кабелю UBS type A-B (вроде как, обычный кабель для принтера).
2.Загрузил с оф. сайта arduino ArduinoIDE
3.Открыл IDE
4.Выбрал модель платы Arduino/Genuino Uno
5.Выбрал порт /dev/ttyUSB0
6.Пытаюсь загрузить стандартный скетч Blink из библиотеки Basics
В консольном окно IDE выводит

Скетч использует 928 байт (2%) памяти устройства. Всего доступно 32256 байт.
Глобальные переменные используют 9 байт (0%) динамической памяти, оставляя 2039 байт для локальных переменных. Максимум: 2048 байт.
avrdude: ser_open(): can't open device "/dev/ttyUSB0": Permission denied
Произошла ошибка при загрузке скетча
Как я понял проблемы с правами доступа
Вывод команды

ls -l /dev/ttyUSB0
crw-rw-rw- 1 root uucp 188, 0 мар 14 05:40 /dev/ttyUSB0
Т.е доступ к этой директории есть у root и uucp

Все решилось установкой пакета arduino из community и добавлением пользователя в группу uucp

yaourt arduino
sudo usermod -a -G uucp userName
В ArduinoIDE не определялась плата, решилось установкой пакета arduino-avr-core из community

yaourt arduino-avr-core
