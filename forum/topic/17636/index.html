Подскажите, пожалуйста, как можно разрешить обычному пользователю монтировать NTFS-разделы?
С Арчвики ознакомился ознакомился, ntfs-3g-fuse поставил, присваивал папку для монтирования себе, fstab настроил:
/dev/sda2 /mnt/Windows ntfs-3g gid=users,x-gvfs-show,noauto 0 0
/dev/sda3 /mnt/Storage ntfs-3g gid=users,x-gvfs-show 0 0
Пробовал по коментариям в Aur к этому пакету делать так:
# groupadd ntfsuser
# chown root:ntfsuser $(which ntfs-3g)
# chmod 4750 $(which ntfs-3g)
# gpasswd -a <allowed-user> ntfsuser,disk
А все равно запрашивает пароль(

Использую bashmount, который использует polkit. Нижеизложенное не использую, но нашел в архивах.

/etc/polkit-1/rules.d/10-enable-mount.rules

polkit.addRule(function(action, subject) {
    if (action.id == "org.freedesktop.udisks2.filesystem-mount-system" && subject.isInGroup("storage")) {
        return polkit.Result.YES;
    }
});

Работоспособность сейчас не проверял. Пробуйте...

P.S.: bashmount работает и без добавления пользователя в группу storage.
