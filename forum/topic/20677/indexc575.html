Если для диска нет загрузочных записей в nvram материнки, то прошивка UEFI сканирует диск на наличие раздела ефи EF00 и ищет на этом разделе файл /EFI/Boot/bootx64.efi
Если такой файл найден он запускается.
Все понятно - а то мне не давал покоя принцип дефолтного нахождения загрузчика Grub
Ошибки не исчезают с опытом - они просто умнеют

Мож ефи раздел не примонтировал, может не туда примонтировал, может заморочки с nvme. Во второй раз вообще ,,биос,, устовочную команду вводил. Гадать без вывода команд и вывода mount …..
Я пробовал не один раз.
Вот что я вводил:
mkfs.vfat -F32 /dev/nvme0n1p1
mkswap /dev/nvme0n1p2
mkfs.ext4 /dev/nvme0n1p3
mkfs.ext4 /dev/nvme0n1p4

mount /dev/nvme0n1p3 /mnt
mkdir -p /mnt/boot/efi
mount /dev/nvme0n1p1 /mnt/boot/efi
mkdir -p /mnt/home
mount /dev/nvme0n1p4 /mnt/home
swapon /dev/nvme0n1p2

Да и как бы там не было, проблему я решил через bootctl.

mkfs.fat -F32 /dev/nvme0n1p1

pacman -S intel-ucode

bootctl install

nano /boot/loader/loader.conf
default arch
timeout 5
editor 1

nano /boot/loader/entries/arch.conf
title Arch Linux
linux  /vmlinuz-linux
initrd  /intel-ucode.img
initrd  /initramfs-linux.img
options root=UUID=”a609fa76-c5f4-4895-be7f-c18743f88dae”  rw
