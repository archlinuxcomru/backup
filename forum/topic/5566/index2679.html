Привет всем!
Извиняюсь, если вопрос глупый, но я никак не пойму в чем дело.. Сегодня вот познакомился с самбой, задача банальна: файлпомойка, на которую могут ходить как с линуксов так и с венды (постепенно венда исчезнет). Ходить будут всего 3-4 человека, всем все можно.
smb.conf
[global]
	server string = Samba File Server
	security = SHARE
	log file = /var/log/samba/%m.log
	dns proxy = No
	hosts allow = 192.168.1., 192.168.2., 127.
[public]
	comment = Public Folder
	path = /home/sambauser
	force user = sambauser
	force group = sambausers
	read only = No
	create mask = 0770
	force create mode = 0770
	directory mask = 0770
	force directory mode = 0770
	guest ok = Yes
Все вроде нормально, с вендовой машинки проверил - работает, через gvfs-samba (то есть Переход - Соединиться с сервером в GNOME) тоже работает прекрасно. Но когда пробую смонтировать с терминала:
mount.cifs //192.168.2.3/public /mnt/sambaFiles/ -o user=,password=
монтирование происходит, smbstatus показывает что клиент есть, но.. я ничего не могу попасть в примонтированный каталог, пишет 
Не удалось показать содержимое папки. Вам запрещено просматривать содержимое объекта «sambaFiles».
Подскажите, что-то я наверное монтирую не так.. Заранее спасибо всем кто откликнется!

ps: посмотрел на опции примонтированного каталога через gvfs: 
gvfs-fuse-daemon on /home/gard/.gvfs type fuse.gvfs-fuse-daemon (rw,nosuid,nodev,user=gard)
попробовал их, результат такой же..
