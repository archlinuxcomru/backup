Ещё вот такую штуку прикрутил

<action=urxvtc -name update -e yaourt -Syua>%XMUpdate%</action>

Сам скрипт XMUpdate

#!/bin/zsh
upd1=`checkupdates | wc -l`
if [ $upd1 -gt 0 ]; then echo '<fc=#cccccc>  Update:</fc><fn=2> </fn><fn=1><fc=#ff6500>'$upd1'<fn=2> </fn>pkg</fc></fn>'
fi
exit 0

В общем, суть в том, что если обновлений нет, то и надписи на панели нет. Если обновления появились, то при клике по надписи открывается терминал, в котором нужно ввести пароль для запуска обновлений
Проверено пока только теоретически - обновлений сейчас нет, но каждая фишка по отдельности работает нормально: надпись появляется, как и положено (уже пару дней использую этот скрипт), окно терминала тоже открывается в нужном формате.
Я для него в xmonad.hs прописал отдельное правило namedScratchpad

NS "update"       "urxvtc -name update -e yaourt -Syua" (appName    =? "update")      (customFloating $ W.RationalRect 0.15 0.2 0.7 0.6)
