Информация для тех, кто использует для вывода погоды gis-weather
В новой версии gis-weather-git (0.8.2.60) изменён способ ввода текущей температуры и погоды. В ближайшее время вывод в каналы cur_temp и cur_weather будет отключён, а переменные удалены.
В настоящий момент вывод реализуется следующим образом
    "pipe1": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/pipe1",
        "text": "{t_now}"
    },
    "pipe2": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/pipe2",
        "text": "{t_now}"
    },
    "pipe3": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/pipe3",
        "text": "{pressure_now}"
    },
    "text_file1": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/text_file1",
        "text": "{t_now}"
    },
    "text_file2": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/text_file2",
        "text": "{t_now}"
    },
    "text_file3": {
        "active": false,
        "path": "/home/haron/.config/gis-weather/text_file3",
        "text": "{t_now}"
    },
Данный кусок кода (или его часть) следует самому добавить в $HOME/.config/gis-weather/gw_config1.json, активировать ("active": true) нужные каналы (или файлы) и указать, что именно туда будет выводиться
К примеру
    "pipe1": {
        "active": true,
        "path": "/home/haron/.config/gis-weather/pipe1",
        "text": "{icon_now}"
    },
    "pipe2": {
        "active": true,
        "path": "/home/haron/.config/gis-weather/pipe2",
        "text": "{t_now}"
    },
    "pipe3": {
        "active": true,
        "path": "/home/haron/.config/gis-weather/pipe3",
        "text": "{pressure_now}"
    },
* Предупреждение - данные из каналов должны считываться по очереди в порядке возрастания, т.е. pipe1, pipe2, pipe3
Имейте это в виду при настройке своих скриптов
