Приветствую, пытаюсь после обновы заставить pipewire-pulse делать апмикс с стерео 2.0 на 5.1, как это ловко делал pulseaudio.

Собственно ранее в конфиге пульсы добавлялись такие строки и все красиво играло.

remixing-produce-lfe = yes
lfe-crossover-freq = 80
remixing-use-all-sink-channels = yes
Получалось что, если приложение выдавало звук 2.0, пульса разлаживала его на 5.1, причем для саба был ещё и софтовый кроссовер.

Пытаюсь тоже самое завести на новомодном PipeWire. Получается пока не очень.
Покопав немного документации при помощи опции в конфиге
channelmix.mix-lfe = true
удалось только корректно запустить микс на центр и сабвуфер, тыловые колонки молчат.
Пытался курить документацию и issues, пока ни черта не понимаю, везде обсуждают какие-то Джеки и Карлы.

Может кто шарит что там происходит? Подтолкните куда копнуть.

Я тоже задался таким вопросом. Как я это решил для себя:
sudo cp /usr/share/pipewire/pipewire-pulse.conf /etc/pipewire/
sudo vim /etc/pipewire/pipewire-pulse.conf

pulse.default.position = [ FL, FR, RL, RR, FC, LFE ]
channelmix.upmix = true

systemctl --user restart pipewire pipewire-pulse
systemctl --user daemon-reload

pactl info

----
Пробовал играться с настройками типа:
#channelmix.lfe-cutoff = 200
#channelmix.mix-lfe = true

В итоге понял, что стандартные настройки и так хорошо, для меня, работают. Возможно, мои колонки и так хорошо это делают сами. Или это делает pipewire. Я не понял... но саб работает хорошо. Колонки у меня: edifier s550, аудио встроенное в материнку

Мои настройки для звука 5.1, мне нравится, т.к. басс мягкий идет с сабвуфера.

#node.latency = 1024/48000
#node.autoconnect = true
#resample.quality = 4
#channelmix.normalize = false
#channelmix.mix-lfe = true
channelmix.upmix = true
channelmix.lfe-cutoff = 150
Настройки выставленны для /pipewire/client.conf,client-rt.conf,pipewire-pulse.conf

Еще дополнительно имспользовал совет настройки для alsa - https://forum.rosalinux.ru/viewtopic.php?t=5449
