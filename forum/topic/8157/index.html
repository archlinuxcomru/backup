балин, моя проблема пятилетней давности, но я как бы даже без инета решение нашел :)

этот самый getch есть как раз в ncurses а согласно man getch:
The getch, wgetch, mvgetch and mvwgetch, routines read a character
from the window. In no-delay mode, if no input is waiting, the val‐
ue ERR is returned. In delay mode, the program waits until the sys‐
tem passes text through to the program. Depending on the setting of
cbreak, this is after one character (cbreak mode), or after the
first newline (nocbreak mode). In half-delay mode, the program
waits until a character is typed or the specified timeout has been
reached.

дальше бежим в самый низ и смотрим See also: man curs_inopts, а там:
int halfdelay(int tenths);
int nodelay(WINDOW *win, bool bf);

The halfdelay routine is used for half-delay mode, which is similar to cbreak mode in that characters typed by the
user are immediately available to the program. However, after blocking for tenths tenths of seconds, ERR is re‐
turned if nothing has been typed. The value of tenths must be a number between 1 and 255. Use nocbreak to leave
half-delay mode.

The nodelay option causes getch to be a non-blocking call. If no input is ready, getch returns ERR. If disabled
(bf is FALSE), getch waits until a key is pressed.
