Вчера поставил ядро с BFS. Ощущения приятные, ютуб в HQ и фулскрине действительно не тормозит(но грузит С2D T7700 на ~70%):

Ставил из AUR, пакет kernel26-bfs. Ядро 2.6.31, патч 230 - самые свежие. Ядро поставилось в одно движение yaourt'ом, но слетели драйвера на видео от nVidia. Пакет nvidia-bfs ставиться не захотел: в dependency прописано kernel26-bfs только 2.6.30. Пришлось ставить nvidia-bfs-beta; в pkgbuild ошибки, он может поставиться только в систему x86_64, у меня же просто x86, кроме того, там указан kernelname(который uname -r) 2.6.31-bfs; по дефолту ядро ставится как 2.6.31-bfs230-bfs. Правильный pkgbuild такой:
#Maintainer: <SpeedVin at archlinux dot us>
pkgname=nvidia-bfs-beta
pkgver=190.32
_kernver='2.6.31-bfs230-bfs'
pkgrel=1
pkgdesc="NVIDIA drivers for kernel26-bfs."
arch=('i686' 'x86_64')
[ "$CARCH" = "i686"   ] && ARCH=x86
[ "$CARCH" = "x86_64" ] && ARCH=x86_64
url="http://www.nvidia.com/"
depends=('kernel26-bfs>=2.6.31' 'kernel26-bfs<2.6.32' 'nvidia-utils-beta=190.32')
conflicts=('nvidia-96xx' 'nvidia-173xx')
license=('custom')
install=nvidia-bfs.install
source=("http://download.nvidia.com/XFree86/Linux-$ARCH/${pkgver}/NVIDIA-Linux-$ARCH-${pkgver}-pkg0.run")
build() {
cd $srcdir
sh NVIDIA-Linux-$ARCH-${pkgver}-pkg0.run --extract-only
cd NVIDIA-Linux-$ARCH-${pkgver}-pkg0
cd usr/src/nv/
ln -s Makefile.kbuild Makefile
make SYSSRC=/lib/modules/${_kernver}/build module || return 1
mkdir -p $pkgdir/lib/modules/${_kernver}/kernel/drivers/video/
install -m644 nvidia.ko $pkgdir/lib/modules/${_kernver}/kernel/drivers/video/
sed -i -e "s/KERNEL_VERSION='.*'/KERNEL_VERSION='${_kernver}'/" $startdir/nvidia-bfs.install
}
md5sums=('0cf2686f8ff4112ed07ed6d35c013827')
и еще нужно будет поправить md5sum в конце, этот для версии x86_64. Драйвер работает хорошо(несмотря на то, что он beta), еще ни разу не падал.
Сообщение считать отчетом о проделанной работе :-)
