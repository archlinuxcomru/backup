Доброе...
Стоит сервер на CentOS7, самба север с таким конфигом


[global]
unix charset = UTF-8
workgroup = WORKGROUP
server string = Samba Server %v
netbios name = fh13
hosts allow = 127. 192.168.1.
security = users
map to guest = bad user
dns proxy = no
guest account = nobody
        load printers = no
        printing = bsd
        printcap name = /dev/null
        disable spoolss = yes

log file = /var/log/samba/log.%m
max log size = 50
#============================ Share Definitions ==============================
[upload]
comment = upload_this
path = /share
guest ok = yes
guest only = yes
browsable =yes
writable = yes
read only = no
force create mode = 0777
public = yes
directory mode = 0777

конфиг был простой, разросс по причине того, что думал или думаю, что проблема в сервере, но ...
По виной нормально подключается, создаются папки, папки в паках как и должно быть.

под арчем пробую следующую команду


sudo mount.cifs //192.168.1.13/upload ~/fh13_upload -o user=nobody,passwork=,workgroup=WORKGROUP,rw,ip=192.168.1.13

Ошибки нет. шару вижу, но не могу в ней создать папку, и файл - пишет что отказано в доступе...
как понимаю, что не подключился мой арч как nobody. То же самое если делаю шару по паролю, винда нормально, арч нивкакую...

почитал тут и нашел вот это:

sudo mount.cifs //192.168.1.13/upload ~/fh13_upload -o password=,user=,workgroup=WORKGROUP,rw,ip=192.168.1.13,nosuid,nodev,users,uid=nobody,gid=users

но результат тот же

вот вывод команды ls:

-rwxr-xr-x+  1 nobody    users 1,5M дек  6 12:47 Dav_Player.exe
-rwxr-xr-x+  1 nobody    users  24M дек  6 12:47 KMPlayer_3.3.0.33.exe
-rwxr-xr-x+  1 nobody    users  27M дек  6 12:47 vlc-2.1.3-win32.exe
drwxr-xr-x+  4 nobody    users    0 дек  8  2015 Новая папка
или

-rwxr-xr-x+  1 nobody    nobody 1,5M дек  6 12:47 Dav_Player.exe
-rwxr-xr-x+  1 nobody    nobody  24M дек  6 12:47 KMPlayer_3.3.0.33.exe
-rwxr-xr-x+  1 nobody    nobody  27M дек  6 12:47 vlc-2.1.3-win32.exe
drwxr-xr-x+  4 nobody    nobody    0 дек  8  2015 Новая папка
В зависимости какой из команд монтирую...
Что не так?) в сервере или арч недомонтирует чего-то?

Пробовал - ругается на права так же...
пробовал на при монтировании права давать - не меняются
это под рутом

mkdir: невозможно создать каталог «/home/firefedot/fh13_upload/new»: Отказано в доступ

Поменял не много последовательность


sudo mount -t cifs -o user=nobody,password= //192.168.1.13/upload ~/fh13_upload/

После этого под рутом могу файлы создавать и они получаются как созданные nobody, но под простым юзером не дает, и права менять не дает - говорит отказано в доступе

Если делать chmod -R 777 /~fh13_upload, то конечно файл создаст, а если создаст директорию, то внутри нее, уже пользователь снова безсилен

получается, что если через sudo - то как nobody к самба серверу подключается root, а не мой пользоваатель
прчем - понес бук на рабту и там спокойно, тойже командой подключился к местной самбе...
