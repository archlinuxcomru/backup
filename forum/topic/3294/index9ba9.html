dr-kart
А я никак не вкурю что делать? Уже и затертый файлик с изменением внутри вернул. И конечно в группу disk добавился. Так есть решение или пока еще нерешенный баг?
Ставьте polkit-gnome, настраивайте авторизацию:
В файл /etc/polkit-1/localauthority.conf.d/60-desktop-policy.conf пишем
# This allows users in the wheel group to authenticate as the
# administrator.
#
# DO NOT EDIT THIS FILE, it will be overwritten on update.
[Configuration]
AdminIdentities=unix-group:wheel
В файл /var/lib/polkit-1/localauthority/10-vendor.d/10-desktop-policy.pkla пишем
# Authorizations/policy for the wheel group.
#
# DO NOT EDIT THIS FILE, it will be overwritten on update.
#
# Allow users in the wheel group to do certain actions without being
# interrupted by password dialogs
[Wheel Group Permissions]
Identity=unix-group:wheel
Action=org.gnome.settingsdaemon.datetimemechanism.*;org.kde.kcontrol.kcmclock.save;org.freedesktop.RealtimeKit1.*;org.freedesktop.udisks.filesystem-mount-system-internal;org.freedesktop.hostname1.set-static-hostname
ResultAny=auth_admin
ResultInactive=auth_admin
ResultActive=yes
Добавляем себя в группу wheel. Радуемся (наверное).
