( и если вдруг там RM=1, то где покрутить настройки?)
В части RM - изменить это ты не сможешь ....
Значение 1 означает, что это USB-накопитель (вообщем съемный носитель), а 0 НЕ съемный носитель.
Пример: sda - HDD, sdb - USB-HDD, sdc - USB (флэшка), sr - дисковод
lsblk -o NAME,RM,MODEL
NAME    RM  MODEL
sda           0     TOSHIBA_MK6465GSX
├─sda1   0
├─sda2   0
├─sda3   0
├─sda4   0
├─sda5   0
├─sda6   0
└─sda7   0
sdb          0     HDD_SH93
└─sdb1  0
sdc          1     Transcend_16GB
└─sdc1  1
sr0          1     hp_CDDVDW_TS-L633R

И приведи вывод: lsblk -o +HOTPLUG,ROTA,DISC-GRAN,MODEL

В части причины шелчков USB-HDD - это может быть также связано и с проблемами HDD и с нехваткой питания.
Ошибки не исчезают с опытом - они просто умнеют

коробку скорее выкинул.
В терминологии - плаваю.
док-станция --- SATA DOCKING STATION for 2.5/3.5 SATA HDD --- форма треугольная или квадратная, в которую вставляется обычный HDD (от одного и более), как правило, имеет отдельное питание (но не всегда - что выберешь сам) и соединение с компьютером по USB кабелю. Вообщем тот же самый HDD, но вставленный в эту док-станцию и соединенный с компом по USB кабелю.
У меня с отдельным питанием, проблем не замечал, как и шелчков.

USB-HDD - как бы по точнее сказать, почти обычный HDD, но, как правило, сверху покрыть эластичным материалом, типа резины, (то есть нет обычного вида HDD), с боку всего один разъем для короткого USB шнура соединяемого с компом.
Кстати, сколько не пользовался USB-HDD, шелчков никогда не слышал, а вот отваливались (пропадали из системы, то есть не было видно) в ноутбуке частенько, но это было связано с нехваткой питания … рекомендуется в таких случаях использовать USB-HDD с отдельным питанием.

Рекомендую внимательнее просмотреть вывод S.M.A.R.T.
Ошибки не исчезают с опытом - они просто умнеют
