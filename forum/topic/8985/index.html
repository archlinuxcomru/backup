Доброго времени суток всем!

С недавних пор имеется проблема с samba. Стал замечать, процесс smbd стал забирать 100% процессорного времени. Установлена самба версии samba 3.2.5-2

Вывод top:
top - 21:52:22 up 1 day, 22 min,  1 user,  load average: 1.11, 1.17, 1.12
Tasks: 165 total,   3 running, 162 sleeping,   0 stopped,   0 zombie
Cpu(s):  7.5%us, 51.3%sy,  0.0%ni, 41.0%id,  0.2%wa,  0.0%hi,  0.0%si,  0.0%st
Mem:   3636052k total,  3513088k used,   122964k free,   272756k buffers
Swap:  4000176k total,        0k used,  4000176k free,  2684852k cached
  PID USER      PR  NI  VIRT  RES  SHR S %CPU %MEM    TIME+  COMMAND            
 2985 root      20   0 16328 5400 4080 R  100  0.1 663:22.82 smbd

Конфиг самбы(/etc/samba/smb.conf):

[global]
	workgroup = FRELA
	server string = Samba Server 
	security = SHARE
	log file = /var/log/samba/%m.log
	max log size = 50
	dns proxy = No
	wins server = 192.168.7.252
[homes]
	comment = Home Directories
	read only = No
	browseable = No
[printers]
	comment = All Printers
	path = /var/spool/samba
	printable = Yes
	browseable = No
...
далее куча шар с параметрами
guest ok = Yes
read only = yes #у некоторых No
browseable = Yes

Перезапуск самбы не помогает: 
 /etc/rc.d/samba restart
:: Stopping Samba Server                                                 [DONE] 
:: Starting Samba Server                                                 [FAIL]

При этом smbd продолжает висеть в списке процессов:
ps ax | grep smbd
 2985 ?        R    673:58 /usr/sbin/smbd -D

Самое интересное в том, что kill -9 не убивает “подвисший” процесc smbd. Помогает только ребут.
Может кто-нибудь сталкивался с подобной проблемой и может мне помочь?
