Если не использовать блоб (amdgpu-pro или там catalyst), то я лично использую следующий вариант. Имеем ноут с гибридной графикой ATI\Intel:

1. в mkinitcpio.conf
MODULES="radeon"
и
$sudo mkinitcpio -p linux

2. в /etc/X11/xorg.conf.d/ следующий конфиг:
20-gpu.conf

[Section "ServerLayout"
    Identifier "layout"
    Screen 0 "intel"
    Screen 1 "radeon" LeftOf "intel"
EndSection

Section "Device"
    Identifier "radeon"
    Driver "radeon"
    BusID "PCI:3:0:0"
    Screen 1
    Option "DRI" "3"
    Option "TearFree" "on"
    Option "AccelMethod" "glamor"
    Option "EnablePageFlip" "on"
EndSection

Section "Device"
    Identifier "intel"
    Driver "intel"
    BusID "PCI:0:2:0"
    Screen 0
    Option "DRI" "3"
EndSection

Section "Screen"
    Identifier "intel"
    Device "intel"
    Monitor "monitor"
    DefaultDepth 24
EndSection

Section "Screen"
    Identifier "radeon"
    Device "radeon"
    Monitor "monitor"
    DefaultDepth 24
EndSection

Section "Monitor"
    Identifier "monitor"
    Option "Enable" "true"
EndSection

ну ессна подправить там тонкости типа
BusID "PCI:3:0:0"
для себя.
Если видяшка работает со свободным драйвером amdgpu - можно просто в конфиге поменять radeon на amdgpu (точно так же MODULES="radeon" на MODULES="amdgpu") и тогда можно из секции радеон конфига выкинуть.

Option "DRI" "3"
    Option "TearFree" "on"
    Option "AccelMethod" "glamor"
    Option "EnablePageFlip" "on"

и добавить в /etc/modprobe.d/

конфиг radeon.conf с содержимым


blacklist radeon

Ну и ессна надо установить дополнительно пакет xf86-video-amdgpu

Собсна вся настройка. По умолчанию будет работать интелевский драйвер, атишный будет задействоваться через
DRI_PRIME=1 имя проги

Чтоб блоб поставить - читай wiki, там надо xorg даунгрейдить до 117 из Виоловского репа. Но вообще это ненужный гемор - действительно лучше пользоваться или свободным драйвером radeon\amdgpu или если уж заморачиваться с блобом, то (в случае если карточка позволяет) лучше поковыряться с amdgpu-pro, для этого есть в AUR пакет. Но имхо и свободного драйвера позаглаза, я к примеру на xf86-video-radeon\amdgpu спокойно играю в EVE online с галлиум вайном. 60-75 fps

З.Ы. кстати самое гнусное занятие - это ставить официальные драйвера с сайта (неважно АТИ или нвидия), ставить надо адаптированные под конкретный дистрибутив пакеты из его родных (или неофициальных хотя бы) репозиториев. Дрова на оф-сайте больше нужны мейнтейнерам, чтоб их в полноценный пакет превратить, а не конечному пользователю.

[[email protected] ~]$ pacman -Ss xf86-video-ati
extra/xf86-video-ati 1:7.9.0-1 (xorg-drivers) [установлен]
    X.org ati video driver
[[email protected] ~]$

$sudo pacman -S xf86-video-ati mesa lib32-mesa mesa-vdpau lib32-mesa-vdpau mesa-demos lib32-mesa-demos opencl-mesa vulkan-radeon lib32-vulkan-radeon

P. S. вафлю брутить можно к слову и виндовозным аиркряком, есть и для офтопика тулзы удобные. Мучаться ТОЛЬКО ради этого с арчем не стоит.
