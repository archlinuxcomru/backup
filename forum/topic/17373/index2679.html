Не запускается Arch после разрядки батареи на ноутбуке.

newb
# 5 лет, 4 месяца назад
Темы: 1
Сообщения: 2
Участник с: 17 апреля 2017
Здравствуйте!
Суть: разрядился ноутбук с Arch. После зарядки и включения сообщил о "duplicate entry" в fstab, затем запускал file system check. Далее всё шло не по плану и меня выкидывало в emergency mode с предложением ввести пароль рута или Control-D для продолжения.
В fstab действительно было 2 записи с /dev/sda1. Комментирование одной и другой по очереди положительного результата не принесло.
Через лайв-образ сгенерировал новый fstab:

#
# /etc/fstab: static file system information
#
# <file system>	<dir>	<type>	<options>	<dump>	<pass>
# /dev/sda3
UUID=9aacc92b-b0fe-46e6-b1c6-befc3b3c4901	/         	ext4      	rw,relatime,data=ordered	0 1

# /dev/sda1
UUID=2596e9ee-df3e-492f-af18-94f39ece7cc9	/boot     	ext4      	rw,relatime,stripe=4,data=ordered	0 2

# /dev/sda4
UUID=8dc53115-6b87-4494-bb46-0a9453707d74	/home     	ext4      	rw,relatime,data=ordered	0 2

# /dev/sda2
UUID=2863f81c-55d4-4051-b5d0-eb03763c3b04	none		swap		default				0 0
Раздел со swap дописал в него руками. На всякий случай сгенерировл новый grub.cfg под arch-chroot.
Сейчас на дубликат в fstab не ругается, однако снова валится при file system check в emergency mode. Перед этим пишет:
[failed] failed to start file system check on /dev/disk/by-uuid/8dc53115-6b87-4494-bb46-0a9453707d74
[depend] dependency failed for /home.
[depend] dependency failed for Local File Systems.
Подскажите, пожалуйста, что я сделал не так, и как это исправить.

У тебя повреждена файловая система из-за экстерeнного отключения питания. Там же вроде пишется что прописать надо, чтобы отремонтировать систему. В этом случае в emergency mode:
fsck /dev/sda4
ну, или
fsck UUID=8dc53115-6b87-4494-bb46-0a9453707d74
