Вообще то в последних xorg.conf использую "Coolbits" "28", где 2 это разрешение управление кулером, а 8 - разрешение на вмешательство в PowerMizer.
Правда у меня gtx 1650 OC, где вентилятор изначально вращается на 30%.
Да, и забыл: в конфиге автозапуска nvidia-settings -a [gpu:0]/GPUFanControlState=1 для разрешения ручного управления.
Если в nvidia-settings -a [gpu:0]/GPUFanControlState=1 изменить на
nvidia-settings -a [gpu:0]/GPUFanControlState=0 то при загрузке будет автоматический режим вентилятора, и скорее всего как раз и будет ошибка failed to set new fan speed.

Ну это вряд ли. Попробуйте вариант, который у меня срабатывает на все сто процентов.
По порядку.
/etc/X11/xorg.conf
Section "Device"
Identifier "Device0"
Driver "nvidia"
VendorName "NVIDIA Corporation"
BoardName "GeForce GTX 1650"
Option "Coolbits" "28"
EndSection
Перезагрузка

В /home/username/.local/ создаем текстовый документ со скриптом с любым названием:
#!/bin/sh
nvidia-settings -l
nvidia-settings -a [gpu:0]/GPUPowerMizerMode=2
nvidia-settings -a [gpu:0]/GPUGraphicsClockOffset[2]=-500
nvidia-settings -a [gpu:0]/GPUMemoryTransferRateOffset[2]=-150
nvidia-settings -a [gpu:0]/GPUFanControlState=1
exit 0

Вас интерисует только нижняя строка: nvidia-settings -a [gpu:0]/GPUFanControlState=1; можете оставить только её.
В свойствах файла разрешаем запуск как программы (делаем исполняемым).
Затем добавляем его(полученный файл) в автозапуск: меню-параметры системы-значек Автозагрузка.
Лучше с незначительной задержкой для надежного применения настроек в файле .nvidia-settings-rc в домашке (яркость, контрастность и т.д.).
Например: sh -c "sleep 0.5 && /home/username/.local/имя файла"
Перезагрузка.

P.S. Извините, забыл: строку nvidia-settings -l оставьте тоже сверху. Она назначает настройки файла .nvidia-settings-rc по умолчанию при загрузке, перезагрузке. Незначительная задержка в автозапуске как раз и нужна этой строке.
