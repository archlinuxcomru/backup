UPD: Переосмыслил структуру, сведя всё к одному компьютеру, вопрос закрыт за ненадобностью.

Доброго времени суток, господа форумчане.

Впервые пользуюсь ArchLinux и Linux в целом, просидел около недели в попытках самостоятельной настройки, сталкиваясь с разными проблемами, но решил всё же создать тему)

Раздаю интернет с телефона, используя его как USB-модем. Провайдер Tele2, ограничение на раздачу обхожу сменой TTL.
Пытаюсь расшейрить интернет через локальную сеть ( роутер, без интернета ) на два компьютера с Win7.

Суть проблемы: Получить доступ к интернету удалось, однако с обоих компьютеров на Win7 все сайты с браузеров грузятся по пару минут ( DNS пробовал как оператора, так и Cloudflare, разницы нет ).

Интерфейс с интернетом от телефона получает настройки провайдера через DHCP ( dhcpcd установлен ).
Интерфейс с локальной сетью настроен по wiki ( Internet Sharing ):
- Статический IP установлен через systemd-networkd ( сервис включен ).
- Пересылка пакетов указана как в /etc/sysctl.d/30-ipforward.conf, так и в /etc/systemd/networkd.conf
- Команды iptables введены и сохранены в /etc/iptables/iptables.rules.

Лог ip a:
2: lan: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 00:80:48:38:7a:64 brd ff:ff:ff:ff:ff:ff
    inet 192.168.0.100/24 brd 192.168.0.255 scope global lan
       valid_lft forever preferred_lft forever
3: inet: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UNKNOWN group default qlen 1000
    link/ether 5a:ad:e1:d2:c3:8c brd ff:ff:ff:ff:ff:ff
    inet 192.168.42.22/24 brd 192.168.42.255 scope global dynamic noprefixroute inet
       valid_lft 3573sec preferred_lft 3006sec

Лог ip r:
default via 192.168.42.129 dev inet proto dhcp src 192.168.42.22 metric 203
192.168.0.0/24 dev lan proto kernel scope link src 192.168.0.100
192.168.42.0/24 dev inet proto dhcp scope link src 192.168.42.22 metric 203
