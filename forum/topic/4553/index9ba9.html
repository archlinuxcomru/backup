Ясно, что с курсорами пока не ясно.
Новый совет – для параноиков, таких, как я:
Как очистить историю недавно открытых файлов.
Системных средств в G3 для этого пока нет. Поэтому воспользовался идеей с англоязычного форума и создал в ~/.bashrc такой псевдоним:
alias clearhistory='rm -i ~/.local/share/recently-used.xbel'
Эта команда спрашивает (на всякий случай), прибить ли файл с историей. При следующем входе в сеанс (или перезапуске gnome-shell) открытые ранее файлы вас не выдадут.

Такая тема постоянно возникает: Расширения GNOME-Shell (gnome-shell-extensions)
Поэтому краткий ликбез.
На данный момент в extra доступны следующие расширения:
  • gnome-shell-extensions-common – common files
    gnome-shell-extension-alternate-tab – Alternate Tab extension
    gnome-shell-extension-alternative-status-menu – Alternative Status Menu
    gnome-shell-extension-auto-move-windows – Auto Move Windows extension
    gnome-shell-extension-dock – Dock extension
    gnome-shell-extension-drive-menu – Drive Menu extension
    gnome-shell-extension-native-window-placement – Native Window Placement extension
    gnome-shell-extension-places-menu – Places Menu extension
    gnome-shell-extension-user-theme – User Theme extension
    gnome-shell-extension-windows-navigator – Windows Navigator

    О них можно подробнее почитать здесь: GNOME Shell extensions. А увидеть здесь: GNOME SHELL EXTENSIONS: ADDITIONAL FUNCTIONALITY FOR GNOME 3 (в том числе видео).
    Ставятся расширения по одному стандартным образом и так же стандартно удаляются. Т.е. не нужен док – сносим gnome-shell-extension-dock.
    UPD: Читаем также Wiki.

Как добавить приложения в автозагрузку.
Сам мучался пару часов, и, судя по интернетам, там тоже много кто задавался этив вопросом. Может, кому-то и здесь будет интересно.

1. Открываем менеджер сессии:
$ gnome-session-propetries
2. Нажимаем “Добавить”, заполняем все поля. Готово. Однако, есть одно “но”: в поле “Команда” нужно вводить ПОЛНЫЙ ПУТЬ до исполняемого файла (на пример, /usr/bin/tilda). Иначе не заработает и при следующем логине добавленного пункта просто не будет.
