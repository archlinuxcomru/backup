Кста, нашли как сделать светлыми totem и eog?
Штатного способа нет (или я его не знаю). Можно сделать это ручками, но такой трюк перестанет работать после очередного обновления gnome-themes-standard. Поэтому я написал для себя скрипт no-gtk-dark.sh, который (понятное дело) запускается из-под рута:
#!/bin/bash
cd /usr/share/themes/Adwaita/gtk-3.0/ &&
mv gtk-dark.css gtk-dark.css.bak ;
mv gtk-widgets-assets-dark.css gtk-widgets-assets-dark.css.bak &&
ln gtk.css gtk-dark.css ;
ln gtk-widgets-assets.css gtk-widgets-assets-dark.css
totem и eog теперь светлые, но на sushi это, к сожалению, не распространяется.

Подскажите, почему на ntfs разделах файлы удаляются не в корзину, а безвозвратно, если эти разделы примонтированы посредством fstab. Если в ntfs разделы в fstab не прописаны, а в ходе работы монтированы вручную, то все работает нормально. В КДЕ с этим fstab все нормально работает.
мой fstab:
# /etc/fstab: static file system information
#
# <file system>	<dir>	<type>	<options>	<dump>	<pass>
tmpfs		/tmp	tmpfs	nodev,nosuid	0	0
UUID=a6f3b03e-4665-4cc7-80a6-912550fbda52 / ext4 defaults 0 1
/swapfile swap swap defaults 0 0
UUID=01CC4921E2FFDFF0 /media/Work auto defaults 0 1
UUID=01CC49219FF56630 /media/Media auto defaults 0 1
