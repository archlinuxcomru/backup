[РЕШЕНО] Глюк ntfs-3g при использовании в правилах udev

Делаю автомонтирование флешек и другого усб-барахла через правила udev. Всё ок за исключением одного. При выполнении
(опции прав доступа не привожу для сокращение кода)
ACTION=="add", KERNEL=="sd[c-z]", ENV{ID_FS_TYPE}=="ntfs", RUN+="/bin/mount -t ntfs-3g /dev/%k /media/%E{ID_VENDOR}"

монтирование сначала выполняется, а затем каталог точки монтирования становится красным со знаком "?" перед именем. При этом вручную
# mount -t ntfs-3g /dev/sdc /media/<нужный каталог>

всё проходит нормально, при задании прав доступа монтируется с нужными правами. Удаление пакета ntfs-3g (и замена на -t ntfs после этого) приводит к тому что мнотирование проходит через правило, однако монтируется только для чтения (я так понимаю для записи надо задействовать драйвер ntfs-3g)

Вроде было (без решения, да).

В вики прямо прописали:
If you want to mount removable drives please do not call 'mount' from udev rule. In case of fuse filesystems (e.g. ntfs-3g) you'll get "Transport endpoint not connected" error. Instead use udisks that handles automount correctly.
