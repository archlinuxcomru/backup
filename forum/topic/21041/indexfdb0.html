Сегодня специально выделил время для более детального экспериментирования с ntfs3 в части чтения, записи, редактирования, создания файлов/директорий виндового раздела от простого юзера (no-root). Привожу результы (возможно кому то будет и интересно) ...
В основном использовал Wiki NTFS, которая написана для ntfs3
ntfs-3g - удалять не стал, а поступил по рекомендации Wiki - для использования ntfs3 в качестве драйвера по умолчанию для разделов ntfs, создал правило udev, точнее, создал файл /etc/udev/rules.d/99-ntfs3.rules и прописал в нем строчку
SUBSYSTEM=="block", ENV{ID_FS_TYPE}=="ntfs", ENV{ID_FS_TYPE}="ntfs3"
Хотя Wiki это делать не рекомендует - это может сбить с толку некоторые сторонние приложения - какие, там не указано, нужно проверять, могу преположить, что, например, это может быть gparted ...
Первым делом проверил вариант с udisks2 - у меня не сработало - используя pcmanfm можно читать, но вот записать/создать от user не дает.
Тогда внес в fstab строчку для виндового раздела (с минимальными опциями, но их достаточно для простой работы)
grep DATE /etc/fstab
UUID=E54C59DF584D5AFE  /run/media/vasek/DATE  ntfs3 noatime,prealloc,uid=1000,gid=100  0 0
Значения uid и gid можно узнать, используя команды: id -u и id -g
reboot ... и смотрим
ls -l /run/media/vasek/DATE | awk '{print $3,$4}' | head -3
vasek users
vasek users
Вроде все нормально - но всеравно, проверил все (чтение, запись/редактирование, создание файла/директории) - все нормально.
Вот чего не проверил, так это скорость записи ... работу gparted и др. утилит, работающих с ntfs ... но я ими и не пользуюсь.

PS - в итоге пока вернулся к прежнему виду ... пока рановато ... но удалять ничего не стал, просто закомментировал строки.
