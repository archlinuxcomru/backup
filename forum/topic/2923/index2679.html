Имеем роутер NetGear ProSafe FVS114.Который раздает интернет по дому.
Есть машинка с арчем у которого иногда слетает сеть. Почему? Так и не разобрался, по этому прошу помощи у народа.
Получаем IP по dhcp с роутера:
в /etc/rc.conf 
interface=eth0
address=
netmask=
broadcast=
gateway=
DAEMONS=(syslog-ng dbus @network alsa crond gpm sshd acpid samba slim !vboxdrv !vmware @ntpdate)
$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 16436 qdisc noqueue state UNKNOWN 
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP qlen 1000
    link/ether 00:1d:92:f4:16:4f brd ff:ff:ff:ff:ff:ff
    inet 192.168.0.51/24 brd 192.168.0.255 scope global eth0
cat /etc/resolv.conf 
# Generated by dhcpcd from eth0
# /etc/resolv.conf.head can replace this line
nameserver 87.240.1.1
nameserver 87.240.1.2
# /etc/resolv.conf.tail can replace this line
$ ip ro
default via 192.168.0.1 dev eth0  metric 202 
192.168.0.0/24 dev eth0  proto kernel  scope link  src 192.168.0.51  metric 202

Если вначале работает, а потом вдруг перестает, то скорее всего проблемы железные. Попробуйте последовательно поменять пачкорды с другими хостами, поменять порты на роутере, если возможно, заменить (или поменять с другим хостом в сети) сетевую карточку. Посмотреть, на результат. Может удасться уловить какую-либо тенденцию.
Почему я так думаю: был аналогичный случай, когда я вдруг перестал получать адрес по dhcp. Ставлю вручную, начинает работать, потом постепенно умирает. По tcpdump видел приходящие пакеты. Сеть работала при подключении к другому хосту точка-точка. Потом вспомнил, что проблемы начались после пропаданий питания в сети. Так и пришлось менять материнку (сетевой порт был встроенный). После замены работает уже пару лет без проблем.
