Я ставил blueman, через него все соединял, потом в /etc/asound.conf надо было шаманить, прописал туда:
pcm.btheadset {
   type plug
   slave {
       pcm {
           type bluetooth
           device 00:23:7f:cf:5a:22
           profile "auto"
       }
   }
   hint {
       show on
       description "BT Headset"
   }
}
ctl.btheadset {
  type bluetooth
}
Далее в любой программе можно выбирать гарнитуру как микрофон и как динамики.

ANtlord
systemctl status bluetooth.service
Failed to get D-Bus connection: No connection to service manager.
Такая строчка типична, при установке systemd, в комбинированном режиме. Но по прежнему юзается initscripts. D-bus автоматом запускается systemd. Если его запустило не systemd, то доступа у systemd к нему не будет. Насчет указания dbus-launch - не ставил таких экспериментов без него грузить иксы. Попытка спиритеческого сеанса блин.
