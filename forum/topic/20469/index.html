В какой-то момент пару месяцев назад после очередного обновления машины, с включенным USB 2.0/3.0 начали крашиться. Выдает следующую ошибку 
The VM session was aborted.
Result Code:
NS_ERROR_FAILURE (0x80004005)
Component:
SessionMachine

ISession {c0447716-ff5a-4795-b57a-ecd5fffa18a4}

❯ sudo /sbin/rcvboxdrv setup

virtualbox 6.1.16-4
virtualbox-guest-dkms 6.1.16-4

virtualbox-guest-utils-nox 6.1.16-4
virtualbox-host-dkms 6.1.16-4
virtualgl 2.6.4-1
Версия ядра:
❯ uname -a
Linux Mars 5.10.6-arch1-1 #1 SMP PREEMPT Sat, 09 Jan 2021 18:22:35 +0000 x86_64 GNU/Linux
Расширение
Oracle VM VirtualBox Extension Pack 6.1.12
установлено.
Когда ставлю USB 1.1 машины запускаются корректно.

думаю, многие держат в качестве запасного ядра linux-lts
Вообще-то lts-ядро - нормальная практика. Более того: считаю, что свежие ядра больше нужны для нового железа. До сих пор 4.4 поддерживается, если что.
И напротив, для старого железа новые ядра могут стать злом. Например, этому багу три месяца, и только сегодня вроде бы найден виновник.
На древнющем ноуте со свежим ядром вообще даже киношку не могу посмотреть - все тормозит.

Так что для некоторых/многих lts-ядро очень даже может быть основным. У меня ситуация пока следующая:
$ pacman -Qqs kernel
linux
linux-589
linux-lts
