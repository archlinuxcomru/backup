правильно ли я понимаю суть сайта вообще )

есть некий httpd (apache2 lighttpd nginx cheeroky и тд) демон который обрабатывает все запросы приходящие на порт который указан в конфигурации. - это вообщем то и есть веб сервер.
есть некий движОк для сайта написанный на каком либо языке (в случае wordpress это PHP)
существует некая база данных, где хранится … (что именно? контент - тексты, картинки, какие либо объекты?) про сайт (всяческие sql реализации)
все эти элементы каким то образом связаны. (каким?)

В случае запроса пользователя - а покажи ка мне такую то страницу
запрос обработан httpd, в конфиге вебсервера описанно как именно обрабатывать запрос (какому движку отдать), запрос обрабатывается движком сайта, движок “собирает” страницу из “шаблона” наполняя его контентом из базы данных и … (я бы упаковывал, если броузер умеет распаковывать на лету ))) возвращается вебсерверу, вебсервер выдает пользователю - смотри отрок, какая у меня станичка есть!!!


Соответсно, имея бэкап базы данных сайта можно в случае хоста развернуть фарш обратно? Все остальное (конфиги вебсервер и движка) вторично, хотя тоже неплохо бы бэкапить?
