У меня (если не у всех) такое же. Гугл пестрит довольно свежими подобными вопросами. Похоже, nvidia снова облажались:
There is no stable module API.

Therefore this is a problem for the nvidia developers to solve.
https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=899201

The nvidia driver uses an alternate stack, so it's expected for it to do usercopies to and from that. I think we just need to allocate the alternate stacks with kmem_cache_create_usercopy() rather than kmem_cache_create().

We have internal bug number 2107220 filed to track this enhancement.
https://devtalk.nvidia.com/default/topic/1031067/linux/-linux416-nvidia-390-48-nvidia_stack_cache-rip-0010-usercopy_warn-0x7e-0xa0/post/5255457/#5255457
