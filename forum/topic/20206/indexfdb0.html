redix
только нужно ли это делать - большой вопрос.
На самом деле всё просто: цель резервирования блоков (Reserved block count) предотвратить фрагментацию файлов, особенно когда свободное пространство в ФС заканчивается. По дефолту при создании это 5%. Не рекомендуется уменьшать это значение в системных разделах и тех, размер которых меньше 50 ГБ. Но его без опаски можно уменьшить / не создавать в файловых системах с большим размером или тех, где присутствуют архивные (статичные) данные, которые редко перезаписываются.
У мну давно так:
система:
# tune2fs -l /dev/sda1 | grep 'Reserved block count'
Reserved block count:     131072
хомяк:
# tune2fs -l /dev/sda2 | grep 'Reserved block count'
Reserved block count:     0
