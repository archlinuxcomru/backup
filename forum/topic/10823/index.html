После очередного обновления ядра я вновь столкнулся с той же проблемой.

Напомню:
Couldn't set tty to PPP discipline: Invalid argument

Если попробовать подгрузить модуль ppp_async, то modprobe вообще молчит, а modinfo выдает вот что:
modinfo: ERROR: Module alias ppp_async not found.
(хотя такой алиас есть, я проверил)

Решение проблемы фантастически простО:
После обновления pacman -Syu перезагрузка системы не происходит, что и является причиной проблемы. Для того, чтобы modinfo перестал тупить, достаточно перезагрузить систему.
