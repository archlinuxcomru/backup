Проблема с mount

32reg
# 7 лет, 4 месяца назад
Темы: 29
Сообщения: 526
Участник с: 17 мая 2011
Добрый день.
Подскажите с такой проблемой: есть раздел жесткого диска sda5 с NTFS (диск D от винды). В fstab он прописан так:
/dev/sda5 /mnt/disk_d ntfs-3g defaults,noexec,iocharset=utf8 0 0
В этом разделе есть папки "Мои документы/Linux" и "Install/Linux". В домашней папке есть папки "Документы" и "Install".
Для удобства, я забиндил папки из раздела в соответствующие папки в домашнем каталоге с помощью строчек в fstab:
/mnt/disk_d/Install/Linux /home/arch/Install none bind 0 0
/mnt/disk_d/Мои\040документы/Linux /home/arch/Документы none bind 0 0
Долгое время все работало как надо, но где-то с пол-года назад (раньше руки не доходили до этой проблемы) появились проблемы: в забинденные папки из домашней директории входит правильно, а если войти в папку "Жесткий диск" (в закладках слева в кедовом дельфине), то открывается папка "Install/Linux" вместо "/mnt/disk_d".
Если в fstab закомментировать строку с биндом Install/Linux, то все становится нормально: по клику "Документы" открывается "Мои документы/Linux", по клику на "Жесткий диск" открывается /mnt/disk_d.
В чем может быть проблема?
Попробовать создать mount юниты для systemd вместо этих строк в fstab?
