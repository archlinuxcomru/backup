Помоги пожалуйста. После отказа от GDM не могу регулировать яркость экрана. При использовании GDMдостаточно добавить строку 
Option "RegistryDwords" "EnableBrightnessControl=1"
в файл /etc/X11/xorg.conf. Уже поднимал эту тему http://archlinux.org.ru/forum/viewtopic.php?f=8&t=8606 все написанное там уже сделал/попробовал. Сейчас использую KDM.
При использовании ползунка регулировки яркости в настройках питания KDE ничего не происходит, хотя если загружаться с помощью GDM то все работает.

в статьях вики про самсунги встречается действие, связанное с сменой владельца файла с подсветкой:

To enable you to change the screen brightness without having to manually execute the “sudo chown”-command on every startup, you have to do the following.
First, generate a shell script named “brightnessControl” (or whatever you wish) with the following content:
[brightnessControl]
 #!/bin/bash 
 sudo chown <usrname> /sys/class/backlight/acpi_video0/brightness
тут написаны все пляски с LXDE (в приведенной выше ссылке на тему я давал эту сцылку)
