надо было еще и задержку включать

В KDE у меня так настроено. Из шпаргалки.
Смотрим темы.
plymouth-set-default-theme -l
Допустим, что выбираем тему script (при каждой смене темы необходимо пересобирать initrd).
sudo plymouth-set-default-theme -R script

Для активации плавного перехода следует заменить существующий сервис дисплейного менеджера на поставлющийся вместе с Plymouth
sudo systemctl disable sddm.service
sudo systemctl enable sddm-plymouth.service
Регулируем параметры загрузки plymouth, в том числе время задержки (здесь 8 секунд).
sudo vim /etc/plymouth/plymouthd.conf
---
[Daemon]
Theme=script
ShowDelay=0
DeviceTimeout=8

P.S. На всякий случай решил дополнить про GRUB.
sudo vim /etc/default/grub
---
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash loglevel=3 rd.udev.log_priority=3 vt.global_cursor_default=1"
0 - без курсора. 1 - мигающий подстрочный.
sudo grub-mkconfig -o /boot/grub/grub.cfg

Разумеется.

Всё по вики. Сначала подготовка к установке плимута - включить интерфейс DRM-KMS и PRIME Synchronization в проприетарном драйвере Nvidia (в моём случае). Создание соответствующего хука для пересборки mkinitcpio каждый раз после обновления видеодрайвера.

Затем установка пакета plymouth, добавление в initramfs модулей nvidia, там же включаем хук plymouth и пересобираем загрузочный образ ядра.

Уже после этого вносим изменения в GRUB, выбираем тему и прочие действия как указал в предыдущем посте.

И если в процессе Вы не допустили никаких ошибок, то всё прекрасно работает.
