Выполняя команду pptpsetup, вы создаете файл /etc/ppp/peers/freedom. Попробуйте добавить туда (подсмотрел тут):
mppe-stateful
require-mppe-128
refuse-chap
usepeerdns
defaultroute
.. именно добавить, файл не пустой; a затем уже запускать
pppd call freedom nodetach debug

UPD: c refuse-chap можно поиграться.. на самом деле у них там непонятно написано.. и попробовать добавить
mtu 1416
mru 1416

Как?
Попробовать вставить/убрать. В общем воспользуйтесь методом научного тыка. Судя по всему сервер понимает авторизацию по простому chap'у и ms-chap'у, а остальные категорически не приемлет убирайте эту строчку.
Теперь надо добавить в /etc/ppp/peers/freedom только:
mppe-stateful
require-mppe-128
Если не поможет, то добавьте еще:
noaccomp
nopcomp

Странный у вас провайдер. У них оказывается есть инструкция как настроить pppd, только ваши настройки совпадают с ихними => инструкция нерабочая. Самый действенный способ - спросить на ихнем форуме, ибо тот кто настроил сервер лучше всех знает как к нему подключится. Кстати вы можете сделать
pppd call freedom nodetach dump debug > pppd_log
, скопировать файл на флэшку и вставлять не картинки, но текст.
