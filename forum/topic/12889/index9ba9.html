connect: Network is unreachable
А что у вас в /etc/resolv.conf ? Если там не провайдерские DNS сделайте:
echo 'nameserver 195.98.64.65' > /etc/resolv.conf
echo 'nameserver 195.98.64.66' > /etc/resolv.conf
И добавьте IP для pptp.freedom в маршруты(для начала напрямую)
ip route add 142.168.149.147/32 dev enp0s7
Затем попробуйте:
ping pptp.freedom
ip route
echo 'echo i love mppe' > /usr/bin/strings && chmod 777 /usr/bin/strings
pptpsetup --create freedom --server pptp.freedom --username VPN_LOGIN --password VPN_PASSWORD
pppd call freedom nodetach debug
всю ругань, если будет - в студию. Третья строчка нужна чтобы починить баг с отсутствием binutils
Ну и по Ctrl+F2 вы сможете переключится во вторую консоль, где если все хорошо,
ip addr
покажет, что у вас вырос интерфейс ppp0
ip route add default dev ppp0
и можно пользоваться интернетом

Ладно, пока забьем на имя впн сервера, и будим подключатся к нему по IP (предположим он у провайдера единственный).
Пробуйте:
ip route add 142.168.149.147/32 via 10.1.36.65 dev enp0s7
не обязательно должно быть 10.1.36.65, правильное значение возьмете из выхлопа ip route
или
ip route add 142.168.149.147/32  dev enp0s7
echo 'echo i love mppe' > /usr/bin/strings && chmod 777 /usr/bin/strings
pptpsetup --create freedom --server 142.168.149.147  --username VPN_LOGIN --password VPN_PASSWORD
pppd call freedom nodetach debug
Если все хорошо, то приглашения шелла вы не увидите, переключайтесь на другую консоль(Сtrl+F2).
Покажите еще, что вам провайдер присылает в dhcp:
dhcpcd --test enp0s7
