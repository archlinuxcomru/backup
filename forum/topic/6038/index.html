Привет всем. До недавнего времени (точнее не уследил) всё было шёлково и шоколадно, потом случились может обновления, а может и нет…
Возможно, проблема не стоит и выеденного яйца, но:
(GNOME, система обновлялась вчера, да и вообще, регулярно), вобщем, вставляю CD-R, DVD+R с данными (фотки, тексты, папки с доками) - автомаунтится на Раб.стол как “Чистый CD-R” или чистый DVD-RW (хотя вставлял DVD+R), в наутилусе при этом появляются ДВА чистых носителя. Ладно, зайдём с другой стороны (в данном случае втыкаю в лоток CD-R с фотками, на столе появился “Чистый CD-R”, “не примонтирован”): 
# mount /dev/sr0 /media/cd
mount: /dev/sr0 уже примонтирован или /media/cd занят
при этом в /media/cd тоже нифига нет.
Хорошо, 
# umount /dev/sr0 /media/cd
umount: /dev/sr0: не примонтирован
umount: /media/cd: не примонтирован
И
# mount /dev/sr0 /media/cd
mount: блочное устройство /dev/sr0 защищен от записи, монтируется только для чтения
, лезу в /media/cd - вижу в дире фотки. Что за фигня.., на столе и в наутилусе, по-прежнему “пустой cd-r”, для проверки проделываю все шаги заново, и получаю
# mount /dev/sr0 /media/cd
mount: /dev/sr0 уже примонтирован или /media/cd занят
# umount /dev/sr0 /media/cd
umount: /dev/sr0: не примонтирован
umount: /media/cd: не примонтирован
# mount /dev/sr0 /media/cd
mount: блочное устройство /dev/sr0 защищен от записи, монтируется только для чтения
mount: /dev/sr0 уже примонтирован или /media/cd занят
…
Но, при этом другие диски, напр. DVD+R (.iso или содержащие .avi-данные) монтируются и показывают содержимое, короче, всё нормально.

Вот конфиги:
cat /etc/fstab
# 
# /etc/fstab: static file system information
#
# <file system>        <dir>         <type>    <options>          <dump> <pass>
devpts                 /dev/pts      devpts    defaults            0      0
shm                    /dev/shm      tmpfs     nodev,nosuid        0      0
#/dev/cdrom            /media/cd   auto    ro,user,noauto,unhide   0      0
#/dev/dvd               /media/dvd  auto    ro,user,noauto,unhide   0      0
/dev/sda1 /boot ext4 defaults,noatime,nodiratime  0   1
/dev/sda2 / ext4 defaults,noatime,nodiratime  0   1
/dev/sda3 swap swap defaults  0   0
/dev/sda5 /home ext4 defaults,noatime,nodiratime  0   2
/dev/sda6 /media/multimedia  ext4 defaults,rw,noatime,nodiratime  0   2

Кусок rc.conf
DAEMONS=(syslog-ng iptables network hal @netfs @crond @alsa @xl2tpd)
В /etc/udev/rules.d/ оставил дефолтным, т.к. без hal не стартует nerolinux=), но и там курочил сперва, пытаясь найти причину (делал просто по wiki) - диски-флэшки-плюшки маунтятся по правилам (без hal), но с дисками_с_данными такая-же беда.., что ещё показать? С ~/.xinitrc вроде всё в норме: 
exec ck-launch-session gnome-session
Есть у кого-нибудь мысли, как лечить? Потому что своих больше нет.

Тоже недавно заметил подобную ерунду. Проблема похоже именно в Гноме. Ни nautilus, ни pcmanfm такие диски нормально не монтирует. Добавление правила (вики) проблемы не решает, наоборот - при выборе в меню “извлечь”, лоток открывается и сразуже закрывается, диск можно достать только нажав кнопку на самом приводе. Файловая система на диске - cdfs (вроде бы), диск - DVD+R, содержимое - фотки, пара фильмов и папка с мультиками. Других дисков для проверки, к сожалению не нашел.
Что характерно, при вставке диска и без ручного примонтирования, в каталоге /dev/disk/by-label создается ссылка на блочное устройство (/../sr1).

Я тоже решил, что это очередная гномофича. Потому что, при “отсоединении” через наутилус, диск затем нормально монтируется # mount /dev/sr0 /media/cd|dvd и его содержимое открывается, на рабочем столе даже появляется ещё один значок чистого диска, который открывается по клику и кажет содержимое, (при этом висит и “другой” диск, на самом деле “пустой”).
зы. с удавом такая-же фигня с лотком, ты всё совершенно точно описал, потому я удав и не использую пока для этого, с ним дисковод вообще невменяемый.
ЗЫ. Откат udev 161-0 > 161.1 решил проблему, временно, само-собой. Дело похоже, не в гноме. Зря обидели.
