Лаунчер не стимовский, скачан напрямую с сайта WT

Как я понял, подобную проблему тут разобрали, цитата:

This actually seems to be a bug with the device select layer as it tries to call vkGetPhysicalDeviceProperties2 on a Vulkan 1.0 device with a Vulkan 1.1 instance. This is actually exactly the same problem that DXVK used to have. I don't normally have the layer installed so that would explain why it's working fine here.

Собственно, War Thunder я запускаю с помощью ярлыка с рабочего стола (символической ссылки), там в свойствах прописал
VK_ICD_FILENAMES=/usr/share/vulkan/icd.d/radeon_icd.i686.json:/usr/share/vulkan/icd.d/radeon_icd.x86_64.json *путь до лаунчера*
и нормально запускается. Может с каким-нибудь последующим обновлениях где-нибудь поправят, пока пусть будет так
