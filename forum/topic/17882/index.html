Привет. Подскажите пожалуйста пару моментов по настройке сети.

Есть ПК с 2 Lan(встроенная enp4s0 и PCI enp3s0), на enp4s0 - внешняя сеть, DHCP, на enp3s0 - внутренняя, статика
Сеть поднимал через systemd-networkd
/etc/systemd/network/enp3s0-lan.network
[Match]
Name=enp3s0

[Network]
192.168.1.1/24
Gateway=192.168.1.1

/etc/systemd/network/enp4s0-inet.network
[Match]
Name=enp4s0

[Network]
DHCP=ipv4

systemctl start/enable systemd-networkd systemd-resolved

Вопрос 1: при настройке через netctl можно было указать параметр "SkipNoCarrier=yes" для поднятия соединение без подключенного линка, как указать подобный параметр при настройке через systemd-networkd?

На интерфейсе enp3s0 необходимо поднять DHCP-сервер. Установил dnsmasq.
/etc/dnsmasq.conf
interface=enp3s0
bind-interfaces
dhcp-range=192.168.1.2,192.168.1.10,12h

systemctl start/enable dnsmasq

Вопрос 2: Без dnsmasq, при подключении кросс-кабелем и присвоении вручную на другом ПК ip 192.168.2.2 - машины друг друга пингуют, при старте dnsmasq - вторая машина не получает по DHCP ip адрес. Раньше я поднимал сеть через netctl - все работало.

Вопрос 3: Если мне нужно через интерфейс enp3s0 видеть ПК не только по ip, но и по имени - в какую сторону копать?(resolv.conf?)

UPD:
По вопросу 2: Решил, невнимательно настраивал dnsmasq.conf, неправильно указал опцию tftp-root.
