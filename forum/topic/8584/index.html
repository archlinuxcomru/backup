Хочу поделиться опытом настройки торрент-клиента RTorrent.

История такая: имеется в наличии ADSL-модем с маршрутизатором (роутером) и портом Ethernet. Динамичный внешний ip. Специально для p2p сетей в настройках роутера выключен DHCP, для компьютера выделен статичный ip, и для каждого протокола настроены определённые порты. В частности, для торрентов настроен порт 6881 с разрешёнными пакетами TCP и UDP. Раньше использовал Deluge и проблем с настройками портов в самом торрент-клиенте не возникало (вбил порт в настройки и забыл). Но, установив rtorrent, столкнулся с проблемой - куда нужно прописать порт 6881?

Для того, чтобы RTorrent начал использовать настроенный в роутере порт (для моего случая 6881), необходимо в настройках (~/.rtorrent.rc) прописать следующие строчки:

port_range = 6881-6881
port_random = no


Также, для увеличения количества участников (пиров), рекомендую добавить в настройки строчку (разрешает шифрование запросов):

encryption = allow_incoming,try_outgoing,enable_retry


Ещё интересные опции - это минимальное и максимальное количество подключаемых пиров. Не знаю, по какому критерию программа выбирает минимальное количество пиров, но для того, чтобы rtorrent задействовал все доступные источники, необходимо минимальное количество пиров приравнять к максимальному. В моём случае, например, максимальное количество пиров равно 2000, тогда:

min_peers = 2000
max_peers = 2000


p.s. Уж извините, что по такому мелкому поводу создаю тему, но сам мучался, когда искал информацию… Может кому будет полезно.

Штука. Слушай, а в rtorrent есть возможность указать сколько одновременной активных торрентов может быть? причем лучше если разные параметры для скачивания и раздачи.

Количество одновременных раздач можно определить параметроми (настройки по умолчанию):
min_peers_seed = 1 # минимальное количество участников, которым будет раздаваться торрент
max_peers_seed = 50 # максимальное количество участников, которым будет раздаваться торрент
max_uploads = 15 # максимальное количество раздач

Возможно, имеет смысл увеличить значение параметра min_peers_seed (как в случае с min_peers). Я ещё не проводил эксперименты с параметрами раздачи, с меня и при таких настройках хорошо тянут торренты =)

Настройки на максимальное количество активных закачек у rtorrent нет. Но можно написать скрипт на баше, который будет управлять закачками rtorrent. В этом скрипте можно указать максимальное количество одновременных закачек. Я этот скрипт менеджера закачек не использовал, поэтому не могу ничего о нём сказать.
Ссылка (1): http://flexget.com/ticket/167
Ссылка (2): http://forum.synology.com/enu/viewtopic.php?f=38&t=8992
