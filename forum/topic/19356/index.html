Приветствую, форумчане. Заранее извиняюсь за мою "некомпетентность", ибо являюсь новичком :)

Появилась необходимость настроить прокси на моём arch linux (xfce), решил сделать это утилитой proxychains-ng, так как ее везде советуют. Скачал этот пакет из блекарчевского репозитория, зашел в /etc/Proxychains.conf , установил режим dynamic_chain путём его раскомментирования, закомментировал изначальный режим - strict_chain. Нашел рабочий прокси в инете (Американский, Socks4), проверил его ping"ом, записал его в proxylist (внизу кинул), сохранил файл. Запустил firefox командой proxychains firefox, зашел на 2ip, там был мой старый ip(( Решил проверить это все дело командой proxychains wget -qO- eth0.me, и что вы думаете, команда показала правильный и рабочий прокси.. Но если вписать эту команду без proxychains, то показывается мой старый ip... Подскажите, если не сложно, в каком направлении искать.. Перерыл несколько форумов, но так ничего на эту тему и не нашел.
Proxychains.conf::

# proxychains.conf VER 4.x
#
# HTTP, SOCKS4a, SOCKS5 tunneling proxifier with DNS.
# The option below identifies how the ProxyList is treated.
# only one option should be uncommented at time,
# otherwise the last appearing option will be accepted
#
dynamic_chain
#
# Dynamic - Each connection will be done via chained proxies
# all proxies chained in the order as they appear in the list
# at least one proxy must be online to play in chain
# (dead proxies are skipped)
# otherwise EINTR is returned to the app
#
#strict_chain
#
# Strict - Each connection will be done via chained proxies
# all proxies chained in the order as they appear in the list
# all proxies must be online to play in chain
# otherwise EINTR is returned to the app
#
____________________________________________________________________________________________________________________________________________________
