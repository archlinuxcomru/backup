С флешки арч не устанавливал, однако, пришлось делать netinstall системы. Провайдер у меня - корбино-билайн, в интернет эти парни пускают через pptp-vpn. Адрес Ethernet-интерфейсу назначается динамически, через DHCP. Конечно, не от хорошей жизни пришлось мне устанавливать систему таким образом, сугубо по-причине того, что я оказался одним из тех “счастливчиков”, у которых диск, записанный из ISO-образа отказался загружать инсталл, зависнув на грубе. Пришлось, соответственно, нарезать образ isolinux, а там был только netinstall. Что ж, делать нечего. Делаем нетинсталл. На удивление, с арчем никаких проблем с этим не оказалось! К примеру, опенсусю 11.2 с её нетинсталл-образа установить в моей ситуации оказалось неможно.
Делал так:
1. Загрузился с диска с ISOLINUX-netinstall.
2. Загрузил установщик.
3. Запустил скрипт #aif -p partial-configure-network
4. В ответ на вопросы скрипта ввёл имя интерфейса eth0. Скрипт сообщил о том, что сеть настроена.
5. Проверил, что настройки сети от DHCP получены. #cat /etc/resolv.conf - вывод команды показал, что адреса DNS-серверов получены. #route -n - смотрим таблицу маршрутизации для определения шлюза по-умолчанию.
6. Проверяем, пингуются ли DNS-сервера, если нет, добавляем шлюз по-умолчанию вручную
#route add default gw ваш_шлюз
Снова проверяем, пингуются ли сервера DNS. Если шлюз указали верно, пинговаться должны.
7. Создаём временное соединение (на время установки)
# vi /etc/ppp/peers/provider
часть опций, которые пишутся в этот файл, провайдерозависимы. Я файл /etc/ppp/pptp.options не использовал. В моём случае, подходит такое содержание файла:
noauth
nodeflate
nobsdcomp
refuse-pap
refuse-mschap
refuse-mschap-v2
refuse-eap
persist
pty "pptp адрес_вашего_впн_сервера –nolaunchppd"
user имя_пользователя
password "пароль"

Всё! Сохраняем файл.
Проверяем, чего наваяли:
# pppd call provider debug nodetach
На экран будет выведен лог подключения к провайдеру. В случае, если подключение удалось, будут получены адреса тоннеля (свой и удалённый), код завершения скрипта подключения будет нулевым.
Если чего не так - смотрим по логу, что не так, правим.
Если всё получилось, с другой консоли (если с консоли № 2 - нажать Alt+F2 и залогиниться root) сделать
#route del default (удаляем старый шлюз по-умолчанию, который привязан к адресу сетевой карты)
#route add default dev ppp0 (отправляем весь трафик через тоннель)
#ping ya.ru (проверяем, всё ли получилось - должны идти ответы от узла)
# killall pppd
перейти на первую консоль, выбрать повтор команды, (курсорная стрелка вверх), сделать
#route del default (удаляем отправку трафика через несуществующий уже ppp0)
#route add default gw ваш_шлюз (снова прописываем шлюз к днс-серверам провайдера и даём возможность разрешить имя сервера vpn, если вводите его как vpn.internet.corbina.ru, а не как ip-aдрес)
# pppd call provider
#route del default (удаляем старый шлюз по-умолчанию, который привязан к адресу сетевой карты)
#route add default dev ppp0 (отправляем весь трафик через тоннель)
В общем, можно было бы и не прибивать pppd, пусть бы он висел в состоянии debug, а установку можно было бы продолжить с другой консоли.
Далее - как написано в ЧаВо - /arch/setup
Далее - по всем пунктам… После установки системы и перезагрузки можно уже делать всё основательно.
