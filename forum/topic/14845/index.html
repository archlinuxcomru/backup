pacman -S xorg - графический сервер, как я понял

pacman -S slim - logon-скрин, менеджер авторизации (графический), экран входа в систему

Что осложняет ситуацию: осложняет ее наличие 2 видюх в ноуте:

lspci | grep -e VGA: VGA compatible controller Intel corp 3rd Gen Core processor graphic Controller (rev 09)
lspci | grep -e 3D: 3D controller Nvidia corp GF117M [много всякой писанины] - это видеокарта NVIDIA GT 620M

Как быть в этой ситуации?

Следует установить драйвер для первой и второй карты, как я предполагаю. Случайным образом нашел пометку:

Note: If you have a NVIDIA Optimus enabled laptop which uses an integrated video card combined with a dedicated GPU, see Bumblebee.
https://wiki.archlinux.org/index.php/Bumblebee

Installing Bumblebee with Intel/NVIDIA

Install:

bumblebee - The main package providing the daemon and client programs.
mesa - An open-source implementation of the OpenGL specification.
xf86-video-intel - Intel driver.
nvidia - NVIDIA driver.

For 32-bit (Multilib must be enabled) applications support on 64-bit machines, install:

lib32-nvidia-utils
lib32-mesa-libgl

поправьте, если где ошибся ..
pacman -S xorg
Не так чтобы ошиблись, но ставить всю группу xorg абсолютно не обязательно. В ее состав входят утилиты, которыми вы в жизни можете не пользоваться.
Для работы иксов достаточно установить:
xorg-server (xf86-input-evdev, xorg-fonts-misc и др. подтянется автоматически в качестве зависимостей);
видео-драйвер;
xorg-xinit.

Все остальное по необходимости всегда можно доставить.
