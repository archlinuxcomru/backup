Вопросы:
1. В выводе команды “systemctl -a” высветилось красное:
auditd.service            error  inactive dead          auditd.service
rc-local.service          error  inactive dead          rc-local.service
Мои варианты действий:
1а. Все пропало, у арча нет будущего, ухожу на винду.
1б. Ничего страшного, нехай будэ, от этих красных строк ни жарко ни холодно.
К врачу-урологу тихонько постучались в дверь кабинета. На приглашение
входит такой аккуратненький мужичок. Естественно док спрашивает на что
пациент жалуется.
Мужичок аккуратненько снимает одежку, аккуратненько складывает ее на
спинку стула и показывает доктору хозяйство.

левого…
Д: - Ну и…?

В случае ответа 1в как можно убрать обращения к этим службам (если они не нужны) или как их запустить (если жизненно необходимы)?

2. Since boot these days tends to be clearly IO bound (and not CPU bound) it might make sense to reduce the IO load for the pack file by compressing it. Since we already have a dependency on XZ we'd recommend using XZ for this.”). Где эти файлы найти? Сэкономить десятую долю секунды при загрузке очень важно для меня :)

privoxy.service вариант более правильный но не совершенный.
[Unit]
Description=Privoxy Proxy Server
After=syslog.target network.target
[Service]
Type=forking
PIDFile=/var/run/privoxy.pid
EnvironmentFile=/etc/conf.d/privoxy
ExecStart=/usr/sbin/privoxy --pidfile /var/run/privoxy.pid --user ${PRIVOXY_USER}.${PRIVOXY_GROUP} ${PRIVOXY_CONF}
#ExecStop=-/bin/kill $MAINPID
[Install]
WantedBy=multi-user.target
