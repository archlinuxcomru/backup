Чтобы груб понимал ФС, у него должен быть модуль поддержки этой ФС.
Если он "unknown filesystem", значит или у него нет этого модуля, или груб не может его прочитать из-за неправильной/поломанной установки.
В каталоге /boot/grub/i386-pc/ (или /boot/grub/ ) должно быть много файлов *.mod , среди них драйверы ФС с узнаваемыми именами.

Если у тебя особо ничего не загублено и ты можешь зайти в консоль grub — попробуй самый простой способ. Мне приходилось так заходить, когда под рукой не было никаких инструментов.
1. зашел в консоль (PS...имел ввиду консоль grub)
ls — список доступных разделов — например, твой будет (hd0,msdos3) (PS...соответствует /dev/sda3)
2. задаем раздел, в который нужно перейти
set root=(hd0,msdos3)
3. указываем ядро (для случая отсутствия раздела boot)
linux /boot/vmlinuz-linux root=/dev/sda3
(подсказка - если не знаешь какое ядро, например в бубунте их навалом, после набора linux /boot/vmlinuz нажми tab — покажет все ядра)
4. указываем initrd (пишу для Arch)
initrd /boot/initramfs-linux.img
(для бубунты будет что-то типа /boot/initrd.img......вообщем можешь опять воспользоваться tab)
5. стартуем
boot
Все — пошла загрузка, если ничего серъезного не наворочал.
PS............... редко использую этот метод по одной причине - под рукой, как назло, не оказывается бумажки с нужными командами, а память уже плохая - быстрее вставить флешку.
