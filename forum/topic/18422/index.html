Содержание ~/.ssh/config

Host *
HostkeyAlgorithms +ssh-dss
IdentityFile = ~/.ssh/id_rsa

Пробовал перегенерить публичный таким образом:

ssh-keygen -f ~/.ssh/id_rsa -y > ~/.ssh/id_rsa.pub

получил такую ошибку:

Load key "/home/edward/.ssh/id_rsa": invalid format

Из всего этого вижу, что проблема у меня на хосте. Недавно снес предидущую систему, конфиги и ключи конечно перенес со старой и началась такая ерунда.

Помогите разобраться, кто сталкивался или видит в чем причина.

Добавлю, приватный ключ в таком формате:

-----BEGIN RSA PRIVATE KEY-----
Proc-Type: 4,ENCRYPTED
DEK-Info: AES-128-CBC, 32 знака

Сам ключ, 64 знака в строке, 25 строк
-----END RSA PRIVATE KEY-----

Публичный:
ssh-rsa key comment

Ключ расшифровывается, судя по тому, что при вводе неверного пароля, например:

ssh-add ~/.ssh/id_rsa
Enter passphrase for /home/edward/.ssh/id_rsa:
Bad passphrase, try again for /home/edward/.ssh/id_rsa:

но при указании верного пароля, опять же:

Error loading key "/home/edward/.ssh/id_rsa": invalid format
