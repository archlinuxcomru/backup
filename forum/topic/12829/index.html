Давно не требовался, и вот потребовалось. Ранее под 32б Бубунтой он был, так его директорий с собой на Арч и перенес. Однако по-старинке не запускается, пишет об ошибке -
ERROR - ...Flashtool/x10flasher_lib/adb.linux: error while loading shared libraries: libncurses.so.5: cannot open shared object file: No such file or directory

. Скачанный с XDA также. Также и с adb - вылетает с той же ошибкой, что и Flashtool -

./adb: error while loading shared libraries: libncurses.so.5: cannot open shared object file: No such file or directory

Спросишь систему, она говорит, что libncurses.so.5 есть (не сама отдельно, а включающий ее пакет).
Скачаешь новый флэштул с http://www.flashtool.net/index.php, так при запуске выдает ошибку

libusb.LibUsbException: Libusb not found. Minimum libusb version is 1.0.15.
В Ауре есть adb, скачал, работает, уже легче, на libncurses.so.5 не жалуется. А вот поиск flashtool выдает только -

yaourt -Ss flashtool
aur/rkflashtool-git 20130227-1 (2)
A tool for flashing rk2808, rk2818, and rk2918 based tablets
aur/usbprog 0.2.0-2 (14)
Flashtool for the USBprog Programming Adapter

т.е. флэштул для рокчиповских процессоров, а у меня квалком.

Кто что подскажет? - а то стыдно подкрадываться к ноутбуку жены с 32р. бубунтой (она напрочь отказывалась мигрировать под Arch и теперь ухмыляется).

Частично решил таким образом (мыслить вслух все же полезно) - поставил из Ауры adb

yaourt -S adb-git

поставленный таким образом adb скопировал в директорий flashtool -

cp /bin/adb место/Flashtool/x10flasher_lib/adb.linux

и оно заработало, т.е. заработал старый флэштул, а чтобы он видел новые телефоны, в Flashtool/devices скопировал от нового флэштула нужные девайсы. Но при подключении девайса флэштул 0.9.9.0 попросту вылетает.
