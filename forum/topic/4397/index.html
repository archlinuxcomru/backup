Pulse+mic

sandsnake
# 11 лет, 3 месяца назад
Темы: 3
Сообщения: 13
Участник с: 05 июня 2011
Есть две отдельные машинки с разным набором харда но схожими проблемами.
На обоих стоит Arch и 3-м гномом. Соответственно, pulseaudio. И на обоих не могу заставить работать микрофоны с гарнитуры.
на первой:
$ lspci  | grep -i audio
00:1b.0 Audio device: Intel Corporation N10/ICH 7 Family High Definition Audio Controller (rev 01)
На второй:
lspci | grep -i audio
02:00.1 Audio device: nVidia Corporation GF104 High Definition Audio Controller (rev a1)
03:00.0 Audio device: Creative Labs X-Fi Titanium series [EMU20k2] (rev 03)
На второй же есть еще и вебкамера с микрофоном. Вот он-то, зараза, работает “из коробки” и вопросов с ним не возникает.

Поскольку беда на обоих, то думаю что это настройки самого пульса.

В настройках звука на закладке “вход” светится два устройства:
PCVC675K Webcam Analog Mono
X-Fi Titanium series Analog Stereo

Когда выбираю камеру, то ее микрофон начинает ловить звук что и отображается на индикаторе уровня записи. При выборе X-Fi индикатор не показывает ничего абсолютно.
В закладке оборудования, при выборе X-Fi есть выпадающий список с профилями устройства. Перебрал, безрезультатно.

/etc/asound.conf
# Use PulseAudio by default
pcm.!default {
  type pulse
}
ctl.!default {
  type pulse
}
# Explicit PulseAudio device
pcm.pulse {
  type pulse
}
ctl.pulse {
  type pulse
}
# vim:set ft=alsaconf:
В /etc/pulse/daemon.conf раскоментарена одна стрка:
resample-method=speex-float-0
и то не мной.

Гуглил. много подобных бед но не помогло, а скорее запутало. Из-за этого и прошу помощи сообщества.
