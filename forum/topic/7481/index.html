Доброе время суток, господа.
Занимаюсь web-разработкой, меньше месяца назад мигрировал на Arch. Естественно потребовался LAMP, который был успешно поставлен по мануалу LAMP и его английской версии LAMP. Всё блестяще работало, но в определенный момент потребовался Zend Optimizer, который на настоящий момент не имеет поддержки PHP 5.3, как выяснилось на официальном форуме. Встала задача откатиться на PHP 5.2.* Пробовал разные способы, наломал дров в итоге: apache ставится pacman'ом и работает. PHP 5.2 из AUR'а ставится, но не работает. О нем позже.
MySQL ставится pacman'ом и не работает.
То есть:
# /etc/rc.d/mysqld start
:: Starting MySQL
Причем полминутки думает (BUSY). После этого на повторный старт выдает FAIL не задумываясь, а на стоп говорит DONE.
На обращение к mysql, bash говорит команда не найдена.
Причем появляется после “долгого” старта с FAIL в конце появляется два процесса
21839 pts/1 S 0:00 /bin/sh /usr/bin/mysqld_safe –datadir=/var/lib/mysql
21940 pts/1 Sl 0:00 /usr/sbin/mysqld –basedir=/usr –datadir=/var/lib/mysql –user=mysql –log-error=/var/lib/mysql/vaiolaptop.err –pid-file=/var/lib/mysql/vaiolaptop.pid –socket=/tmp/mysql.sock –port=3306

В /etc/my.cnf добавил строчку про лог:

port = 3306
socket = /tmp/mysql.sock
datadir = /var/lib/mysql
log = /var/log/mysql.log
skip-locking
key_buffer = 16M
max_allowed_packet = 1M
table_cache = 64
sort_buffer_size = 512K
net_buffer_length = 16K
myisam_sort_buffer_size = 8M
Создал mysql.log ручками, дал права 666. После фейла старта mysql.log выглядит так:
/usr/sbin/mysqld, Version: 5.1.39-log (Source distribution). started with:
Tcp port: 3306 Unix socket: /tmp/mysql.sock
Time Id Command Argument
Вот очищенный log-error сразу после долгого старта с фейлом, на который ссылается загруженный процесс:
# cat /var/lib/mysql/vaiolaptop.err
091117 01:57:01 mysqld_safe Starting mysqld daemon with databases from /var/lib/mysql
091117 1:57:01 The syntax ‘–log’ is deprecated and will be removed in MySQL 7.0. Please use ‘–general_log’/'–general_log_file' instead.
091117 1:57:01 Plugin ‘FEDERATED’ is disabled.
091117 1:57:01 Plugin ‘ndbcluster’ is disabled.
091117 1:57:01 InnoDB: Started; log sequence number 0 44233
091117 1:57:01 Event Scheduler: Loaded 0 events
091117 1:57:01 /usr/sbin/mysqld: ready for connections.
Version: ‘5.1.39-log’ socket: ‘/tmp/mysql.sock’ port: 3306 Source distribution
откуда следует что mysqld запущен, но по прежнему
# mysql
bash: mysql: команда не найдена

В первой строчке всё нормально исключая вопрос почему собственно mysqld_safe
Во второй mysql ругается на устарелый параметр –log, но где он его получает неизвестно. От этого всё ломаться не должно.
Третья и четвертые замечают что некие плагины FEDERATED и ndbcluster выключены. Их назначение и необходимость для работы сервера мне неизвестны.
Следующие тстрочки вроде бы говорят что всё хорошо, хотя настораживает, что дело происходит в .err логе :)
мой вывод - mysql стартует, но в безопасном режиме.
Как с ним работать и в чем причина FAIL'а не понимаю.
Куда копать, посоветуйте знатоки :)
