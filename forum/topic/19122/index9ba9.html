Т.е я должен буду написать PhysicalDrive1?

Нет. Вам давали ссылку на ВИКИ, где именно про dd Виндовс говорится:
vs220
https://wiki.archlinux.org/index.php/USB_flash_installation_media

Там и пример есть:
Simply replace the various null spots (indicated by an "x") with the correct date and correct drive letter. Here is a complete example.
# dd if=ISOs\archlinux-version-x86_64.iso od=\\.\d: bs=4M

Только сначала:
dd for Windows
A GPL licensed dd version for Windows is available at http://www.chrysocome.net/dd. The advantage of this over Cygwin is a smaller download. Use it as shown in instructions for Cygwin above.
To begin, download the latest version of dd for Windows. Once downloaded, extract the archive's contents into Downloads or elsewhere.

Гугл-перевод:
Лицензионная версия dd для Windows доступна по адресу http://www.chrysocome.net/dd. Преимущество этого по сравнению с Cygwin заключается в меньшей загрузке. Используйте его, как показано в инструкциях для Cygwin выше.
Для начала загрузите последнюю версию dd для Windows. После загрузки извлеките содержимое архива в раздел «Загрузки» или в другое место.

vasek
Образ archiso isohybrid-ный …… и grub4dos его не загрузишь …
vasek, зачем дурить маленьких, он и так на вторые сутки поисков пошел :)
Все, что нужно знать : метку/идентификатор образа и флешки. Я предпочитаю метку. Для установочного образа арча метка ARCH_201903, обзову файл образа archlinux.iso и закину в корень флешки, которая имеет метку ANACONDA, пока ни чего, что не возможно узнать в Вин7. Тогда конфиг для grub4dos будет примерно таким:
title "Arch Linux Install ISO"
find --set-root --ignore-floppies /archlinux.iso
map /archlinux.iso (hd32)
map --hook
root (hd32)
kernel /arch/boot/x86_64/vmlinuz archisolabel=ARCH_201903 img_dev=/dev/disk/by-label/ANACONDA img_loop=archlinux.iso earlymodules=loop archisodevice=/dev/loop0
initrd /arch/boot/x86_64/archiso.img
Профит.
ЗЫ. На флешке НТФС( 0х07), но я думаю, что это только ухудшает ситуацию, т.е. с ФАТ - вообще без проблем.
