Привет!

Вот конфигурация сети:


    [телевизор sumsung] - подключен к сети по кабелю       [server] - получает сеть по кабелю, есть wifi-модуль
                           ^                                           ^
                           |                                          /
                           v                                         v
{Интернет}  <-> [router Alcatel-Lucent - раздает сеть по кабелю и wifi]
                               ^                      ^
                               |                       \
                               v                        v
                            [смартфон 1]              [смартфон 2]  получают сеть через wifi

Задача: раздать медиа-контент на телевизор и смартфоны

Медиа-контент - аудио, видео и фото, все различного формата.
Выбор пал на сервер minidlna - самсунговский allshare c ним работает вполне уверено.
Однако тут же наступил на грабли - если server подключен к сети по кабелю, то медиаконтент видит только телевизор sumsung, если сервер подключить к сети через wifi, то медиаконтент видят оба смартфона, однако телевизор уже не видит.

Как разрулить данную ситуацию?

сервер - статичный адрес 192.168.1.2 (роутер раздает соответственно с адреса 192.168.1.3 и дальше) маска 255.255.255.0
телевизор и прочие девайсы получают адрес по dhcp : телек в настоящее время 192.168.1.7 маска 255.255.255.0
смартфон1 192.168.1.3, маска 255.255.255.0
смартфон2 192.168.1.6, маска 255.255.255.0

default gateway и сервер dns - везде 192.168.1.1

постоянные маршруты нигде не прописывал
