Не могу установить принтер Minolta PagePro 1200W
На всякий случай, заранее говорю, что в /etc/cups/cupsd.conf
# Administrator user group...
SystemGroup sys root lp
Юзер в группе lp
Принтер видится в системе.
# lsusb
Bus 003 Device 004: ID 0686:2004 Minolta Co., Ltd PagePro 1200W
Драйвер есть. Поставил из AUR без ошибок.
yaourt -Ss min12xxw
PPD скачал c OpenPrinting
Права на запись в устройство есть.
# ls -l /dev/bus/usb/003/004
crw-rw-r-- 1 root lp 189, 259 мая   15 08:50 /dev/bus/usb/003/004
Запускаю браузер. Вхожу на вебинтерфейс CUPS: там список:
 Backend Error Handler 
 Протокол интернет-печати (ipp) 
 Хост или принтер LPD/LPR 
 Протокол интернет-печати (https) 
 Протокол интернет-печати (ipps) 
 Протокол интернет-печати (http) 
 Windows Printer via SAMBA 
 AppSocket/HP JetDirect
Т.е. тут моего принтера нет. CUPS его не видит. Это проблема.
При попытке указать в качестве URI: http://hostname:631/ipp/ задание зависает со статусом Processing пока не удалишь задание.
В /var/log/lpr.log
May 15 08:50:17 localhost udev-configure-printer: device devpath is /devices/pci0000:00/0000:00:04.0/usb3/3-3
May 15 08:50:17 localhost udev-configure-printer: MFG:MINOLTA-QMS MDL:PagePro 1200W SERN:- serial:-
May 15 08:50:18 localhost udev-configure-printer: no corresponding CUPS device found

Пытался блеклистить модуль: 
# cat /etc/modprobe.d/usblp.conf
blacklist usblp
Все по прежнему. Ума не приложу, что делать дальше.
Куда шагать дальше?

Спасибо. Вопрос решился.
$ cat /etc/udev/rules.d/10-usbprinter.rules
ATTR{idVendor}=="0686", ATTR{idProduct}=="2004", MODE:="0666", GROUP:="lp"
$ cat /etc/modprobe.d/usblp.conf 
blacklist usblp
