Что можно придумать?
Начни с проверки, ручками ….... действительно ли нет файла /usr/lib/libpsl.so.5
Если файл действительно имеется, копай почему он не подхватывается ....
Также проверь
$ ldd /usr/bin/wget | grep libpsl
	libpsl.so.5 => /usr/lib/libpsl.so.5 (0xb73d8000)
$ ldd /usr/bin/curl | grep libpsl
	libpsl.so.5 => /usr/lib/libpsl.so.5 (0xb74dd000)
$ ldd /usr/bin/psl | grep libpsl
	libpsl.so.5 => /usr/lib/libpsl.so.5 (0xb776d000)

UPD …...
$ pacman -Qo /usr/lib/libpsl.so.5
/usr/lib/libpsl.so.5 принадлежит libpsl 0.16.1-1
$ pacman -Ss libpsl
core/libpsl 0.16.1-1 [установлен]
$ pacman -Qi libpsl
….......................
Требуется            : curl  wget
….......................

PS .... можешь проверить открытие/чтение файла /usr/lib/libpsl.so.5, например, так
$ strace -e trace=open wget http://......
Ошибки не исчезают с опытом - они просто умнеют
