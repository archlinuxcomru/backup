Mysql configuration

Так как в чистом баше команды вида "CREATE DATABASE postfix;" вызывали ошибку, то подумал что их нужно отдавать на съедение Mysqld'у.
попробовал запустить mysqld

[[email protected] ~]# systemctl start mysqld

команда просто подвисла и не отпускала консоль, пришлось прибивать по Ctrl+C.
Так как в данной вики не было подробностей про mysql, то я пришел к мысли что статья расчитана на экспертов по установке и первично настройке mysql.
По этому я решил почитать вики по mysqld:
https://wiki.archlinux.org/index.php/MySQL
Тут все закончилось еще быстрее - команда mysql_secure_installation выдала:
[[email protected] ~]# mysql_secure_installation

password for the root user. If you've just installed MySQL, and
you haven't set the root password yet, the password will be blank,
so you should just press enter here.

Enter current password for root (enter for none):
ERROR 2002 (HY000): Can't connect to local MySQL server through socket '/run/mysqld/mysqld.sock' (2)
Enter current password for root (enter for none):
Aborting!

Cleaning up...
[[email protected] ~]#
В независимости от того, правильный я пароль рута вводил или нет.
Кстати следующей строчкой в вики идет: Then restart MySQL (mysqld)., что означает что либо нужно было ЗАПУСТИТЬ демон mysqld, но об этом в вики не было сказано ни слова, либо сам скрипт mysql_secure_installation должен был запустить демон mysqld.

При дефолтном (из пакета) конфиге и дефолтном же каталоге с базой юнит mysqld должен запускаться успешно.
При запущенном сервере БД команда mysql_secure_installation должна спросить у вас пароль от пользователя root В БАЗЕ (а не в системе!), который у вас пока не задан, а значит пустой, после ввода пустой строки вам будет предложено задать пароль для рута и т.д.

Более того – прямо при установке пакетов mysql вы должны были получить в консоли ма-аленькую простынку, где чистым английским языком изложено примерно то же самое.
