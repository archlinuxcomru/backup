Дефолтный шрифт кириллицу не всегда и везде поддерживает.

Примеры для поддержки:

URxvt.font:  -misc-fixed-*-*-*-*-15-*-*-*-*-*-koi8-*

URxvt.font:  xft:xos4 Terminus:style=Medium:size=10:antialias=false:hinting=true

Так, а где там настройки шрифтов и не только для xterm? )) Пример для xterm:

*faceName:  xft:xos4 Terminus:style=Medium:size=10:antialias=false:hinting=true

Кроме того, если Вы используете LightDM, то еще нужен ~/.xprofile следующего содержания:

[[ -f ~/.Xresources ]] && xrdb -merge -I$HOME ~/.Xresources
...как минимум.

Warning: locale not supported by Xlib, locale set to C
Вот что пишут про это сообщение:
Если какая-нибудь из команд, приведенных ниже, завершится неудачно и выдаст подобное сообщение, то это значит, что либо ваша локаль не была установлена , либо ваша локаль не поддерживается в Glibc при установке по умолчанию.
$ locale language
Russian
$ locale charmap
UTF-8
$ locale int_curr_symbol
RUB
 $ locale int_prefix
7
Если это случится, вы должны либо с помощью команды localedef (или другим способом) установить требуемую локаль, либо выбрать другую локаль.
Попытался добится, чтобы в Xterm не вводились русские символы — получилось:
запустил Xterm с французской локалью
$ LANG="LANG=fr_FR.iso-8859-15" xterm
Warning: locale not supported by C library, locale unchanged
Fontconfig warning: ignoring LANG=fr_FR.iso-8859-15: not a valid language tag
Xterm запустился (после ругани), en вводится,а ru нет.
Ошибки не исчезают с опытом - они просто умнеют
