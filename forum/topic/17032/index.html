Здравствуйте, в общем проблема в том что ранее все работало без проблем. Но не заметил как, перестал работать openvpn в качестве демона.
РАботает только если запустить руками. А через systemctl нет. Не могу понять что послужило причиной и как бороться. Нигде ничего не менял.

В общем если руками запустить:
sudo openvpn blacksrv.conf

Все работает замечательно. Но если

archhome /etc/openvpn $ systemctl start openvpn@blacksrv.conf
Failed to start openvpn@blacksrv.conf.service: Unit openvpn@blacksrv.conf.service not found.
archhome /etc/openvpn $ systemctl start openvpn@blacksrv
Failed to start openvpn@blacksrv.service: Unit openvpn@blacksrv.service not found.

Вот так правильно вроде

See "systemctl status openvpn-client@blacksrv.service" and "journalctl -xe" for details.

1. Перенес все из папки /etc/openvpn в /etc/openvpn/client
2. удалил не рабочие юниты openvpn
3. sudo systemctl start openvpn-client@blacksrv.service
4. sudo systemctl enable openvpn-client@blacksrv.service
