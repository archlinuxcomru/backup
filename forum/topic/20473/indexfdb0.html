Не ставьте бездумно кучу всего, разберитесь сначала с одним.
sudo systemctl disable dhcpcd.service
sudo pacman -R openresolv

Прописать в /etc/NetworkManager/conf.d/no-systemd-resolved.conf
[main]
systemd-resolved=false
https://wiki.archlinux.org/index.php/NetworkManager#Unit_dbus-org.freedesktop.resolve1.service_not_found

Проверить что не прописан сторонний днс клиент в
/etc/NetworkManager/conf.d/dhcp-client.conf или /etc/NetworkManager/conf.d/dns.conf
если файл есть удалить его для использования встроенного днс клиента

Проверить что не прописаны адреса в /etc/NetworkManager/conf.d/dns-servers.conf

Проверить что /etc/resolv.conf не является символической ссылкой
 file /etc/resolv.conf
если да удалить ее и создать файл заново
sudo rm /etc/resolv.conf
sudo touch /etc/resolv.conf

Проверить что не активны еще какие то сервисы помимо NetworkManager.service для управления сетью или днс
systemctl --type=service

проверить правильность /etc/hosts и /etc/hostname
https://wiki.archlinux.org/index.php/Network_configuration#Set_the_hostname

Перезагрузиться и проверить статус сервиса
systemctl status NetworkManager.service

попробовать заново настроить сеть и подключиться
смотреть журнал
 journalctl -b -u NetworkManager.service
Где прописать в etc/NetworkManager/conf.d/ - папка пустая
