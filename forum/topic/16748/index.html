Коллеги, требуется сделать осмысленный выбор для решения такой задачи - на ВПС поднят небольшой магазин, котрому нужно отправлять мейл. У меня свой домен, свой почтовый сервер - на другом железном сервере. Гад Магазин не вполне мною управляем - я не могу заставить его во всех случаях генерации писем отказаться от дефолтных имен отправителей, соотв. отправка почты не проходит. Есть мысль сделать локальный smtp, который будетп ринимать без авторизации почту только с localhost и дальше доносить ее до адресатов через клиентское подключение к работающему демону почты (postfix) с авторизацией и т.д. starttls.

Какие будут мнения?

# /etc/postfix/main.cf
# sasl
smtp_sasl_auth_enable = yes
smtp_sasl_password_maps = hash:/etc/postfix/sasl_passwd
smtp_sasl_security_options =
smtp_tls_security_level = may
relayhost = smtp.mail.com:587
# /etc/postfix/sasl_passwd
