vint
Как написать,где взять его?
Ни разу не сталкивался, но могу поделиться опытом, полученным путём пятиминутного гугления.

Открываем консоль, заходим под рутом и выполняем команды по очереди:

1) переходим в папку /etc
cd /etc

2) создаём (если её нет) папку cron.daily, куда скинем скрипт обновления:
mkdir cron.daily

3.1) создаём файл mycrontab с правилами (расписанием) для cron:
touch mycrontab

3.2) редактируем его с помощью любимого консольного текстового редактора (в моём случае nano):
nano mycrontab
и приводим содержимое файла mycrontab к виду:
00 02 * * * /usr/sbin/run-cron /etc/cron.daily

4) идём в папку /etc/cron.daily:
cd /etc/cron.daily

5.1) создаём файл “sysupd”, в который пропишем скрипт автообновления системы:
touch sysupd

5.2) редактируем его с помощью любимого консольного текстового редактора (в моём случае nano):
nano sysupd
и приводим содержимое файла sysupd к виду:
#!/bin/sh
# Проверяем, включен ли eth0, и обновляем
# все пакеты системы без предупреждений:
if /sbin/ifconfig | grep eth0 &> /dev/null ; then
  /usr/bin/pacman -Syu --noconfirm
fi

6) проверяем, прописан ли crond в демонах DAEMONS файла настроек /etc/rc.conf:
nano /etc/rc.conf
ищем строку DAEMONS и убеждаемся, что там прописан crond.
DAEMONS=(syslog-ng network crond … … … …

7) показываем крону новый файл расписаний:
crontab /etc/mycrontab

Всё, перезагружаемся.

Итого: каждый день, ровно в 2 часа ночи демон крона будет заходить в папку /etc/cron.daily и выполнять хранящиеся там файлы скриптов. Среди этих файлов (а может и единственный) есть и наш со скриптом обновления системы. Обновляться система будет через интерфейс eth0.


Примечания:
Cron - средство запуска программ по расписанию
cron jobs
