vasek, я ни в коем случае зла не держу, это просто была отсылка к первому посту. Наоборот, я благодарен сообществу и этому форуму в частности за ответы. Для оффтопика такие же вменяемые и квалифицированные ответы можно получить разве что на oszone. Просто так сложилось, что в мире линукса пока ещё больше грамотных вменяемых людей, нежели детей (причём не по возрасту, но по душе).
Теперь по теме. Что делает команда
/usr/bin/systemd --test --system --log-level=debug
? Ибо, во-первых, путь уже как бы изменился, а на официальном вики ещё не поправили, во-вторых, невозможно выполнить сию команду, т.к. выполнив от пользователя сие, получаем на выходе
$ /usr/lib/systemd/systemd --test --system --log-level=debug
systemd 208 running in system mode. (+PAM -LIBWRAP -AUDIT -SELINUX -IMA -SYSVINIT +LIBCRYPTSETUP +GCRYPT +ACL +XZ)
Failed to set hostname to <bigboyuser>: Operation not permitted
Failed to open /dev/tty0: Permission denied
Using cgroup controller name=systemd. File system hierarchy is at /sys/fs/cgroup/systemd/user.slice/user-1000.slice/session-1.scope.
Release agent already installed.
Failed to create root cgroup hierarchy: Permission denied
Failed to allocate manager object: Permission denied
а если от рута
sudo /usr/lib/systemd/systemd --test --system --log-level=debug
[sudo] password for bigboyuser:
Don't run test mode as root.
Сдаётся мне, нужна какая-то особая группа для выполнения этой команды? С cgroup ещё пока не разобрался, но, в ближайшее время сделаю это, т.к. это стало частью systemd...
P.S.: Сколько ни пытаюсь найти подробную документацию по systemd, максимум что выдаётся - копипаста перевода Сергея Пташника статей Леннарта либо Арчвики. Да даже из списка рассылок можно узнать более пикантные подробности, нежели из скудной документации! Сдаётся мне, что, пока ещё системда активно допиливается и развивается/изменяется, проблема с документацией так и останется. Т.к. лучше самого разработчика её никто не напишет.
