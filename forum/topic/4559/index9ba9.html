объяснить винде (в дуалбуте) что аппаратные часы в UTC…
Параметр HKLM\SYSTEM\CurrentControlSet\Control\TimeZoneInformation\RealTimeIsUniversal типа DWORD нужно установить в 1 - будет в UTC ;)
Действительно, поставив этот параметр, винда показывает время корректно, согласно часового пояса. Но, спустя час, возвращает свои часы = часы в биосе. Я даже службу времени виндоус отключил. Все равно спустя час винда стрелки переводит. Что интересно, после перезагрузки винды - вначале часы снова выставляются корректно, но спустя час - опять двадцать пять.

Windows Vista SP2 and Windows 7 have a preliminary support to UTC. The solution is add the DWORD value HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\TimeZoneInformation\RealTimeIsUniversal with hexadecimal value 1 in the registry (through regedit). Windows XP and Windows Vista SP1 also have support to UTC and can be activated in the same way, but there's a bug after resume from suspend/hibernation that resets the clock to localtime, so it's not recommended to use UTC in these systems.
