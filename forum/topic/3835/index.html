После обновления gnutls до версии 3.0.1-1 столкнулся с тем. что некоторые программы (у меня agave, но, судя по стонам на аглицйком формуме, и другие) отказываются запускаться и выдают:
error while loading shared libraries: libgnutls.so.26: cannot open shared object file: No such file or directory
Даунгредить до версии 2.8.6-1 нет охоты, пришлось создавать симлинк:
sudo ln -s /usr/lib/libgnutls.so.28 /usr/lib/libgnutls.so.26
Может, не самое правильное решение, но пока другого не нашел.

Уточнение: в пакете gnutls-3.0.2-* файлы libgnutls.so и libgnutls.so.28 являются символьными ссылками на libgnutls.so.28.0.2, поэтому делаем так:
sudo ln -s /usr/lib/libgnutls.so.28.0.2 /usr/lib/libgnutls.so.26
