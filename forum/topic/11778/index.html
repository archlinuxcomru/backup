Приветствую. Захотел поставить oss, чтобы поиграться, но облом наступил сразу же после установки. Всё по порядку:
1) Для начала "Приглушил" alsa с помощью файла /etc/modprobe.d/alsa_blacklist.conf
2) Ребутнулся на всякий случай
3) Поставил oss сам из офф реп
4) Установка прошла успешно, начинаю запуск oss:
[[email protected] santic]# systemctl start oss
Job for oss.service failed. See 'systemctl status oss.service' and 'journalctl -xn' for details.
Ладно, посмотрим что покажет systemctl status oss.service:
[[email protected] santic]# systemctl status oss.service
oss.service - Open Sound System v4
   Loaded: loaded (/usr/lib/systemd/system/oss.service; disabled)
   Active: failed (Result: exit-code) since Пт 2013-06-07 00:31:54 MSK; 5min ago
  Process: 1910 ExecStart=/usr/bin/soundon (code=exited, status=1/FAILURE)
июн 07 00:31:54 Fumofuu systemd[1]: Starting Open Sound System v4...
июн 07 00:31:54 Fumofuu soundon[1910]: Previous start of OSS crashed the system
июн 07 00:31:54 Fumofuu soundon[1910]: Please resolve the situation and rem...e
июн 07 00:31:54 Fumofuu systemd[1]: oss.service: main process exited, code...RE
июн 07 00:31:54 Fumofuu systemd[1]: Failed to start Open Sound System v4.
июн 07 00:31:54 Fumofuu systemd[1]: Unit oss.service entered failed state.
[[email protected] santic]#

Что-то не то, смотрим лог:
[[email protected] santic]# journalctl -xn
-- Logs begin at Вт 2012-12-11 17:56:49 MSK, end at Пт 2013-06-07 00:37:55 MSK. --
июн 07 00:37:55 Fumofuu systemd[1]: Failed to start Open Sound System v4.
-- Subject: Unit oss.service has failed
-- Defined-By: systemd
-- Support: http://lists.freedesktop.org/mailman/listinfo/systemd-devel
-- Documentation: http://www.freedesktop.org/wiki/Software/systemd/catalog/be02cf6
--
-- Unit oss.service has failed.
--
-- The result is failed.
июн 07 00:37:55 Fumofuu systemd[1]: Unit oss.service entered failed state.
Совсем разочаровался...решил попробовать soundoff && soundon:
[[email protected] santic]# soundoff && soundon
OSS not loaded.
Previous start of OSS crashed the system
Please resolve the situation and remove file
"/usr/lib/oss/starting". Then start OSS by
running soundon again.
[[email protected] santic]#
Косяк, удаляю файл starting и заново исполняю команду:
[[email protected] santic]# soundoff && soundon
OSS not loaded.

В итоге получаю сообщение:
[[email protected] santic]# soundoff && soundon
OSS not loaded.
modprobe: ERROR: could not insert 'osscore': Invalid argument
Loading the osscore module failed
[[email protected] santic]#
Куда копать - не знаю, гуглил - ничего толкового.
Пульса снесена, аудиоустройства определяются:
[[email protected] santic]# ossdetect -v
Detected CMedia CMI8788
USB support available in the system, adding USB driver
Detected Generic USB audio/MIDI device (BETA)
Система:
Linux Fumofuu 3.9.4-1-ARCH #1 SMP PREEMPT Sun May 26 11:56:48 CEST 2013 i686 GNU/Linux
Куда копать, товарищи? Спасибо за внимание!
