Доброго дня. Помогите, пожалуйста, разобраться

Обновил железо (мать, проц, видео), теперь никак не хочет загружаться grub и, соответственно, арч линукс. При загрузке ПК выдаёт ошибку "reboot and select proper boot device or insert boot media in selected boot device and press a key". Винда на этом же железе загрузилась нормально. В настройках биос стоит способ загрузки Legacy+UEFI, пробовал выбирать только UEFI, тогда вообще диск не отображается. Мать MSI B450 PRO MAX

Запустил с флешки Boot-Repair-Disk, пишет "/boot обнаружена. Проверьте параметры". Обычного варианта "Рекомендуемый способ восстановления" нет. Создал отчёт по ссылке http://paste.ubuntu.com/p/Fn5jkzt46z/ В отчёте вроде написано "No errors found in the Boot Parameter Block." Зато в расширенных настройках почему-то "Параметры GRUB" неактивны. Может дело в том, что диск зашифрован? Хотя шифрование только на sda3, sda1 и sda2 должны грузиться.

Судя по
sda1: __________________________________________________________________________

File system: vfat
Boot sector type: FAT16
Boot sector info: No errors found in the Boot Parameter Block.
Operating System:
Boot files: /efi/arch/grubx64.efi

Старая система была в UEFI установлена

Загрузитесь с лайва любого примонтируйте ефи раздел например в /mnt и скопируйте /mnt/EFI/arch/grubx64.efi в /mnt/EFI/Boot/bootx64.efi

в настройках загрузки выберите ефи режим и загрузку с диска sda

Или поставьте рефинд с того же лайва
refind-install --usedefault /dev/раздел фат ефи
или груб

или прописать загрузочную запись с вашим грубом в материнку
https://wiki.archlinux.org/index.php/Unified_Extensible_Firmware_Interface#efibootmgr

И если в настройках груба и фстаб прописано было по /dev/sd* а не по UUID то тоже исправлять надо.
И если уже груб загрузится выбирать fallback загрузку
