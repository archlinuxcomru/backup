На офф сайте AMD официально заявлено что fglrx работает на xorg версии 1.17 максимум, если на Маньяро воспользоваться системным установщиком каталистов - там автоматом версия xorg понижается с текущей 1.19 до 1.17. В arch это приходится делать руками (хотя сложного ничего нет). Для видяшек ATI под 1.19 особого выбора нет, только опенсорсные драйвера, либо amdgpu либо radeon.
Набираем в консоли
$lspci -kv
смотрим в самый низ вывода:

02:00.0 VGA compatible controller: Advanced Micro Devices, Inc. [AMD/ATI] Oland PRO [Radeon R7 240/340]
Subsystem: Hightech Information System Ltd. Device 2335
Kernel driver in use: radeon
Kernel modules: radeon, amdgpu
доступны radeon, amdgpu, работает radeon. Чтоб задействовать amdgpu, надо заблеклистить radeon (он является приоритетным)
sudo nano /etc/modprobe.d/radeon.conf
blacklist radeon

и перегрузить комп. Но оптимальней будет задействовать возможность гибридной графики, устанавливаем драйверы для всех имеющихся видеокарт (пример для intel+ATI):

sudo pacman -S xf86-video-ati xf86-video-amdgpu xf86-video-intel

в этом случае система по умолчанию будет задействовать Intel в качестве основной видяшки, дискретныая АТИ будет доступна через DRI_PRIME=1, например:

%DRI_PRIME=1 kinfocenter

Опытный пользователь, если есть возможность задействовать amdgpu, может попытаться с плясками и бубном водрузить себе из AUR для этого драйвера дополнительный проприетарный обвес под названием amdgpu-pro
