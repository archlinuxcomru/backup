1) Доступ извне есть, IP динамический, мой комп за NATом, порт проброшен, сайт извне открыть можно
2) Чтобы открыть сайт извне, пользователю достаточно вписать мой текущий IP или же (как и бывает чаще всего) имя dynDNS (gard.selfip.org), которое содержит мой текущий IP, пользователь в этом случае попадет на сайт, расположенный в каталоге /srv/http на моей машинке (фактически там сейчас ничего нет).

То что порт проброшен, гут, идём дальше:

[[email protected] ~]# cat /etc/httpd/conf/extra/httpd-vhosts.conf 
#
# Use name-based virtual hosting.
#
NameVirtualHost *:80
<VirtualHost *:80>
    ServerAdmin [email protected]
    DocumentRoot "/home/gard/Site"
    ServerName gard.selfip.org
    ServerAlias www.gard.selfip.org
        <Directory /home/gard/Site/>
		DirectoryIndex index.htm index.html
		Options -Indexes
		AllowOverride None
	</Directory> 
    ErrorLog "/var/log/httpd/site_error_log"
    CustomLog "/var/log/httpd/site_access_log" common
</VirtualHost>
Теперь все кто запрашивает gard.selfip.org будут попадать в /home/gard/Site по 80 порту.
