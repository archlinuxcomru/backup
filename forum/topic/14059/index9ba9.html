soupault
Буквально на неделе накатывал на новый ноутбук - поставил систему, а на моменте установки загрузчика оказалось, что надо бы раздел в 1MB для поддержки GPT.

GRUB при разметке MBR обычно устанавливается в "зазор" между бут-сектором диска с MBR и первым разделом.

Старые версии программ разметки обычно по-умолчанию предлагали начинать первый раздел с сектора 63, новые (в том числе для GPT) – с сектора 2048. Соответственно, размер зазора в первом случае 31кБ, во втором – тот самый мегабайт, за вычетом места под таблицу разделов.

Так что если вы создали на диске GPT, ничто не мешает вам всё равно установить GRUB в этот "зазор", просто создав загрузочный раздел прямо в этом "зазоре", ничего не переразбивая. Для груба места хватит.

Вот это моя флешка с установленным Арчем:
Number  Start (sector)    End (sector)  Size       Code  Name
   1            2048        20973567   10.0 GiB    8300  Linux filesystem
   2        20973568        63389662   20.2 GiB    8300  Linux filesystem
   3              64             191   64.0 KiB    EF02  BIOS boot partition
   4             192            2047   928.0 KiB   EF00  EFI System
Как видите, в начальном "зазоре" на ней создан не один, а целых два загрузочных раздела – для BIOS (64.0 KiB) и для UEFI (928.0 KiB).

Сам загрузочный образ GRUB2 занимает гораздо меньше места, чем под него отведено на этих разделах:
$ ls -lh grub/i386-pc/core.img grub/x86_64-efi/core.efi 
-rw-r--r-- 1 root root  25K сен 28 19:45 grub/i386-pc/core.img
-rw-r--r-- 1 root root 118K сен 28 19:44 grub/x86_64-efi/core.efi
