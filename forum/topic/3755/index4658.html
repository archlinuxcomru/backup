Рассказываю.
Команда ip ro (она же ip route list ) показывает у вас 4 сетевых маршрута, из них два принадлежат локалке, интерфейс eth0 , и два вашему ваймаксу, wimax0

Сначала локалка:
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.137
Этот маршрут означает, что что ваш (src) адрес в локалке – 192.168.1.137, диапазон адресов локалки 192.168.1.0/24 , то есть первые 24 бита, они же первые три числа адреса, постоянные, а последние 8 бит, или одно последнее число, может быть любое. По-другому это часто пишут как ip 192.168.1.0, маска 255.255.255.0, и все обращения к другим адресам в этом диапазоне будут идти через устройство dev eth0, то есть очевидно вашу сетевую карту.

default via 192.168.1.177 dev eth0
Этот маршрут означает, что ко ВСЕМ ОСТАЛЬНЫМ (default) адресам, то есть фактически к интернету, ваша система будет обращаться через другой (via) адрес 192.168.1.177 в этой самой локальной сети, которая подключена к сетевой карте (dev eth0).
Вспомните, что в вашей локальной сети имеет такой адрес, и каким образом оно у вас оказалось прописанным в качестве маршрута по-умолчанию. Возможно, надо посмотреть, что и почему у вас написано в /etc/rc.conf в строках
interface=
address=
netmask=
gateway=

Соответственно, для ваймакс у вас другие два маршрута:
10.174.48.0/21 dev wimax0 proto kernel scope link src 10.174.55.2 metric 204
Этот означает, что через ваше беспроводное устройство, (dev wimax0) будут идти обращения ко всем адресам некоторой подсети 10.174.48.0/21 (видимо, провайдера), и ваш (src) адрес в этой подсети – 10.174.55.2

default via 10.174.48.1 dev wimax0 metric 204
А тут написано, что ко всем остальным (default) адресам, то есть к интернету, ваша система обращалась БЫ через другой (via) адрес 10.174.48.1 (видимо, роутер провайдера) по беспроводному интерфейсу wimax0, если БЫ у вас УЖЕ НЕ был прописан другой маршрут по-умолчанию с более приоритетной метрикой 0 в локалку, тогда как у тут только metric 204

Каждый из этих маршрутов может быть добавлен или удалён от рута консольной командой ip ro или ip route , соответственно для добавления:
ip route add маршрут
Для удаления
ip route delete маршрут
Не бойтесь что-то таким образом сломать, поскольку выполненные из консоли команды добавления или удаления роутов действуют максимум до перезагрузки.

Для начала вы можете после подключения ваймакса попробовать удалить маршруты в локалку, которые своей наивысшей нулевой метрикой перебивает вам доступ к нему. Если после этого у вас появится беспроводной интернет, значит вам надо искать в настройках того, что вы для него установили (потому что я не знаю, что вы установили), как убрать оттуда эту метрику 204.
