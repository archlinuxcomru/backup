Правильно, поскольку я врядли бы объяснил лучше.
Некоторое время назад в AUR был PKGBUILD, подготовленный hatred'ом, но потом он, по всей видимости, удалил его сам или.., вобщем, вот:
# Contributor: Alexander 'hatred' Drozdoff <[email protected]>
pkgname=openbsd-malloc
pkgver=1.83
pkgrel=2
pkgdesc="Alternative malloc() implementation from OpenBSD (useful for firefox/thunderbird)"
arch=(i686 x86_64)
url="http://mr.himki.net/index-alloc.html"
license=('custom:Beerware')
depends=('glibc')
makedepends=('gcc')
install=openbsd-malloc.install
source=(http://truediamon.ru/system/files/OpenBSD_malloc_Linux.c license.txt)
md5sums=('be4ad18f9bbb76b3299a12f3b6dc946f'
 '0b578bb0df968894254e323ef9bd794a')

build() {
  mkdir -p "$srcdir/$pkgname-$pkgver"
  cd "$srcdir/$pkgname-$pkgver"

  cp ../OpenBSD_malloc_Linux.c ./
  gcc -shared -fPIC $CFLAGS OpenBSD_malloc_Linux.c -o malloc.so

  install -pD -m755 malloc.so      $pkgdir/opt/lib/malloc.so
  install -pD -m644 ../license.txt $pkgdir/usr/share/licenses/$pkgname/license.txt
}

Файла openbsd-malloc.install у меня нет, ссылка (source) исправлена уже мною на актуальный, md5sums придётся поправить.
Если у кого-нибудь возникнет желание и возможность поддерживать - перезалейте исходник в надёжное для вас место, отредактируйте PKGBUILD и пр., пришлите в AUR...
Сделал бы и сам, но не могу гарантировать его долговременную поддержку. Кто-нибудь, держащий своё зеркало, сервер и т.п. - более подходящий кандидат.
