или свое без ключа обычным zip:
[Desktop Entry]
Type=Service
ServiceTypes=KonqPopupMenu/Plugin
MimeType=all/all
Actions=Encrypt
[Desktop Action Encrypt]
Name=Зашифровать мне это быстро!
Exec=zip -r -e -P `kdialog --password "Введите пароль для шифрования. Используйте заглавные и строчные буквы, цифры и знаки препинания." --title "Ввод пароля для шифрования"` %U.zip %U

положить в ~/.kde4/share/kde4/services/ServiceMenus с расширением .desktop

Использую для папок Cryptkeeper — очень удобная штука, а для файлов GPA — GNU Privacy Assistant. Не знаю, как это будет выглядеть в KDE (использую Xfce4 и Openbox), но попробуйте. Cryptkeeper работает по типу TrueCrypt: создаёт зашифрованную папку, в которую можно сбрасывать файлы. Но, в отличие от TrueCrypt'a, размер папки может меняться. А GPA вообще сделал по правому клику “Зашифровать файл” (в Thunar добавил такую команду: gpg -r “имя_ключа” –encrypt %f && rm -f %f). Быстро и просто.

Что может быть проще подключать зашифрованную папку одной галочкой в Cryptkeeper'е? А GPA (в вашем случае, наверное, будет KGPA) сам пароль запрашивает. Кликаешь по файлу, он требует пароль, а потом распаковывает. Но ведь в условии значилось: никаких ключей?
Кстати, Amigo идею подсказал. Теперь у меня и папки шифруются по правому клику. Добавил в Thunar:
zip -r %N.zip %N | gpg -r “имя ключа” –encrypt %N.zip && rm -Rf %F.zip %F
Теперь ПКМ по папке, выбираем “Зашифровать папку”, всё шифруется, исходная папка удаляется. Надо расшифровать? Просто кликаем дважды по зашифрованной папке, GPA затребует пароль, вводим, получаем расшифрованную папку. Что может быть проще?
