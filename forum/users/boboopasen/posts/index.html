ИМХО самый оптимальный вариант - использовать DM от рабочего окружения.

Если стоит xfce или *box, т.е. у которых нет своих нативных менеджеров, использовать slim (или другой)

Я когда на xfce4 запускал, то ставил slim, и приводил /etc/inittab к такому виду:
id:5:initdefault:
rc::sysinit:/etc/rc.sysinit
rs:S1:wait:/etc/rc.single
rm:2345:wait:/etc/rc.multi
rh:06:wait:/etc/rc.shutdown
su:S:wait:/sbin/sulogin -p
c1:2345:respawn:/sbin/agetty -8 38400 tty1 linux
c2:2345:respawn:/sbin/agetty -8 38400 tty2 linux
c3:2345:respawn:/sbin/agetty -8 38400 tty3 linux
c4:2345:respawn:/sbin/agetty -8 38400 tty4 linux
c5:2345:respawn:/sbin/agetty -8 38400 tty5 linux
c6:2345:respawn:/sbin/agetty -8 38400 tty6 linux
ca::ctrlaltdel:/sbin/shutdown -t3 -r now
x:5:respawn:/usr/bin/slim >/dev/null 2>&1

В файле ~/.xinitrc писал:
#!/bin/bash
exec ck-launch-session startxfce4 # для xfce4

Ну и пользователя добавлял в группу power ( gpasswd -a power username ), чтобы можно было при использовании иксов управлять питанием компьютера.

Если же есть нативный DM, то просто добавить демон в rc.conf. Тогда пользователя в power добавлять не обязательно (вроде) и в ~/.xinitrc ck-launch-session можно будет не писать, т.е. просто exec startkde (для KDE)
