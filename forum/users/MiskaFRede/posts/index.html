но когда я решил поменять картинку через Grub Customizer
В "Grub Customizer" на последней вкладке слева вверху есть раскрывающийся список для выбора темы. Если темы в папке есть, то они будут отображаться в этом списке. Если нет тем, то вытащить из любого LiveCD, они есть практически у всех. В Manjaro тема лежит в другой папке (смотреть путь в конфиге), но у неё фишка, что не портится при любом разрешении экрана. В Mint точно есть тема. Ну и по ссылке что я давал выше там вообще 5 анимационных тем, я бы их не ставил, ну или убрал анимацию из конфигов, там есть моя тема, но она на ноутбуках плющится, иконки овальные, если не выставить разрешение, а также "Grub Customizer" не видит мою тему, почему не знаю, чтобы видел я переименовываю какую нибудь тему в своё название, потом выбираю её в "Grub Customizer", потом подставляю туда свою тему. Много костылей я тут наобъяснял, но это работает, сам пользуюсь и на компы ставлю. Ну если ставил тему через "Grub Customizer" она не слетит, а можно ещё найти имя темы в конфиге и просто заменить имя, но после обновления Grub2 и сборки конфига скриптами всё это снесётся, поэтому я ещё делаю копию конфига, чтобы возвратить привнесённое.
Честно говоря, ничего не понял...

через Grub Customizer
Там, в конфиге, столько лишнего мусора…
Фактически и так работает.. Это только начало конфига, дальше собственно выбор операционки

insmod gpt
insmod pc
insmod gfxmenu
insmod videotest
insmod tga
insmod png
insmod gfxterm
insmod lspci
insmod vbeinfo
insmod vbe
insmod ntfs
insmod chain
insmod biosdisk
insmod font

set default=0
set timeout=30
set fallback=1
search --no-floppy --fs-uuid --set=root 894a
search --no-floppy --fs-uuid --set=boot 23c6
set root=${root}
loadfont ($boot)/grub/fonts/unicode.pf2
loadfont ($boot)/grub/themes/SteamBP/DejaVuSans-20.pf2
set gfxmode=1920x1080
if terminal_output gfxterm ; then true ; else
terminal gfxterm
fi
set theme=($boot)/grub/themes/SteamBP/theme.txt
export theme
#####
1,2 загрузка шрифта
3 разрешение экрана
4 собствено тема
