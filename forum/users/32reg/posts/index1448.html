Natrio, KDE. В дельфине, в домашней папке нажимаю на Документы - открываются документы которые реально находятся в /mnt/disk_d/Документы/Linux.
Нажимаю на Install - открывается install по пути /mnt/disk_d/Install/Linux. Тут все правильно.

В левой части окна дельфина (там где точки входа, закладки) в самом низу нажимаю на "Жесткий диск 450 Гб" (disk_d который) - а открывается опять /mnt/disk_d/Install/Linux вместо /mnt/disk_d
В крусадере тоже самое.

Если в дельфине нажать на "Корневую папку", потом на /mnt, потом на disk_d, то он откроется нормально.

Такое поведение, если в fstab раскомментирована последняя строка, если ее убрать, то все нормально начинает открываться почему-то

Всем привет. Раз уж начал задавать вопросы, может и этот решится. Больно он надоел уже.
Проблема в следующем: есть такого вида fstab:
#
# /etc/fstab: static file system information
#
# <file system>	<dir>	<type>	<options>	<dump>	<pass>
#tmpfs		/tmp	tmpfs	nodev,nosuid	0	0

UUID=d4e04f6d-cf9c-4653-985a-a97f87c484ed / ext4 defaults 0 1
UUID=dd5e28aa-d0f1-4a8f-9fac-72ca3aecebf4 swap swap defaults 0 0
UUID=9d7b66c7-3269-46cd-8470-8c6fc5413947 /home ext4 defaults 0 2
UUID=200C10C91374FF92 /mnt/disk_d ntfs-3g defaults,noexec,iocharset=utf8 0 0

/mnt/disk_d/Документы/Linux /home/arch/Документы none bind 0 0
/mnt/disk_d/Install/Linux /home/arch/Install none bind 0 0

То есть, монтирую виндовый диск D по пути /mnt/disk_d и потом монтирую биндом две папки оттуда в домашнюю папку пользователя.
До какого-то момента все работало правильно, но потом сломалось (где-то уже около года назад). Хоть и не критично, но на нервы действует.

Теперь при переходе в /mnt/disk_d открывается не disk_d, а /mnt/disk_d/Install/Linux. Т.е. забинденная папка Install. При переходе же в /home/arch/Документы - открывается, как и положено, /mnt/disk_d/Документы/Linux. И при переходе в /home/arch/Install открывается /mnt/disk_d/Install/Linux. Тут тоже нормально.

Если последнюю строку в fstab закомментировать, то /mnt/disk_d начинает открываться правильно.

Подскажите, в чем может быть проблема или в какую сторону хоть посмотреть?

PS: может пригодится, строка хуков из mkinitcpio.conf:
HOOKS="base udev autodetect block filesystems"
