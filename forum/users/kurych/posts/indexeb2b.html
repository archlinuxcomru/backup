Предлагаю такой рабочий вариант:
Этот файлик по вкусу, что бы через меню запускать можно было. Главное, конечно, строка Exec=…
$ cat ~/.local/share/applications/chromium-pad.desktop 
[Desktop Entry]
Version=1.0
Name=Chromium - PAD
GenericName=Web Browser - PAD
Comment=Access the Internet
Exec=chromium --proxy-pac-url=file:///home/user/Public/home-wpad.dat %U
Terminal=false
Icon=chromium
Type=Application
Categories=GTK;Network;WebBrowser;
MimeType=text/html;text/xml;application/xhtml_xml;
В файле /home/user/Public/home-wpad.dat прописано, куда как ходить:
$ cat /home/user/Public/home-wpad.dat
function FindProxyForURL(url, host) {
	if (isInNet(host, "192.168.134.48", "255.255.255.240")
	 || isInNet(host, "192.168.160.0", "255.255.255.192")
	)
		return "PROXY proxy.local:3128";
//	if (isInNet(host, "192.168.1.0", "255.255.255.0") ||
//	    dnsDomainIs(host, "libo.ru"))
//		return "SOCKS 127.0.0.1:1080";
        if (isPlainHostName(host)
	 || dnsDomainIs(host, "localhost")
	 || isInNet(host, "127.0.0.0", "255.0.0.0")
	 || isInNet(host, "192.168.0.0", "255.255.0.0")
	 || isInNet(host, "10.0.0.0", "255.0.0.0")
	)
                return "DIRECT";
	else	return "PROXY 192.168.1.1:3128";
}
Вон даже через socks примерчик есть, только закомментирован. В общем, правила можно накручивать любой сложности. При этом, можно даже менять при запущеном браузере без перезагрузки.
Работает не только в хромиуме. В Firefox можно в настройках указать. Главное, указывать не просто путь к файлу, а в виде URL: file:///home/user/Public/home-wpad.dat
