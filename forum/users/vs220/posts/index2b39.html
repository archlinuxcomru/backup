Попробуйте 
own_window_type override
.conkyrc
#############################
# - Window specifications - #
#############################
own_window_class Conky
own_window yes
own_window_type override
own_window_transparent yes
own_window_hints undecorated,below,sticky,skip_taskbar,skip_pager

http://conky.sourceforge.net/config_settings.html
own_window_type : if own_window is yes, you may specify type normal, desktop, dock, panel or override (default: normal). Desktop windows are special windows that have no window decorations; are always visible on your desktop; do not appear in your pager or taskbar; and are sticky across all workspaces. Panel windows reserve space along a desktop edge, just like panels and taskbars, preventing maximized windows from overlapping them. The edge is chosen based on the alignment option. Override windows are not under the control of the window manager. Hints are ignored. This type of window can be useful for certain situations.

По хоткеям рекомендую xbindkeys плюс в том что работает на разных WM и DE.

Для переключения между приложениями , изменения размеров и положения окон wmctrl
Запускаем приложение, а если уже запущено то переключаемся на него
Запоминаешь хоткеи и переключаешся сразу на нужное окно

Динамические хоткеи для окон

Хоткеи для изменения размеров активного окна:
Слева на пол экрана 
WIDTH=`xdpyinfo | grep 'dimensions:' | cut -f 2 -d ':' | cut -f 1 -d 'x' `&& HALF=$(($WIDTH/2)) && wmctrl -r :ACTIVE: -b add,maximized_vert && wmctrl -r :ACTIVE: -e 0,0,0,$HALF,-1
Справа на пол экрана 
WIDTH=`xdpyinfo | grep 'dimensions:' | cut -f 2 -d ':' | cut -f 1 -d 'x' `&& HALF=$(($WIDTH/2)) && wmctrl -r :ACTIVE: -b add,maximized_vert && wmctrl -r :ACTIVE: -e 0,$HALF,0,$HALF,-1
Произвольные размеры 
wmctrl -r :ACTIVE: -e 0,X-позиция,Y-позиция,ширина,высота
Для работы требуется wmctrl и xdpyinfo
