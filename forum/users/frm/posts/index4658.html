# 2 года, 3 месяца назад (отредактировано 2 года, 3 месяца назад)
Темы: 1
Сообщения: 11
Участник с: 19 мая 2020
Ситуация следующая. Установлен VirtualBox, в нем была создана вирутальаня машина с виндой 7. Все работало с полгода.
Где-то пару недель назад после каких-то из обновлений (и, возможно, удалений пакетов-сирот) возникла проблема. При попытке запустить виртуальную машину или получить доступ к ее настройкам вся программа падала.
Была попытка скопировать рабочую виртуальную машину с другого компьютера, но при добавлении ее в VirtualBox на моем компьютере результат тот же - при попытке запуска или при открывании меню настроек машины VirtualBox падает.

Что пробовал сделать - удалял все пакеты VirtualBox с машины и все настройки из home. Устанавливал заново, добавлял старую виртуальную машину и скопированную с другого компьютера. Устанавливал вместо virtualbox-host-modules-arch пакет virtualbox-host-dkms. Результат тот же.

Установлено (guest-пакеты, кроме iso, установил от отчаяния):
pacman -Qs virtualbox
local/linux 5.6.13.arch1-1
    The Linux kernel and modules
local/virtualbox 6.1.8-2
    Powerful x86 virtualization for enterprise as well as home use
local/virtualbox-guest-dkms 6.1.8-2
    VirtualBox Guest kernel modules sources
local/virtualbox-guest-iso 6.1.8-1
    The official VirtualBox Guest Additions ISO image
local/virtualbox-guest-utils 6.1.8-2
    VirtualBox Guest userspace utilities
local/virtualbox-host-modules-arch 6.1.8-1
    Virtualbox host kernel modules for Arch Kernel

Ядро Linux 5.6.13.arch1-1
Пакет linux-headers установлен.
Виртуализация в efi включена.

Логи:
VBoxSVC.log
По поводу "VBOX_E_OBJECT_NOT_FOUND" в логах - я в процессе экспериментов одну из машин перемещал, так что все ок. Та, которая обозначена WM_1_name на месте.

В процессе изысканий выяснился вот какой момент. На Арчевики написано про модули ядра vboxdrv, vboxnetadp, vboxnetflt, и vboxpci. При попытке вручную загрузить их через modprobe удалось загрузить все, кроме vboxpci. Получил ошибку:
modprobe vboxpci
modprobe: FATAL: Module vboxpci not found in directory /lib/modules/5.6.13-arch1-1

Он и правда отсутствует:
ls -l /lib/modules/5.6.13-arch1-1/extramodules/
-rw-r--r-- 1 root root 138800 мая 17 01:56 vboxdrv.ko.xz
-rw-r--r-- 1 root root   3984 мая 17 01:56 vboxnetadp.ko.xz
-rw-r--r-- 1 root root  12800 мая 17 01:56 vboxnetflt.ko.xz

Подскажите, может ли быть проблема в отсутствующем модуле ядра (и куда он мог дется)?
И как это можно попытаться починить.
