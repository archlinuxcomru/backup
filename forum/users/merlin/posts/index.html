Ну, во первых, man pacman:
HoldPkg = package …
If a user tries to –remove a package that’s listed in HoldPkg, pacman will ask for confirmation before proceeding.

IgnorePkg = package …
Instructs pacman to ignore any upgrades for this package when performing a –sysupgrade.
HoldPkg - запрещает удалять пакет, а не обновлять.
Вы противоречите ману!

По сабжу
# pacman -S $(pacman -Qq | grep -v "$(pacman -Qmq)")
А вообще glibc лучше удалить из holdpkg, иначе подобная ситуация будет повторяться вновь и вновь.
Не совсем понял, HoldPkg - если юзер попытается УДАЛИТЬ пакет перечисленый в HoldPkg, пакман спросит подтверждение об этом. Разве нет ? В таком случае мне не совсем понятно как связано HoldPkg = glibc с нынешней моей ситуацией.
