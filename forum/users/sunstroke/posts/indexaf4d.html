pacnew-файлы это новые варианты конфигурационных файлов, которые поставляются вместе с обновлением пакетов. Установщик дает им такое расширение, так как не рискует переписывать существующие файлы конфигурации - вдруг пользователь внес там важные изменения.
Все что вам нужно, это сравнить свой старый и новый файлы и подумать нужно ли что-нибудь переносить из нового в старый. Если не нужно, то можно смело удалять. “Дописывать pacnew-файл в конец старого” НЕ НУЖНО - последствия непредсказуемы. :)

тупо указываю в rc.conf
eth0=dhcp
INTERFACES=(eth0)
Сейчас же вроде как надо для dhcp:
# Wired network setup
#   - interface: name of device (required)
#   - address: IP address (leave blank for DHCP)
#   - netmask: subnet mask (ignored for DHCP)
#   - gateway: default route (ignored for DHCP)
interface=eth0
address=
netmask=
gateway=
Еще можно на модеме посмотреть логи, если есть такая функция, на предмет сообщений о реконнектах (если у вас соединение поднимает именно сам модем).
