--/etc/ppp/ip-up--
!/bin/sh
#

#
# Execute all scripts in /etc/ppp/ip-up.d/
for ipup in /etc/ppp/ip-up.d/*.sh; do
  if [ -x $ipup ]; then
    # Parameters: interface-name tty-device speed local-IP-address remote-IP-address ipparam

  fi
done

route add default dev ppp0

gateway="default gw 192.168.169.1"

Добрый вечер! Поставил арч и тоже проблемы с VPN: настроил конфиги так как работал в дебиан

1. Создаем файл /etc/ppp/peers/vpn с содержимым
user "nokman"
pty "/usr/sbin/pptp 10.100.100.100 --nolaunchpppd"
file /etc/ppp/options.pptp
где nokman - мой логин; 10.100.100.100 - адрес VPN-сервера

2. Создаем/редактируем файл /etc/ppp/options.pptp с содержимым
lock
noauth
refuse-eap
refuse-chap
nobsdcomp
nodeflate
usepeerdns
defaultroute
#replacedefaultroute (выкинул так как в арче не работает)

3. Создаем/редактируем файл /etc/ppp/chap-secrets с содержимым
# Secrets for authentication using CHAP
# client                server  secret                            IP addresses
nokman           *       "password"                     *
где nokman - мой логин для VPN, password - мой пароль

4. Подключаемся
sudo pon vpn
Так вот в арче подключение вроде бы как и устанавливается, но пингуется только VPN-сервер (10.100.100.100), а остальные адреса нет - “Unknown host”. 
ifconfig -a

netstat -nr


Все настройки VPN получаю автоматом (DNS, IP). Проверял в /etc/resolv.conf DNS-адреса присутствуют после подключения.
/etc/rc.conf
--NETWORKING--
HOSTNAME="arch"
eth0="dhcp"
INTERFACES=(eth0)
gateway="default gw 192.168.169.1"
ROUTES=(gateway)

В общем нужна ваша помощь.
