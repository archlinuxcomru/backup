В связи с переездом catalyst на linux-lts, пришлось его установить. После этого перестала работать функция отключения тачпада (светодиод в углу тачпада).
Раньше synaptics-led из AUR замечательно с этим справлялся, он создавал psmouse.ko.gz в /usr/lib/modules/extramodules.../ и он подхватывался и работал.
Я пересобрал psmouse.ko.gz для lts ядра, но он не подхватывается при загрузке.
uname -a
Linux vantu5z-book 4.1.8-1-lts #1 SMP Tue Sep 22 17:49:49 CEST 2015 x86_64 GNU/Linux

modinfo '/usr/lib/modules/extramodules-4.1-lts/psmouse.ko.gz'
filename:       /usr/lib/modules/extramodules-4.1-lts/psmouse.ko.gz
license:        GPL
description:    PS/2 mouse driver
author:         Vojtech Pavlik <[email protected]>
alias:          serio:ty05pr*id*ex*
alias:          serio:ty01pr*id*ex*
depends:        led-class,serio,libps2
vermagic:       4.1.8-1-lts SMP mod_unload modversions
parm:           proto:Highest protocol extension to probe (bare, imps, exps, any). Useful for KVM switches. (proto_abbrev)
parm:           resolution:Resolution, in dpi. (uint)
parm:           rate:Report rate, in reports per second. (uint)
parm:           smartscroll:Logitech Smartscroll autorepeat, 1 = enabled (default), 0 = disabled. (bool)
parm:           resetafter:Reset device after so many bad packets (0 = never). (uint)
parm:           resync_time:How long can mouse stay idle before forcing resync (in seconds, 0 = never). (uint)

Но ядро настырно загружает свой psmouse. После загрузки системы имеем:
lsmod | grep mouse
mousedev               20480  0
psmouse               122880  0
libps2                 16384  2 atkbd,psmouse
serio                  20480  10 serio_raw,atkbd,i8042,hp_accel,psmouse

Если вручную сделать, то всё работает:
sudo modprobe -r  psmouse
sudo insmod '/usr/lib/modules/extramodules-4.1-lts/psmouse.ko.gz'
И собственно вопрос, почему он не запускается автоматически? Может в lts нужно, что-то дополнительно настроить для этого?
