Доброго времени суток! Появилась необхоимость расшарить принтер по сети, да так, чтобы к нему имели доступ только две виндовые машины (ну и локалхост, конечно). Принтер работает под управлением CUPS. В ArchWiki (п. 4.4) нашел вроде бы подходящее решение.
Старый блок <Location>…</Location> файла /etc/cups/cupsd.conf заменил на:
<Location />
# Allow shared printing...
Order allow,deny
Deny all
Allow 127.0.0.1
Allow 172.16.34.10
</Location>

И на виндовой машине тоже провел указанные в вики настройки. Только в конце появилось окно с параметрами безопасности:
*Использовать анонимную учетную запись
*Использовать автоматически имя входа и пароль из Windows
*Использовать указанную учетную запись пользователя (и поля "Пользователь" и "Пароль")

Причем в любом случае начинает ругаться на то, что доступа к принтеру нет и надо попробовать другое имя пользователя и пароль.
Может быть кто-то решал подобную проблему раньше или просто может что-то подсказать? Заранее благодарен.

UPD:
Кстати, попробовал прописать
<Location />
# Allow shared printing...
Order allow,deny
Allow all
</Location>
Виндовый клиент перестал ругаться на отсутствие доступа, но, при попытке печати с него, выбивает предохранитель на ИБП сервера! o_O
