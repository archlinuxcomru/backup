Хочу иметь работающий по умолчанию при загрузке сетевой интерфейс. Однако, каждый раз мне нужно выполнить следующие команды, чтобы задействовать сетевой профиль:
$ sudo rfkill unblock all
$ sudo ifconfig wlan0 down
$ sudo netctl start wifi_network

Насколько я понял, для модуля ядра rfkill есть параметр default_state, задающий начальное состояние беспроводных устройств.
$ cat /sys/module/rfkill/parameters/default_state
1
т.е. он настроен так, как надо.

Также есть параметр master_switch_mode, установить который у меня не получилось вовсе:
$ cat /etc/modprobe.d/modprobe.conf
options rfkill master_switch_mode=0
$ ls /sys/module/rfkill/parameters/
default_state

Пробовал тупо заблеклистить rfkill, но это также не помогло - не удавалось поднять сетевой интерфейс с другой ошибкой.

dnsmasq - dns-сервер с dhcp-сервером. Я решил ограничиться простым dhcp.

Вот мой dhcpd.conf:
default-lease-time 600;
max-lease-time 7200;
# If this DHCP server is the official DHCP server for the local
# network, the authoritative directive should be uncommented.
authoritative;
# This is a very basic subnet declaration.
subnet 172.17.0.0 netmask 255.255.255.0 {
  range 172.17.0.50 172.17.0.100;
  option routers 172.17.0.1;
  option subnet-mask 255.255.255.0;
  option domain-name-servers 8.8.8.8;
}

Ещё по арч-вики указал, чтобы сервер dhcp слушал только нужный мне интерфейс wlan0 (/etc/conf.d/dhcp):
...
DHCP4_ARGS="-q wlan0"
...

И, следуя рекомендации, занёс в таблицу маршрутазации такую запись:
route add -host 255.255.255.255 wlan0
