Зайдите в консоль от root
Ctrl+Alt+F1
Попробуйте выполнитть команды в чёткой последовательности:

wpa_passphrase G_Virus_NET ваш пороль без кавычек > /etc/wpa_supplicant.conf'
wpa_supplicant -B -Dwext -iwlp0s29f7u1 -c /etc/wpa_supplicant.conf
dhcpcd wlp0s29f7u1
ping google.com
Ctrl+C

Возвращайтесь в графическое оболочку:
Ctrl+Alt+F7

Когда хеш в /etc/wpa_supplicant.conf' будет сгенерирован от root можно будет выходить в сеть с обычным входом в системму выполнив команды:
sudo wpa_supplicant -B -Dwext -iwlp0s29f7u1 -c /etc/wpa_supplicant.conf
sudo dhcpcd wlp0s29f7u1

Установка be.shell производиться из рабочего окружения KDE

Подготовка:
Копируем папку be.shell из моего архива в /home/****/.kde4/share/apps/
Копируем be.shell.txt в папку /home/****/.kde4/share/config/ как be.shell

С помощью текстового редактора в файле be.shell изменяем пути

в строке 121 MenuUpdater=/home/alf/.kde4/share/apps/be.shell/scripts/netcfg-profiles
в строке 128 MenuUpdater=/home/alf/.kde4/share/apps/be.shell/scripts/netquality
в строке 139 Icon=/home/alf/.kde4/share/apps/be.shell/Themes/Symphony/face.png
в строке 85 Icon=/home/alf/.kde4/share/apps/be.shell/Themes/Symphony/arch-start-here.png

Вместо alf пишем реальное имя вашей домашней папки.

Если нет устанавливаем git
sudo pacman -S git

Скачиваем BE::Shell
# git clone git://git.code.sf.net/p/be-shell/code be-shell

Копируем скрипты из скачанной папки /home/****/be-shell/scripts/ в папку /home/****/.kde4/share/apps/be.shell/scripts/

Установка проходит быстро:

# cd be-shell
# ./configure
# cd build
# make
# sudo make install

# kquitapp plasma-desktop
# kquitapp kuiserver
# kquitapp krunner
# be.shell
И мы уже в BE::Shell

Окончательные настройки:
Вызовите графический редактор служб автозагрузки :
# kcmshell4 kded
и отключите “Диспетчер уведомлений о состоянии”
( be.shell имеет свой встроенный диспетчер)

Создайте папку /home/****/.kde4/share/autostart
Скопируйте и сделайте исполняемыми файлы:be.shell.desktop, plasma-desktop.desktop и krunner.desktop из скачанной спомощю git папки be-shel, находящейся в домашнем каталоге, в ~/.kde4/share/autostart/
Если чтото будет не работать просто удалите их
и вы сможете загрузится в обычное окружение плазмы KDE.

Тему ( в папке /home/****/.kde4/share/apps/be.shell/Themes/ сюда складуються новые скачанные темы )
выберите из меню вызванного кликом правой кнопкой мышки по рабочему столу~ Themes ~ Symphony .
Обои выберите из меню вызванного правым кликом мышки по рабочему столу ~ Wallpaper ~ Select.
Так же размеры панелей можно изменять в графическом режиме.

Примечание:
Вместо звёздочек вставляем имя вашей домашней пепки.
Conky lua конфигурационные файлы находятся в папке с дополнениями к теме расположенные в этом же архиве.
Толщина кнопок глобального меню меняется размером шрифта в строке 36 в /home/****/.kde4/share/apps/be.shell/Themes/Symphony/style.css
также можно открыть этот файл кликнув правой кнопкой мышки по рабочему столу ~ темы ~ редактирование темы .
Толщину шрифта часов можно изменить в строке 92
Размеры панелей меняются выбором изменений из меню вызванного кликом правой кнопкой мышки по панелям.

Подробнее о других возможностях и настройках на BE::Shell - The WIKI

Привожу пример для наглядности с непрозрачными conky окнами .Где видно как можно запуускать одновременно несколько conky окон, при помощи просого скрипта.

Например:
#!/bin/sh
conky -c /home/alf/.conky/TRON/TRON.conkyrc
#sleep 25s
conky -c /home/alf/.conky/Forecast/conkyrc &
sleep 5s
conky -c /home/alf/.conky/TRON/Rings.conkyrc &
exit
Этим мы обходим конфликтующие тезисы и не нужно долго переписывать скрипты для личного приминения.А разнообразного матетьяла в интернете очень много и каждый может сделать то что ему больне по душе…
