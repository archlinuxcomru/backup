# 2 года, 3 месяца назад
Темы: 1
Сообщения: 9
Участник с: 19 мая 2020
Потребуется proot из aur.
Качаем 18.04
Распаковываем систему прямо из образа в любую папку.
su
mkdir /tmp/iso
mount ./ubuntu-18.04.4-desktop-amd64.iso /tmp/iso/
mkdir /tmp/iso-filesystem
mount /tmp/iso/casper/filesystem.squashfs /tmp/iso-filesystem
cp -r -a /tmp/iso-filesystem /home/ubuntu18
umount /tmp/iso-filesystem /tmp/iso
rm -r /tmp/iso-filesystem /tmp/iso
Для игр из gog достаточно такого стартового сценария
#!/bin/sh
#export LD_PRELOAD="$LD_PRELOAD:/usr/\$LIB/libgamemodeauto.so.0"
PROOT_NO_SECCOMP=1 exec proot -r /home/ubuntu18 -b /home -b /proc/ -b /dev -b /sys -b ./ ./start.sh
Остаётся только поправить .desktop файл для запуска игры сменив исполняемый файл.
Для игр из steam заходим в свойства игры и "Установить параметры запуска..."
PROOT_NO_SECCOMP=1 LD_LIBRARY_PATH=./ proot -r /home/ubuntu18 -b /home -b /proc/ -b /dev -b /sys -b ./ %command%
Если библиотека с играми распологается в другом месте, нужно добавить её также через параметр -b, иначе кеш шейдеров работать не будет:
Failed to create /mnt/RAID/tank/SteamLibrary/steamapps/shadercache/48700 for shader cache (No such file or directory)---disabling.
Пробовал этот способ на RimWorld (gog) и Mount & Blade (steam). LD_LIBRARY_PATH ведущий к библиотекам убунты никак не помогал.
