Тема мне очень интересная получаеться чтобы увидеть рабочий стол KDE надо сделать:
1. Файл ~/.xinitrc привести к примерно к токому виду 
!/bin/sh

userresources=$HOME/.Xresources
usermodmap=$HOME/.Xmodmap
sysresources=/etc/X11/xinit/.Xresources
sysmodmap=/etc/X11/xinit/.Xmodmap

# merge in defaults and keymaps

if [ -f $sysresources ]; then
    xrdb -merge $sysresources

fi

if [ -f $sysmodmap ]; then
    xmodmap $sysmodmap
fi

if [ -f "$userresources" ]; then
    xrdb -merge "$userresources"

fi

if [ -f "$usermodmap" ]; then
    xmodmap "$usermodmap"
fi

# start some nice programs

if [ -d /etc/X11/xinit/xinitrc.d ] ; then
 for f in /etc/X11/xinit/xinitrc.d/?*.sh ; do
  [ -x "$f" ] && . "$f"
 done
 unset f
fi

# Executed by startx (run your window manager from here)

if [ -d /etc/X11/xinit/xinitrc.d ]; then
  for f in /etc/X11/xinit/xinitrc.d/*; do
    [ -x "$f" ] && . "$f"
  done
  unset f
fi

В конфиге Refind есть строчки отвечающие за то, что ему искать и откуда грузить (SSD, HDD, USB), попробуйте посмотреть их. У меня в конфиге жестко прописана строчка загрузки Арча, и Win, не помню почему так сделал, вроде бы из-за того что надо было еще образ intel грузить по цепочке.

Вообще у меня сейчас Win и Арч можно загрузить тремя способами:
1. EFISTUB
2. GRUB
3. Refind.
В последнее время пользуюсь только EFISTUB, т.е. строчка загрузки Арча прописано прямо в Nvram EFI, а Win сама туда прописывается при установке. Короче Refind наверно удалю, оставлю только Grub и EFISTUB, Grub нужен для загрузки разных ISO (у меня на диске есть раздел, на котором лежат разные спасательные ISO, в том числе Clonezilla, Paretd, PartedMagic, ну и сам установочный Арча :))
