И вот еще:

Input hotplugging with xorg-server 1.5

Normally xorg-server 1.5 tries to configure your keyboard using the new xf86-input-evdev driver (which in turn uses dbus and HAL) instead of using your configuration settings in xorg.conf. This may result in an undesired auto-configured keyboard layout. The fastest workaround is to disable the hotplugging mechanism by adding the following section to your xorg.conf:

Section “ServerFlags”
Option “AutoAddDevices” “False”
EndSection

Note: This will disable Xorg hotplugging for all input devices and revert to the same behavior as xorg-server 1.4.

For more information see Xorg input hotplugging.

Я тоже присматриваюсь счас к Арчу, уже второй раз. В настоящий момент достаточно давно сижу на MOPSLinux, от бубунт отказался довольнотаки давно уже…
Вобще Арч удивительный дистрибутив - ему удалось вместить всебя массу инструментария по управлению пакетами:
1. pacman - простая установка бинарников с автоматическим разрешением зависимостей, как и везде…
2. pkbuilds - аля SlackBuild (slackware) - скрипты по сборке пакетов из исходников, очень простые относительно тех же SlackBuild_ов
- aur - просто огромная колекция pkbuilds, никакой сайт с SlackBuild_ами не сравнится
- yaourt - программа автоматически скачивающая, собирающая и разрешающая зависимости из AUR
3. ABS - система наподобии портов freeBSD (ну и наверное ебилдов gentoo) - собирай хоть всю систему с нуля из исходников…

Вобщем пользуясь Арчем, получается можно выбрать любой способ управления системой (как и везде, как в gentoo, как в freeBSD) - чем плохо ;D
