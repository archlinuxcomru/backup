Апну тему.

Нашел у отца систему с systemd версии 198
[[email protected]]# systemctl –version
systemd 198
+PAM -LIBWRAP -AUDIT -SELINUX -IMA +SYSVINIT +LIBCRYPTSETUP +GCRYPT +ACL +XZ

CTRL+ALT+F2, логинимся под обычным пользователем
запускаем, например, htop

CTRL+ALT+F1, логинимся под root
[[email protected]]# loginctl list-sessions
   SESSION        UID USER             SEAT
         1       1000 papa             seat0           // это Х-сессия обычного пользователя
        10       1000 papa             seat0          // это  наша тестовая сессия на tty2
        11          0 root             seat0           // это сессия root на tty1

3 sessions listed.
[[email protected]]#


[[email protected]]# loginctl session-status 10
10 - papa (1000)
           Since: Вс 2014-02-23 16:39:40 NOVT; 6min ago
          Leader: 1682 (login)
            Seat: seat0; vc2
             TTY: tty2
         Service: login; type tty; class user
           State: online
          CGroup: name=systemd:/user/papa/10
                  ├─1682 login – papa
                  ├─1708 -bash
                  └─1714 htop    // вот наш запущенный htop
[[email protected]]#


[[email protected]]# ps -A | grep htop
 1801 tty2     00:00:00 htop
[[email protected]]#

выполняем terminate:
loginctl terminate-session 10

проверяем:
CTRL+ALT+F2 - видим чистый экран со строкой приглашения залогиниться,
соответственно:
[[email protected]]# loginctl session-status 10
Failed to issue method call: No such file or directory
[[email protected]]#
[[email protected]]# ps -A | grep htop
[[email protected]]#

ВЫВОД:
В systemd версии 198 loginctl terminate-session завершает все процессы и закрывает сессию, в systemd208 эта команда поломана.

Кто может подтвердить?
