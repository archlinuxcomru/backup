Температура процессора во многом зависит от частоты, на которой он работает.
За регулировку частоты процессоров отвечает cpufreq.

Судя по тому, что вы говорите о “режиме” ondemand - cpufreq у вас есть.
Лично мне не нравится, работает этот регулятор. Чаще использую conservative. Он плавно меняет частоту, потому - процессор большую часть времени работает в щадящем режиме. (ondemand слишком часто и неоправданно включает проц на полную).

Не видел в kde специального виджета для управления регуляторами cpufreq, поэтому завязал управление на “Индикатор батареи”: написал скриптик, который вызывается при смене режимов энергопотребления:
#!/bin/bash
## Желаемый регулятор задается в первом параметре, передаваемом скрипту
## Для надежности проверяем его соответствие возможным регуляторам
echo "Specified name of governor - ($1)."
if [ "$1" = "performance" ]; then 
  GOVERNOR=$1;
  echo "Governor -performance- will be used";
elif [ "$1" = "conservative" ]; then
  GOVERNOR=$1;
  echo "Governor -conservative- will be used";
elif [ "$1" = "powersave" ]; then 
  GOVERNOR=$1;
  echo "Governor -powersave- will be used";
else
  GOVERNOR="conservative";   # conservative - мой любимый режим
  echo "Governor -conservative- will be used in this case";
fi
## Включаем выбранный режим
sudo cpufreq-set -g $GOVERNOR;
echo "OK";
