acpi-eeepc-generic 0.8.0-1 обновились вчера, и так как раз пофиксили баги.


  • New version: 0.8.0-1
    Changelog:
    -Wifi toggle events corrected for 901 and 900A
    -Updated and fixed user autodetection
    -Execute command as user by putting a “@” in front of the command in the configuration file
    -Don't do anything nor notify if volume is at 100% and volume up is pressed. Same thing for 0% and volume down, and same thing for brightness too.
    -Execute “xhost +” and “xhost -” before and after a key press.
    -More verbose KEYSHOW
    -Work began on automatic display toggle using xrandr
    -Before restoring touchpad, check if X is running
    -Hide cursor when disabling cursor (needs “unclutter” from AUR)
    -Refactoring of wifi and bluetooth toggle. Mainly simpler if/then/else
    -Many small fixes

    Важные настройки для Fn-1 Fn-9 и “серебряных кнопок” в /etc/conf.d/acpi-eeepc-generic.conf

    # Fallback user to be used for X session if autodetection fails
XUSER=Ваш_Юзер_в_Х-сах
EEEPC_CONF_DONE="yes" # поставьте, что бы не выскакивало сообщение о том, что вы конфиг не сконфигурировали.
    Далее, смотрим /etc/rc.conf

    MODULES=(.... !rt2860 acpi-cpufreq... ) # при старте системы не грузим модуль вай-фай и грузим модуль для "автобаланса" частоты процессора
DAEMONS=(... acpi cpufreq logbackups eeepc-restore .... ) модули для работы с acpi

    Для проца ставим 
    pacman -S cpufrequtils

    Далее редактируем /etc/conf.d/cpufreq

    #configuration for cpufreq control
# valid governors:
#  ondemand, performance, powersave,
#  conservative, userspace
governor="ondemand"
# valid suffixes: Hz, kHz (default), MHz, GHz, THz
#min_freq="2.25GHz"
#max_freq="3GHz"
min_freq="800MHz"
max_freq="1.6GHz"

    Для назначения действий “серебряных” кнопок смотрим в /etc/conf.d/acpi-eeepc-generic.conf

    # These are the 4 silver buttons, located above the Fn+F2 to
# Fn+F6 on certain models. The first just shutdown your screen
# to save power. The second toggle the touchpad on and off.
# Note the "@" in from of the two user button commands: because
# of this, theese commands will be executed as yourself and not root.
# The User1 & User2 are executed as your username!!!
COMMANDS_BUTTON_BLANK=("xset dpms force off")  #  в Х-сах отключаем экран
COMMANDS_BUTTON_RESOLUTION=("/etc/acpi/eeepc/acpi-eeepc-generic-toggle-touchpad.sh") # отключаем тачпад, что бы при наборе текста не мешал
COMMANDS_BUTTON_USER1=("@terminal")   # тут и следующая кнопка для вызова пользовательских комманд
COMMANDS_BUTTON_USER2=("@pcmanfm --no-desktop")

    вот эту строчку:
    echo 0 > /sys/class/rfkill/rfkill0/state

    #!/bin/bash

    . /etc/rc.conf
    . /etc/rc.d/functions

    PID=`pidof -o %PPID /usr/sbin/acpid`
    case “$1” in
    start)
    stat_busy “Starting acpid”
    && /usr/sbin/acpid
    echo 0 > /sys/class/rfkill/rfkill0/state
    if ; then
    stat_fail
    else
    add_daemon acpid
    stat_done
    fi
    ;;
    stop)
    stat_busy “Stopping acpid”
    && kill $PID &> /dev/null
    if ; then
    stat_fail
    else
    rm_daemon acpid
    stat_done
    fi
    ;;
    restart)
    $0 stop
    $0 start
    ;;
    *)
    echo “usage: $0 {start|stop|restart}”..
    esac
    exit 0

Hal-policy написал для клавы и тача?
вот для клавы
/etc/hal/fdi/policy/10-keymap.fdi
<?xml version="1.0" encoding="ISO-8859-1"?>
<deviceinfo version="0.2">
  <device>
    <match key="info.capabilities" contains="input.keymap">
      <append key="info.callouts.add" type="strlist">hal-setup-keymap</append>
    </match>
    <match key="info.capabilities" contains="input.keys">
      <merge key="input.xkb.rules" type="string">base</merge>
      <merge key="input.xkb.model" type="string">keyboard</merge>
      <match key="/org/freedesktop/Hal/devices/computer:system.kernel.name" string="Linux">
        <merge key="input.xkb.model" type="string">evdev</merge>
      </match>
      <merge key="input.xkb.layout" type="string">us,ru</merge>
      <merge key="input.xkb.variant" type="string">,winkeys</merge>
      <merge key="input.xkb.options" type="string">grp:caps_toggle,grp_led:scroll</merge>
    </match>
  </device>
</deviceinfo>
