У меня аналогичная проблема с rtorrent+rutorrent+nginx+php-fpm.
Когда захожу на web-интрерфейс, то получаю в "Лог" следующие сообщения:
[09.05.2014 00:23:19] WebUI started.
[09.05.2014 00:23:20] rTorrent должен иметь доступ на чтение/выполнение к директории tmp. ruTorrent не будет работать. (/tmp/)
[09.05.2014 00:23:20] rss: Часть функциональности плагина будет недоступна. Веб сервер не имеет доступа к внешней программе (curl).
[09.05.2014 00:23:20] rutracker_check: Плагин не будет работать. Веб сервер должен иметь доступ на выполнение к директории session rtorrent (./.rtorrent-session/).

  • Пункт про /tmp.
    • В "config.php" web-интерфейса rutorrent, указано $tempDirectory = '/tmp/';. А права на /tmp - 777. Почему всплывает это сообщение? Просто для "общего развития" ? Не понятно...
  • Пункт про rss.
    • curl в системе установлен. Путь к программе есть в переменной PATH. Что еще нужно ?
  • Пункт про rutracker_check.
    • Из ErrorMessages.wiki следует, что проблема кроется в open_basedir, либо с назначением прав. У меня в конфигурации nginx.conf прописан "user http;", этот пользователь входит в группу users вместе с моим пользователем. Права на папку: 
    drwxr-xr-x  2 shaman users   4096 Май  9 00:35 .rtorrent-session
    Что более чем достаточно. В open_basedir добавил к имеющемуся, строку взятую из арч-вики. Теперь имеет вид: 
    open_basedir= /srv/http/:/home/:/tmp/:/usr/share/pear/:/usr/share/webapps/:/etc/webapps/:/etc/webapps/rutorrent/conf/:/usr/share/webapps/rutorrent/php/:/usr/share/webapps/rutorrent/
  • На сам rutorrent установлены права http:users
    • sudo chown -R http:users /usr/share/webapps/rutorrent

В системе используются:
  1. extra/nginx 1.6.0-1
  2. local/rutorrent 3.6-2
  3. community/rtorrent 0.9.3-2
  4. extra/php-fpm 5.5.12-1

Как решить эти проблемы ? Не понятно...

location ~ \.php$ {
            root /srv/http;
            fastcgi_pass unix:/var/run/php-fpm/php-fpm.sock;
            fastcgi_index index.php;
            fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
            include /etc/nginx/fastcgi_params;
}
