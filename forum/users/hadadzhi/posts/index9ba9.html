На компе имеется НТФС раздел, а так же пара разделов ext3 с установленной на них Ubuntu. Все эти разделы видны в Арче, в гномовском меню “переход” , однако при тыкании на них они не монтируются и не открываются. (Вообще ничего не происходит, никаких сообщений об ошибке и т.п.) Если маунтить их из терминала, под рутом, все работает. Как бы так сделать, чтоб они маунтились из гнома? Юзер добавлен в группу storage, HAL установлен, вставленные сидюки маунтит автоматом. А вот разделы на харде не хочет.
fstab: (Его я не трогал, в нем только разделы, на которых стоит Арч)
# <file system>        <dir>         <type>    <options>          <dump> <pass>
none                   /dev/pts      devpts    defaults            0      0
none                   /dev/shm      tmpfs     defaults            0      0
#/dev/cdrom             /media/cd   auto    ro,user,noauto,unhide   0      0
#/dev/dvd               /media/dvd  auto    ro,user,noauto,unhide   0      0
#/dev/fd0               /media/fl   auto    user,noauto             0      0
UUID=079c8891-bd8a-4043-aa5e-2675c4d4ca53 	/home 	   ext3    defaults 0 1
UUID=235a2b8e-4108-4caf-ab3a-85571af6f982 	swap 	   swap    defaults 0 0
UUID=f57a98ec-d8bc-4500-9a31-ba3b4773db84 	/ 	   ext3    defaults 0 1


Проблему решил, может комуто поможет, сунуть это в /etc/PolicyKit/PolicyKit.conf
<match action="org.freedesktop.hal.storage.mount-removable">
<return result="yes" />
</match>
<match action="org.freedesktop.hal.storage.eject-removable">
<return result="yes" />
</match>
<match action="org.freedesktop.hal.storage.mount-fixed">
<return result="yes"/>
</match>
<match action="hal-storage-mount-fixed-extra-options">
<return result="yes"/>
</match>
<match action="hal-storage-mount-removable-extra-options">
<return result="yes"/>
</match>
Решение подсмотрено здесь - http://archlinux.org.ru/forum/viewtopic … =15#p10361
