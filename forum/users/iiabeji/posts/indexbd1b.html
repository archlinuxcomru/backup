А при загрузке как это прописать, чтобы автоматически?
Вы вики-то почитайте:
Установка соединения при запуске системы
* Если не создали ранее, то создайте ссылку /etc/ppp/peers/provider
ln -s pppoe /etc/ppp/peers/provider
* Если поддержка ppp сокмпилирована в виде модуля ядра, отредактируйте /etc/rc.conf, указав загружать модуль ppp_generic при старте:
MODULES=(… ppp-generic …)
* Редактируем /etc/rc.conf, добавляя ppp к используемым демонам:
DAEMONS=(… ppp …)

Недавно полазил по форумам, перечитал Conky Variables на сайте, обнаружил пару прикольных фич

if_match - оператор сравнения, то есть для раскрашивания температуры делаем:
Tmp:${color2}${if_match ${acpitemp}>70}${color4}${acpitemp}C${else}${color2}${acpitemp}C${endif}${color}
только для раскрашивания загрузки цпу почему то непрокатывает - сбивает все графики

execpi - выполняется внешний скрипт, возвращающий строку в формате коньков, например:
${execpi 10 ~/.conky/batt}
#!/bin/bash
AC=`cat /proc/acpi/ac_adapter/ACAD/state | cut -b26-32`
MAX=`cat /proc/acpi/battery/BAT1/info | grep full | cut -b26-29`
CUR=`cat /proc/acpi/battery/BAT1/state | grep remaining | cut -d ‘ ’ -f8`
PRC=$(( $CUR * 100 / $MAX ))
if ; then
if ; then
echo “Batt:\${color4}”$PRC“%\${color} ”
else
echo “Batt:\${color1}”$PRC“%\${color} ”
fi
else
if ; then
echo “Batt:\${color2}AC ”$PRC“%\${color} ”
else
echo “”
fi
fi
Если питание от сети и 100% батареи - ничего не показываем,
Если нет 100% - процент нормальным цветом,
Если питание от батареи и меньше 35% - показываем красным.
