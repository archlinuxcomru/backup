Установка линукса на этот девайс - очень долгий танец с бубном. Но машинка хорошая, поэтому хотелось бы всё-таки поставить на неё хорошую систему. И, поскольку там много нюансов, то хочется написать немного документации - пригодится тем, кто попытается повторить мой опыт.

Начнём
Конфигурация машинки:
* Процессор VIA Nano U2250 1,3 ГГц. Первый 64-битный процессор VIA
* Видеокарта VIA Chrome9 HC3 DX9 3D
* Wi-Fi Atheros AR242x
* Marvell 88E8040 PCI-E Fast Ethernet
* Broadcom A-Link BlueUsbA2 Bluetooth
* Z-Star Microelectronics Namuga 1.3M Webcam
* 160 Гб винчестер, 1 Гб DDR2

Я решил поставить 64-битный Arch Linux. Загрузился с LiveCD - все хорошо. Подлючился к сети (к проводной вообще без проблем), поставил ntfsprogs, уменьшил размер разделов под венду, удалил диск D:
# ntfsresize -s 25G /dev/sda2
# fdisk
Создал разделы: корень на XFS, /home на ext3, swap. Подмонтировал к /mnt, прописал свой местный репозитарий обновлений в mirrorlist, запустил инсталлятор и выбрал установку по сети с FTP. Всё прошло успешно.

Установка загрузчика прошла неудачно. Поставить его получилось только после загрузки с LiveCD 32-bit системы. Подробнее здесь.

Далее поставил звук. Установил alsa-lib, alsa-oss, alsa-utils, mplayer. Запустил alsamixer, поднял уровень громкости, открыл в mplayer песню. Играет. Отлично. ::)

А вот теперь проблемные места. Wi-Fi.
Wi-Fi у меня не заработал вообще. Рядом с компьютером точка доступа. Из венды он к ней отлично подключается, из линукса нет.

Х-сервер не может найти экран, если запускать его без конфига. Набросал конфиг, смог запустить иксы (у меня установлен гном, поэтому запускаю через /etc/rc.d/gdm start). Однако не работают устройства ввода. (Для выключения иксов в таком случае использую ssh: захожу по сети и даю команду killall xorg или /etc/rc.d/gdm stop). Добавил их поддержку в xorg.conf. Заработал тачпад. Конфиг такой:
Section "Files"
    FontPath "/usr/share/fonts/local/"
EndSection
Section "Device"
    Identifier "Device0"
    Driver     "fbdev"
    VendorName "VIA"
    BoardName  "VIA Chrome9 HC3 DX9 3D"
EndSection
Section "Screen"
    Identifier   "Screen0"
    Device     "Device0"
EndSection
Section "InputDevice"
    Identifier "Keyboard0"
    Driver     "kbd"
    Option     "CoreKeyboard"
    Option     "Protocol"   "Standard"
    Option     "AutoRepeat" "500 30"
    Option     "XkbRules"   "xorg"
    Option     "XkbModel"   "pc105"
    Option     "XkbLayout"  "us,ru"
    Option     "XkbVariant" ",winkeys"
    Option     "XkbOptions" "grp:alt_shift_toggle,grp_led:scroll"
EndSection
Section "InputDevice"
    Identifier "Touchpad0"
    Driver     "synaptics"
    Option     "Protocol" "Auto"
    Option     "Device" "/dev/input/mice"
EndSection
Section "InputDevice"
    Identifier "Mouse0"
    Driver     "mouse"
    Option     "Protocol"     "Auto"
    Option     "Device"       "/dev/input/mice"
    Option     "ZAxisMapping" "4 5 6 7"
EndSection
Section "ServerLayout"
    Identifier "Layout0"
    Screen   0 "Screen0" 0 0
    InputDevice "Touchpad0" "CorePointer"
    InputDevice "Keyboard0"
EndSection
Section "ServerFlags"
    Option "AllowEmptyInput" "False"
    Option "AutoAddDevices" "False"
    Option "AutoEnableDevices" "False"
EndSection

Если не написать последнюю функцию, устройства ввода не будут работать. Виной всему работа устройств ввода через HAL в современных иксах (об этом много писали на форумах), а как настроить их через HAL - бес его знает. Данная секция запрещает использование HAL для устройств ввода.

С драйвером openchrome у меня иксы не работают, только с fbdev. 3D ускорение не работает, тем не менее, работать в gnome можно, тормозов не видно. Также пока не работает клик мыши в результате двойного удара по тачпаду и подключенная USB-мышь бесится: курсор бегает от одного угла экрана до другого при малейшем движении манипулятора. Кривое разрешение экрана (но это исправимо).
