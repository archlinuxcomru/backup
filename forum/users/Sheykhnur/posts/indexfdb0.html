Начиная с версии dhcpcd-6.10.0-1 поведение демона по умолчанию изменено: теперь хук 10-wpa_supplicant не запускается по умолчанию! Судя по комментариям автора к изменению, он с неохотой принял решение перевести файл 10-wpa_supplicant в разряд примеров (примеры файлов для dhcpcd находятся в /usr/share/dhcpcd/hooks/). Чтобы вернуть всё как было, необходимо либо скопировать, либо слинковать файл обратно в рабочую директорию в /usr/lib/dhcpcd/dhcpcd-hooks/ :
sudo ln -s /usr/share/dhcpcd/hooks/10-wpa_supplicant /usr/lib/dhcpcd/dhcpcd-hooks/10-wpa_supplicant

Зачем запускать WiFi через bashrc, когда можно просто настроить сервис dhcpcd. Можно просто включить его командой 
$ sudo systemctl enable dhcpcd
глобально, либо настроить на определённый интерфейс, он сам всё включит (запустит wpa_supplicant и получит ip-адрес автоматически), предварительно нужно только в конфиг /etc/wpa_supplicant/wpa_supplicant-interface.conf, где interface — ваш WiFi интерфейс, например wlp2s0 прописать имя вашей сети и пароль. Подробнее, как обычно, в Вики. Читайте, пробуйте, там всё понятно, если не понятно — спро́сите.

$ sudo ln -s /usr/share/dhcpcd/hooks/10-wpa_supplicant /usr/lib/dhcpcd/dhcpcd-hooks/10-wpa_supplicant

Ничего не нужно никуда прописывать! Всё у вас нормально определяется и устанавливается. Интерфейс wlp4s0 поднимается у вас, поэтому здесь всё нормально, об этом говорится в сообщении:
3: wlp4s0: <NO-CARRIER,BROADCAST,MULTICAST,UP> mtu 1500 qdisc mq state DOWN mode DEFAULT group default qlen 1000
    link/ether 00:1c:26:16:e9:1f brd ff:ff:ff:ff:ff:ff
Здесь нас интересует то, что за ёлочками:
 <NO-CARRIER,BROADCAST,MULTICAST,UP>
Слово UP говорит о том, что интерфейс поднят, а вот NO-CARRIER говорит о том, что интерфейс не обслуживается ни одной службой. dhcpcd установлен (pacman -Ss dhcpcd)? Настроен ли на данный интерфейс (systemctl status [email protected])? Сообщение state DOWN, которое следует далее отражает общее состояние соединения, т.е. оно будет UP только после полного подключения к службе DHCP или ручного назначения интерфейсу IP-адреса. В точке доступа Wi-Fi обычно по умолчанию установлен и настроен DHCP, поэтому вручную задавать IP-адрес не нужно; в вашем случае достаточно установить и настроить dhcpcd службу.
