Топикстартеру возможно понадобилось изменить ассоциацию определённого файла. Отдельно от файла данный диалог выбора программы не вызвать, ведь нужно кликнуть по файлу (необходимо знать контекст файла), иначе как дельфину узнать к какому файлу будет ассоциация? Если вы в дельфине кликаете правой кнопкой на определённом файле, то там есть пункт меню «Открыть с помощью...» и далее идёт выпадающее боковое меню, в котором можно выбрать либо программу (если уже есть ассоциации с этим типом файла), либо настроить свою, выбрав «Другая программа...» и тогда откроется как раз то окошко, про которое вы написали. Т.е. для этого окошка дельфину нужно обязательно знать файл, для которого оно открывается, а для этого нужно кликать правой крысой по файлу. Если нужно настроить отдельные ассоциации, то это можно сделать либо как в вике, ссылку на которую я уже дал, либо в кде в «Параметры системы» → «Приложения» → «Привязки файлов», либо ввести в терминале команду 
$ kcmshell5 filetypes
Вылезет то же самое меню настройки файловых ассоциаций. Выбирайте, как удобнее. Если вручную править конфиги, то в вики на русском подробно описано, как это делать, если графически, то через настройки системы. Там же можно добавить новый тип файла по расширению и связать с ним необходимую программу или действие.

Похоже, что никто самбой не пользуется, но на всякий случай. В данный момент баг ещё присутствует в пакете. Судя по списку рассылки они ввели новую проверку обязательной принадлежности юзера к встроенным группам. Теперь, если гость NT не сопоставлен по умолчанию с группой Unix, то самба не запустится и мы получим ошибку 
../source3/auth/auth_util.c:1382(make_new_session_info_guest)
create_local_token failed: NT_STATUS_ACCESS_DENIED
../source3/smbd/server.c:2000(main)
ERROR: failed to setup guest info.
Для обхода можно использовать команду 
# net -s /dev/null groupmap add sid=S-1-5-32-546 unixgroup=nobody type=builtin

Не так давно в systemd таки запилили нативно перевод ноута в режим гибернации из состояния ожидания после истечения n-го промежутка времени (здорово экономит батарею, если крышку закрыли и не выключили). Раньше приходилось пользоваться сторонними скриптами для этого, но, начиная с версии 239 systemd поддерживает это нативно. Но, как обычно, не обошлось без ложки мёда: в кедах powerdevil корректно данный режим не поддерживает и не даёт logind'у нормально управлять состояниями, т.к. kde'шный powerdevil перехватывает всё управление кнопками (через интерфейс systemd-inhibit). К слову powerdevil в свою очередь опирается на upower, а тот в свою очередь предоставляет d-bus интерфейс для взаимодействия. Но, т.к. suspend-then-hibernate в systemd появился недавно, то интерфейса dbus в upower у него ещё нет (зато имеется в logind, как видно из файла /usr/share/dbus-1/system-services/org.freedesktop.login1.service). В багтрекере kde уже создали тикет по этому поводу. А пока можно юзать временное решение, пожертвовав обычным режимом ожидания:
sudo ln -s /usr/lib/systemd/system/systemd-suspend-then-hibernate.service /etc/systemd/system/systemd-suspend.service

В недавнем обновлении systemd сломали гибернацию. Т.к. отложенная гибернация нативно в системде появилась совсем недавно, а до этого я пользовался скриптом, то вначале подумал, что что-то изменили в сервис-файлах или зависимостях, и попробовал запустить отложенную гибернацию вручную: 
systemctl suspend-then-hibernate
на что был тактично послан на йух: 
Failed to put system into suspend followed by hibernate via logind: Resume not configured, can't hibernate
Как выяснилось, в новой версии добавили обязательную проверку наличия файлов в /boot/loader, о чём недвусмысленно говорится в журнале: 
systemd-logind[457]: Failed to load bootspec config from "/boot/loader": No such file or directory
systemd-logind[457]: Failed to read boot config from "/boot/loader/loader.conf": No such file or directory
systemd-logind[457]: Failed to open "/boot/loader/loader.conf": No such file or directory
Кто юзает systemd-boot с разметкой под gpt, у того всё ок; остальные могут попробовать временно использовать пешеходную дорожку для проезда.
