Посоветуйте WiFI USB карточку
Желательно подешевле, например TP-LINK
Требования:
1) Должна поддерживать режим AP
2) Стандарт N со скоростью 300Mbps
3) Работать из коробки со стандартным ядром.

В наличии такие диски: 2x250Gb (Hitachi Deskstar HDT722525DLA380) 1x1Tb (Seagate Barracuda ST31000340AS), итого 3 диска
Задача. Разбить диски так, чтобы обеспечить
1) Шифрование всего.
2) Максимальная производительность.
3) Надежность для части информации.
Есть идея:
Положим
sda - 250Gb
sdb - 250Gb
sdc - 1Tb
/boot - 200M - md0 (sda1,sdb1,sdc1) - RAID1
/swap - 3gb - md1 (sda2,sdb2,sdc2) - RAID1 (в согласии с советом отсюда:
Many tutorials treat the swap space differently, either by creating a separate RAID1 array or a LVM logical volume. Creating the swap space on a separate array is not intended to provide additional redundancy, but instead, to prevent a corrupt swap space from rendering the system inoperable, which is more likely to happen when the swap space is located on the same partition as the root directory.
)
/ - остальное от 250Gb + такой же размер на sdc - (sda3, sdb3,sdc3) - RAID5
/mnt/extra - (sdc4) - для тех файлов, которых не жалко потерять.

Что вы думаете об этой схеме? Как лучше реализовать? Стоит ли использовать LVM? Или просто использовать mdadm+LUKS

Вообще проблемы была описана в вики тут
Note: If you are getting the following error: “The service ‘Printer Configuration’ does not provide an interface ‘KCModule’ with keyword ‘system-config-printer-kde/system-config-printer-kde.py’ The factory does not support creating components of the specified type.”
This means you need to give the user rights to manage printers.

For CUPS, this is set in /etc/cups/cupsd.conf.
Adding lp to SystemGroup allows anyone who can print to configure printers. You can, of course, add another group than lp.
File: /etc/cups/cupsd.conf
# Administrator user group…
SystemGroup sys root lp

т.е. делаем
$sudo nano /etc/cups/cupsd.conf
Добавляем lp
в строчку
SystemGroup sys root

И да, следим, чтобы пользователь под которым вы работаете тоже был в группе lp
$sudo usermod -a -G lp ИМЯ_ПОЛЬЗОВАТЕЛЯ
