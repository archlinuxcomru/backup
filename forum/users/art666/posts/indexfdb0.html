netcfg 2.6.6 был перемещен в . Накопилось уже достаточно функционала, обеспеченного многими людьми с прошлогодней версии 2.5.4. С версии 2.6.1 по 2.6.6 были исправлены ошибки замеченные с начала релиза 2.6.

  • начальная поддержка IPv6 конфигурации
  • более детализированы wireless-опции (ad-hoc mode, network by BSSID)
  • больше возможностей для bridge-соединений
  • массив ROUTES для определения пользовательских маршрутов в профилях
  • конфигурационный файл /etc/conf.d/netcfg который в данный момент используется только net-auto-wireless
  • новые типы соединений: openvpn, vlan, tuntap
  • a manpage netcfg-profiles(5) описывает более подробно доступные опции
    Подробности об изменениях и ссылка на баг-трекер могут быть найдены в файле NEWS. Старый профили должны остатся совместимыми.

    Пожалуйста ознакомтесь со списком зависимостей, чтобы узнать что нужно установить для определенной конфигурации.

    Дополнительные функции, наиболее востребованные и указанные на баг-трекере станут доступны с релизом 2.7. Обсуждения по поводу развития netcfg и возможностей уже ведутся списке рассылке arch-projects. Не стесняйся оставить сообщение если есть желание принять участие в обсуждении или возможность помочь с патчами.

    Оригинал новости.
