Отключение видео в параметрах ядра и совет с xorg.conf не помогли, экран всё равно гаснет…
1) сделал такой /etc/X11/xorg.conf:
Section "Monitor"
    Identifier "DVI-D-0"
    Option    "Primary" "true"
EndSection
в Xorg.0.log пишет: AMDGPU(0): Output DVI-D-0 using monitor section DVI-D-0 …… Option "Primary" "true"
2) Ещё добавил в grub параметры ядра: video=DP-1:d video=DVI-D-1:e
правильные названия видео получил так: ls -1 /sys/class/drm | sed -n 's/card.-//p'
Результат: при смене пользователя гаснет экран, нажимаю Ctrl+Alt+Backspace и на основном мониторе появляется экранный менеджер lxdm.

При переключении сеансов пользователей гаснет экран, независимо от используемого экранного менеджера (через lxdm -c USER_SWITCH или SDDM по DBUS). При выходе пользователя всё нормально.
Система Linux 4.20.7-arch1. Asus Prime Z270-K, i7-7700K, Radeon 550.
Монитор подключен к DVI выходу Radeon 550. Настройки /etc/X11/xorg.conf отсутствуют.
Команда переключения сеансов (lxdm -c USER_SWITCH) запускает экран lxdm или SDDM, при этом отрубается основной монитор и становится активным интегрированный видеовыход i915, хотя монитор на него не подключен и в настройках Дисплея монитор один.
Приходится нажимать Ctrl+Alt+Backspace, чтобы DVI-выход включится, а иначе можно увидеть экранный менеджер (lxdm, SDDM), только подключив второй монитор на встроенный VGA-выход.
При выборе пользователя подключается монитор на DVI, т.е. применяются настройки: XFCE > Настройки > Дисплей > Настройки экрана.

Не решают проблему: установка пакета autorandr; параметр загрузки ядра video=DVI-D-0 (основной дисплей).
Не хотелось бы отключать загрузку модуля встроенного видео, есть ли способ закрепить за lxdm конкретный дисплей?
Нужно, чтобы при переключении сеансов экранный менеджер не отключал основной монитор и не переключался на пустой видеовыход.

В автозапуск (требуется пакет kwin):
if [ "$XDG_SESSION_DESKTOP" = "XFCE" ]; then
	if [[ -n $(pidof -s xfwm4) ]]; then # XFCE + kwin
		if [[ -n $(which kwin_x11 2>/dev/null) ]]; then
			(kwin_x11 --replace)&	# менеджер окон
		fi
	fi
fi
Кроме того, может понадобиться инициализация среды plasma, например, для захвата окна курсором в "Специальных параметрах окна"
# Plasma startup. We need to create config folder so we can write startupconfigkeys
[[ -n $XDG_CONFIG_HOME ]] && configDir=$XDG_CONFIG_HOME || configDir=${HOME}/.config # this is the default
mkdir -p $configDir
[ -r $configDir/startupconfig ] && . $configDir/startupconfig
# XCursor mouse theme needs to be applied here to work even for kded or ksmserver
if test -n "$kcminputrc_mouse_cursortheme" -o -n "$kcminputrc_mouse_cursorsize"; then
    kapplymousetheme "$kcminputrc_mouse_cursortheme" "$kcminputrc_mouse_cursorsize"
    if test $? -eq 10; then
        XCURSOR_THEME=breeze_cursors
        export XCURSOR_THEME
    elif test -n "$kcminputrc_mouse_cursortheme"; then
        XCURSOR_THEME="$kcminputrc_mouse_cursortheme"
        export XCURSOR_THEME
    fi
    if test -n "$kcminputrc_mouse_cursorsize"; then
        XCURSOR_SIZE="$kcminputrc_mouse_cursorsize"
        export XCURSOR_SIZE
    fi
fi
