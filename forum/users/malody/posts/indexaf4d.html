Собирал я хорошим знакомым новый системник. И столкнулся с багами. Самое интересное приведу здесь. Приобрели
Корпус и МП Остальные комплектующие тут роли не играют).

Т.К.Одного из хороших корпусов Navi с поддержкой 120 мм вентилятора на фронт не было (будет связка SSD+HDD), взяли GMC (одна из "хороших" фирм, корпуса которым доверяем, покрайней мере когдя я работал на сборке системных блоков (собрал более 3 т. штук) они характеризовлаи себя с лучшей стороны, всё удобно, края обрботаны, корпус не дребежит и т.п.).
Ссылка на корпус с пометками.
Баги корпуса. На картинке я совместил по масштабу МП (по отверстиям для крепления.)
1. Крепление на фронте для 120 мм. вентилятора есть. Но... дырки сделаны гораздо меньшего диаметра, саморезы для вентилятора там просто не пролазиют и не коим образом не вкручиваются. Решение: рассверлил сверлом по металлу.
2. Красиво пробросить провода питания не получилось и не получится. Смотрите на картинке место обведённое зелёным кругом. М.п. перекрывает отверстия для проводов. В идеале провода питания проходят за этой металлической панелью, там скручиваются, а с "лицевой" стороны выходят и крепятся к мп. Отверстия ниже, в металлической панели корпуса для крепления фронт аудио, и кнопок/лампочек фронта находятся на хорошем расстояни, - прикрепляем красиво. А вот если бы МП была формата АТХ, - то отверстия бы опять перекрывались, но провода что индут внизу тонкие, думаю прошло бы спокойно.
3. Подсоединение Жёстких дисков. Только г-образным SATA кабелем. Сверху есть площадка для SSD, там нормально пролазиет кабель (площадка немного утоплена вглубь). А вот для HDD c трудом.

Включаю "режим установки win 7". Жамкаю "установить", далее выдаёт ошибку "Не найден необходимый драйвер носителя" и я не одинок. Что я только не делал... и запускался естественно с УСБ2 порта и биос весь перешерстил, в итоге подключил SSD к своему компу, и оттуда поставил.

38 - это по сути тот же Infinality, но без необходимости устанавливать доп. пакеты
. FreeType2.7 соджержит метод хинтинга Infinality, поэтому его пакеты ставить не надо. Остальное "допиливаем" сами, - что и написано в статье.
Ваш /etc/profile.d/freetype2.sh должен выглядеть так
# Subpixel hinting mode can be chosen by setting the right TrueType interpreter
# version. The available settings are:
#
#     truetype:interpreter-version=35  # Classic mode (default in 2.6)
#     truetype:interpreter-version=38  # Infinality mode
#     truetype:interpreter-version=40  # Minimal mode (default in 2.7)
#
# There are more properties that can be set, separated by whitespace. Please
# refer to the FreeType documentation for details.

# Uncomment and configure below
export FREETYPE_PROPERTIES="truetype:interpreter-version=38"

По поводу быстродействия. Метод хинтинга никак не влияет на него). Возможно слишком много шрифтов стояло, поэтому так? (на оч.старых компах может влиять (кол-во шрифтов over 9000), как на Linux не знаю, но на ms win при огромном количестве шрифтов, заметно снижение быстродействия)
