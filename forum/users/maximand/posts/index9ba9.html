xSPiRiTx
а Dual Architecture это как понимать? и 32 и 64 сразу?
Это означает, что на загрузочном образе есть инсталляторы для 32 и 64ной систем. И нет необходимости иметь два отдельных инсталляционных диска для разных архитектур.
По поводу 64-ки: полностью перешёл на неё месяца 3-4 назад и проблем никаких не заметил. По сравнению с прошлыми 3-5 годами заметен большой прогресс. Никаких танцев с бубнами уже не нужно. Конечно же не обойтись без мультилиба, если используется скайп и wine.
Так что мой совет, если ваше железо поддерживает 64ную архитектуру - то переходить на 64-ную систему можно и нужно.
У меня аналогичное мнение. 32 разрядную систему пора признать анахронизмом. По умолчанию использовать 64-х, а 32-х сборки для старого железа.

То что mehanoid привёл немного упростил.
#!/bin/bash
function get_aur_pkg_version
{
  get_page "http://aur.archlinux.org/rpc.php" "type=info&arg=$1" | sed -r 's/.*"[Vv]ersion":[ "]*([^",]*).*/\1/'
}
function get_page
{
  if [ -z "$2" ]; then
    wget -q -O - "$1"
  else
    wget -q -O - "${1}?${2}"
  fi
}
  NOT_FOUND='{"type":"error","results":"No result found"}'
  IGNORED_PKGS=" "
  # Jozef Riha submitted this next line to replace my "hacky" cat/sed/grep/sed pipe :)
  for LINE in $(sed -n "/IgnorePkg/s/^\s*IgnorePkg\s*=\([^#]*\).*$/\1/p" /etc/pacman.conf); do
    IGNORED_PKGS="$IGNORED_PKGS $LINE "
  done
  pacman -Qm | while read LINE
  do
    PKG=$(echo $LINE | cut -d ' ' -f1)
    [[ " $IGNORED_PKGS " =~ " $PKG " ]] && continue
    VER=$(echo $LINE | cut -d ' ' -f2)
    AURVER=$(get_aur_pkg_version $PKG)
    if [ -z "$AURVER" ] || [ "$AURVER" == "$NOT_FOUND" ]
    then
      continue
    else
      #CMP=$(compare_package_releases $VER $AURVER)
      #[ "$CMP" == 'lt' ] && echo "$PKG-$AURVER"
      [ $(vercmp $VER $AURVER) -lt 0 ] && echo "$PKG $VER => $AURVER"
    fi
  done
Скрипт выводит списком пакеты, которые обновились с указанием текущей версии и обновлённой.
