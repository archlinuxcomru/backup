В принципе на все это можно положить … и даже не использовать этот psd …
Но если есть желание, то можно провести анализ, что пишется на диск - для интереса.
Для начала желательно посмотреть что пишется на диск без использования всяких установленных прог (psd и выноса кеши в озу/tmp), вообщем по дефолту … запустить только один браузер и ничего лишнего … и при мониторинге ничего не делать.
Для определения что пишется на диск можно использовать утилиту iotop, но запускать по хитрому
sudo iotop -obPat
TIME    PID  PRIO  USER     DISK READ  DISK WRITE  SWAPIN      IO    COMMAND
18:08:00 Total DISK READ :       7.77 K/s | Total DISK WRITE :       0.00 B/s
18:08:00 Actual DISK READ:       7.77 K/s | Actual DISK WRITE:       0.00 B/s
…….
TIME    PID  PRIO  USER     DISK READ  DISK WRITE  SWAPIN      IO    COMMAND
18:10:13     188 be/3 root          0.00 B     84.00 K  ?unavailable?  [jbd2/sda3-8]
18:10:13     288 be/4 root          0.00 B     40.00 K  ?unavailable?  vnstatd -n
18:10:13     905 be/4 root          8.00 K      0.00 B  ?unavailable?  python /usr/bin/iotop -obPat
Привел только начало и конец мониторинга.
Чем удобен такой запуск? - тем, что показания записи на диск в каждой строке вывода показаны с накоплением (суммируются).
Из приведенного вывода заключаем, что в течение 2 минут на диск записалось
- 84 К - данные jbd2 (информация, связанная с журналированием файловой системы ext4)
- 40 К - данные vnstat (информация мониторинга сетевой активности)
- 8 К - данные самой утилиты
…. и как видим, что браузер palemoon за эти 2 минуты ничего не записал … хотя должен записывать каждые 15 с (просто я изменил время записи).
Это база, от чего можно плясать - а дальше можно экспериментировать - менять параметры браузера, использовать psd, кеш и прочее. Есть с чем сравнивать и принимать решение о необходимости использования всяких дополнительных прог.
