Конкретно для Вашего телефона официальная разблокировка так не проходит? - http://4pda.ru/forum/index.php?showtopic=260549

На самом деле все просто -

1. региться на том сайте
2. иметь только fastboot, adb и никакие dll не нужны
3. телефон выключен.
2. Зажмите одновременно и держите кнопки питания и уменьшения громкости.
3. При помощи клавиш звука выберите пункт «Fastboot» и нажмите кнопку питания.
4. Соедините ваш аппарат с компьютером при помощи USB-кабеля.

имеем черный экран и какой-нибудь горящий светодиод. Идем в терминал, su

./fastboot oem get_identifier_token

10. Вы увидите один из экранов выше — это ключ. Необходимо нажать правую кнопку мыши, выбрать «пометить», выделить блок и нажать правую кнопку для копирования. Ключ должен начинаться с
<<<< Identifier Token Start >>>>
и заканчиваться
<<<<< Identifier Token End >>>>>
Затем необходимо перейти на официальный сайт (потребуется регистрация), пролистать до Step 10, вставить в поле My Device Identifier Token ваш ключ, и нажать Submit.
11. Спустя несколько секунд на адрес электронной почты, указанный при регистрации, придет сообщение с ключом разблокировки в аттаче — Unlock_code.bin.
Сохраняем файл в папку с фастубтом
скармливаем фастбуту файл -

./fastboot flash unlocktoken Unlock_code.bin
Ничего рискового тут нет. Полностью затрутся пользовательские данные - их можно попытаться сохранить Титаниумом.

И надобности в adb пока не усматривается, адб для другого нужен.

В качестве варианта попробуйте adb в рутовой консоли. Обычно прошивка представляет собой файлы *.img или зип-архивы образа файловой системы (это для рекавери). Как и винт в компьютерах, память телефона нарезана на разделы. Изначально же на большинстве телефонов никакого рекавери попросту нет, его надо прошить - тут тоже есть варианты: если производитель предусмотрел такую возможность отдельного раздела для рекавери (классический пример - Моторола), то шьем рекавери; в остальных случаях шьется ядро с встроенным рекавери (классический пример - Сони). А для того, чтобы прошить ядро как раз и надо разблокировать загрузчик. При этом даже и накат любой новой прошивки с новым ядром уже делается в два этапа - спрева фастбутом шьется новое ядро с рекавери, потом из этого рекавери накатывается zip прошивки.

Разблокрировка же загрузчика каждого телефона имеет свои тонкости - в одних случаях производитель дает такую официальную возможность (Сони - путем ввода индивидуального кода), или шаманством (Сони - замыкание нужных инженерных контактов). В других - только шаманством или вообще никак.

Собственно все правильно говорят. Дополню. Коли Вас видят здесь, стало быть работать в консоли умеете. Значит, adb -то, что доктор прописал. В качестве примера покажу, как у меня ставится дежурный набор программ после установки любой прошивки -

cd место-где-лежит-комплект
./adb install *.apk

ОТносительно рута и пр. - смотрите в Вашем скрипте что-нибудь похожее на это -

./adb wait-for-device
echo "[*] Device found."
echo "[*] Pushing unlock tool..."
./adb push unlock /data/local/tmp
./adb shell chmod 755 /data/local/tmp/unlock
./adb shell su -c /data/local/tmp/unlock
./adb shell rm /data/local/tmp/unlock
./adb reboot bootloader
сразу и РЕКАВЕРИ
./fastboot flash recovery TWRP_2.5.img
./adb kill-server
Признаки таковы, они для каждого телефона с учетом его особенностей таковы - кусок скрипта, в котором в какую-то временно доступную зону заливается файлик, у которого меняются права и затем его засовывают в систем\апп. Обычно это файлик su.apk + Superuser.apk. Под это дело можно распатронить и виндовый скрипт (батник) - содержимое вполне прозрачно переносится в Linux.
