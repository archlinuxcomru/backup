Я похоже малость (нет, не малость, к сожалению) впух с этими мелкими графическими делами :(

На компе с Arch установлен NX Nomachine - для доступа к паре серверов через rdp.

Сейчас обнаружил, что войти клиентом не могу с сообщением "Невозможно обнаружить запущенный дисплей. Вы хотите, чтобы NoMachine создала новый и продолжала подключение к рабочему столу?"

При ответе Да - сообщение Oops! Cannot run startkde. Please ensure thet the requested application is in the system PATH and that you have appropriate credentials to execute it.
При ответе Нет - нет доступных сеансов на этом сервере.

Что делать, пока не понимаю, гугл не очень разговорчив на эту тему.

Вдруг что тут подскажут.

Спасибо.

PS: забыл. Если арч рестартовать, при входе клиентом Nomachine стартовый экран видно. Но сеанс вырубает, как только залогинишься. Ввёл логин-пароль, и привет.
PPS: пробую рецепт https://www.nomachine.com/AR06D00398, заменил startkde на startplasma-X11,

в node.cfg строки запуска
DefaultDesktopCommand "/usr/bin/dbus-launch --exit-with-session startkde-X11"
CommandStartKDE "/usr/bin/dbus-launch --exit-with-session startplasma-X11"

ничего - также про cannot run startkde,
далее написал с полным путём - /bin/startplasma-X11,

в node.cfg строки запуска
DefaultDesktopCommand "/usr/bin/dbus-launch --exit-with-session /bin/startkde-X11"
CommandStartKDE "/usr/bin/dbus-launch --exit-with-session /bin/startplasma-X11"
то же самое, оставил только исполнимый файл

DefaultDesktopCommand "/bin/startkde-X11"
CommandStartKDE "/bin/startplasma-X11"
- ошибка Отказ при согласовании сеанса. Ошибка: Не удалось подключиться к запрашиваемому узлу.
