Раскомментируй те зеркала, которые обновляются регулярно. См. здесь: https://www.archlinux.de/?page=MirrorStatus
У меня, например, это так: в самый конец /etc/pacman.d/mirrorlist, где все репы закомментированы, добавляю микс из следующих зеркал:
# ---------- My own list ----------
# ========== From USA =============
Server = ftp://ftp.archlinux.org/$repo/os/i686
Server = http://mirror.umoss.org/archlinux/$repo/os/i686
# ========== From Great Britain ===
Server = ftp://mirror.lividpenguin.com/pub/archlinux/$repo/os/i686
Server = http://mirror.lividpenguin.com/pub/archlinux/$repo/os/i686
# ========== From Germany =========
Server = ftp://mirror.giantix-server.de/archlinux/$repo/os/i686
Server = http://mirror.giantix-server.de/archlinux/$repo/os/i686
Server = http://archlinux.limun.org/$repo/os/i686
# =========== From Russia =========
Server = ftp://mirror.svk.su/archlinux/$repo/os/i686
Server = http://mirror.svk.su/archlinux/$repo/os/i686
Server = ftp://mirror.yandex.ru/archlinux/$repo/os/i686
Server = http://mirror.yandex.ru/archlinux/$repo/os/i686
# =========== Common ==============
Server = ftp://mirrors.kernel.org/archlinux/$repo/os/i686
Server = http://mirrors.kernel.org/archlinux/$repo/os/i686

Вот на этом в дань теме предлагаю по-подробнее. Не плохо было бы хоть вкратце описать основные косяки дистров, в частности тестового дебиана.

Но при этом я так и не услышал, что будет, если на стабильный дебиан настроить тестовы репозитарий с приложениями?
В определенной степени бубунта - это тестовый дебиан, отсюда и повторение части проблем (хотя ребята из Canonical добавляют и своих). Что больше всего лично меня доставало - нестабильность обновлений: обновился какой-то пакет, а зависимости для него нет, получаешь геморрой. Testing, конечно, получше unstable, но не намного. Сидеть на стабильной ветке и подключать unstable/testing - не вариант, так как версии пакетов привязаны к ядру, а ядро не обновляется (за исключением патчей, связанных с безопасностью). Приходится прикручивать сторонние репозитории и самодельные пакеты, в итоге через полгодика такой веселой жизни система загажена (поломанные зависимости, куча хлама от старых пакетов, которые не убрать даже с опцией purge) и начинает тихо рассыпаться (поэтому, наверное, у бубнты тут и поспевает новый дистр).
Собственно, потому я и на арче, что апдейтоман. Иначе стабильный дебиан меня бы устраивал. Но даже я не влазил в ветку experimental - там точно убьет…
