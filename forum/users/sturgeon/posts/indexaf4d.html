Так и пришлось попробовать настроить интернет Beeline на Арче в обход маршрутизатора. И так все по порядку. Опишу способ поднятия сети с помощью netcfg. Нужно установить пакет netcfg.
pacman -S netcfg
Теперь необходимо настроить сеть Beeline. Редактируем к примеру файл /etc/network.d/beeline
CONNECTION='ethernet'
DESCRIPTION='Beeline'
INTERFACE='eth0'
IP='dhcp'
#DNS
IPCFG=('route add 213.234.192.8 dev eth0' 'route add 85.21.192.3 dev eth0)
#L2TP
IPCFG+=('route add 85.21.66.205 dev eth0')
Вместо 85.21.66.205 пишите свой l2tp сервер. На счет днс я не вкурсе они одинаковые для всех или разные. Далее в /etc/rc.conf нужно добавить такие записи.
NETWORKS=(beeline)
DAEMONS=(... !network net-profiles ...)
Теперь можно запустить сеть. Если раньше у вас сеть запускалась через network то ее для начала надо загасить.
/etc/rc.d/network stop
После этого поднимаем сеть с помощью netcfg.
netcfg beeline
Теперь приступаем к настройке xl2tpd. Редактируем /etc/xl2tpd/xl2tpd.conf
[global]
access control = yes
auth file = /etc/ppp/chap-secrets
[lac corbina]
;lns = tp.internet.beeline.ru
lns = 85.21.66.205
redial = yes
redial timeout = 5
require chap = yes
require authentication = no
name = LOGIN_ISP
ppp debug = yes
pppoptfile = /etc/ppp/options.xl2tpd
require pap = no
autodial = yes
Обратите внимание параметр lns отредактируйте для своего l2tp сервера. По имени сервера у меня соединяться не захотело. Может быть оно и соединилось бы, но уж больно долго. По ip адресу соединяется махом.

Теперь надо отредактировать /etc/ppp/options.xl2tpd
name LOGIN_ISP
remotename L2TP
ipparam corbina
connect /bin/true
mru 1460
mtu 1460
nodeflate
nobsdcomp
persist
maxfail 0
nopcomp
noaccomp
defaultroute
usepeerdns
В файл /etc/ppp/chap-secrets пишем логин и пароль через *
LOGIN_ISP * PASSWORD
Запускаем xl2tpd и ждем небольшое время.
/etc/xl2tpd start

Расскажу как добится вот такой вот красоты. Может кому то интересно будет.

Для начала надо установить два пакета
pacman -S sudo gxmessage
Топаем править /etc/sudoers и добавляем строку
%users localhost=NOPASSWD: /sbin/shutdown -h now, /sbin/shutdown -r now
Добавляем пользователя в группу users и перезапускаем иксы
gpasswd -a username users
openbox --exit
Пишем скрипт на баше
#!/bin/bash
gxmessage -borderless "You really wish to switch off the computer?" -center -title "Exit" -default "Cancel" -buttons "_Cancel":1,"_Log Out":2,"_Reboot":3,"_Shutdown":4 >/dev/null
case $? in
    1) echo "Exit";;
    2) openbox --exit;;
    3) sudo reboot;;
    4) sudo halt;;
esac
Сохраняем его под любым именем. Я сохранил его как exit. Кидаем его в /usr/local/bin или куда вам угодно и делаете его исполняемым
chmod +x /usr/local/bin/exit
Топаем править файл меню Openbox'а и добавляем туда пункт
<item label="Shutdown">
  <action name="Execute">
     <command>/usr/local/bin/exit</command>
  </action>
</item>
В консоли выполням
openbox --reconfigure
И пробуем. Теперь в зависимости от того какую кнопку нажать будет выполнено то или иное действие.

а кто-нить знает как сделать так, чтобы примонтированные диски отображались в файловых менеджерах в гноме?
если убрать строчки монтирования вот эти:
/dev/sda2 /mnt/windows/disk1 ntfs-3g defaults,users 0 0
/dev/sdb2 /mnt/windows/disk2 ntfs-3g defaults,users 0 0
то, тогда в гноме в меню Переход и в левой панели наутилуса появляются примонтированные диски, но при этом их содержимое не видно, будто система их видит, но они какбе не смонтированы еще
если же обратно вернуть эти обе строки, то тогда содержимое дисков открывается по адресу /mnt/windows/disk1, но опять же тогда их не видать в меню и в файловых менеджерах =(
Монтировать их в /media.
