Доброе время суток, господа.
Занимаюсь web-разработкой, меньше месяца назад мигрировал на Arch. Естественно потребовался LAMP, который был успешно поставлен по мануалу LAMP и его английской версии LAMP. Всё блестяще работало, но в определенный момент потребовался Zend Optimizer, который на настоящий момент не имеет поддержки PHP 5.3, как выяснилось на официальном форуме. Встала задача откатиться на PHP 5.2.* Пробовал разные способы, наломал дров в итоге: apache ставится pacman'ом и работает. PHP 5.2 из AUR'а ставится, но не работает. О нем позже.
MySQL ставится pacman'ом и не работает.
То есть:
# /etc/rc.d/mysqld start
:: Starting MySQL
Причем полминутки думает (BUSY). После этого на повторный старт выдает FAIL не задумываясь, а на стоп говорит DONE.
На обращение к mysql, bash говорит команда не найдена.
Причем появляется после “долгого” старта с FAIL в конце появляется два процесса
21839 pts/1 S 0:00 /bin/sh /usr/bin/mysqld_safe –datadir=/var/lib/mysql
21940 pts/1 Sl 0:00 /usr/sbin/mysqld –basedir=/usr –datadir=/var/lib/mysql –user=mysql –log-error=/var/lib/mysql/vaiolaptop.err –pid-file=/var/lib/mysql/vaiolaptop.pid –socket=/tmp/mysql.sock –port=3306

В /etc/my.cnf добавил строчку про лог:

port = 3306
socket = /tmp/mysql.sock
datadir = /var/lib/mysql
log = /var/log/mysql.log
skip-locking
key_buffer = 16M
max_allowed_packet = 1M
table_cache = 64
sort_buffer_size = 512K
net_buffer_length = 16K
myisam_sort_buffer_size = 8M
