У меня вай-фай заработал, как только я нашёл и установил проприетарный драйвер. Ссылку я дал.
Ничего больше не настраивал, разве что, добавил модуль b43
к примеру, tg3 - модуль для сетевых карт. b43 - модуль для моей беспроводной карты. Полное название модуля b43 , что бы быть точным, - b43-pci-bridge. Этот модуль и сейчас подгружен. Он был с самого начала, я ничего не менял.

Какая у вас карта?
Значит, мне не известно, какая у вас карта, хоть и ноуты у нас от одного производителя. По этому пишу, как выяснить тип беспроводной карты;
lspci | grep Network
Далее, если это Broadcom, то дёргаем архивчик с пропиетарным драйвером broadcom-wl-4.150.10.5.tar.bz2 вот с этого узла – http://mirror2.openwrt.org/sources/

Если у вас другая карта, то соотвественно названию вашего производителя какой-то там архив… :)

С архивом делаем вот что:
# export FIRMWARE_INSTALL_DIR="/lib/firmware"
# tar xjf broadcom-wl-4.150.10.5.tar.bz2
# cd broadcom-wl-4.150.10.5/driver
# b43-fwcutter -w "$FIRMWARE_INSTALL_DIR" wl_apsta_mimo.o

Короче говоря, - не морочьте себе голову. Задача ясна - для беспроводной карты нужен лишь её проприетарный миркрокод. Он есть. Всё остальное пусть подождёт. Сеть работает прекрасно и без всяких там специальных утилит.

Не нужно настраивать. Пусть будет, как есть.
Попробуйте погасить eth0, затем подключчиться к беспроводной сети.

На моём примере всё обстоит так:
1. wlan0 - работает и видит сеть, со стандартным драйвером (модулем) подгруженым из ядра
2. Добавил файл в /etc/conf.d/net
(не предпологаю, какое отношение этот файл имеет к процессу, пока не выяснял. Но, этот файл, в сущности, каким-то образом имеет отношение к wpa_supplicant. Хотя, при установке, если бы он был нужен, в Арче он должен был присутствовать. Этот файл я содрал в какой-то статье по настройке супликанта. Думаю, он не играет никакой роли в данной ситуации с Арчем.
#modules=( "wpa_supplicant" )
#wpa_supplicant_eth0="mode managed"
dns_domain_lo="MIGNET"
config_wlan0=( "dhcp" )
modules_wlan0=( "!iwconfig" "wpa_supplicant" "!ifplugd" )
wpa_supplicant_wlan0=( "-Dwext" )
dhcp_wlan0="nonis nontp"
preup () {
        if [ "${IFACE}" == "wlan0" ]; then
                ifconfig wlan0 up > /dev/null 2 >&1
                iwlist wlan0 scanning > /dev/null 2 >&1
        fi
}
postdown () {
        if [ ${IFACE} == "wlan0" ]; then
                einfo "Disabling Wireless"
                ifconfig ${IFACE} down
        fi
}
Без приувеличения, на этом всё.
Далее:
ifconfig eth0 down
ifconfig wlan0 up
И через вафай-радар подключаюсь в сеть.

Теперь вы можете просканировать тут же на присутствие точек доступа:

Сеть сообщается? Получен ли список доступных точек? Думаю, да… :) Это уже успех. Если бы карта не работала, то полученного списка небыло бы совсем.
