Найдите свой сканер по lsusb :
 $  lsusb
Bus 002 Device 001: ID 1d6b:0003 Linux Foundation 3.0 root hub
Bus 001 Device 003: ID 04f2:0833 Chicony Electronics Co., Ltd
Откройте файл /lib/udev/rules.d/49-sane.rules и впишите в него:
# Kyocera Color M5521cdw
ATTRS{idVendor}=="xxxx", ATTRS{idProduct}=="xxxx", MODE="0664", GROUP="scanner", ENV{libsane_matched}="yes"
Где первые хххх ваш ID из lsusb, второй хххх ( после двоеточия : ) idProduct.

Нашел виновника. Скрипт из https://github.com/hectorm/hblock заменяет "родной" файл hosts в /etc/ на свой, с блокировкой "нехороших" сайтов по "списку".
За что он стал "блокировать" яндекс - не понятно. Сайт давно внесен в "белый список" в /etc/hblock.d/whitelist.
Заменил файл hosts на "родной", и с яндексом, все стало нормально. Возможно, верну все обратно - рекламы и спама с hblock-ом меньше.
Все теперь [РЕШЕНО]. Всем Спасибо!

P.S. Переустановил из /var/cache/pacman/pkg/ hblock-git-2.0.4-1-any.pkg.tar.xz, внес в белый список все сайты с упоминанием слова yandex (ок.37 шт.) из блоклиста файла hosts, обновил блок-лист - сайт яндекс и почта заработали нормально.
Еще заметил один плюс - с hblock сайты открываются "ну очень быстро" (проверил: когда вернул родной hosts - пошли "тормоза").
