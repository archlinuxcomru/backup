А по статусу?
У Вас могут быть быстрые, но неактуальные зеркала
Для начала, конечно же необходимо уставить reflector (есть в реп community), а уже alias запускает этот скрипт. Вот выписка: Reflector — это скрипт, который автоматизирует процесс настройки зеркал, включающий в себя загрузку свежего списка зеркал со страницы Mirror Status, фильтрацию из них наиболее обновленных, сортировку по скорости и сохранение в /etc/pacman.d/mirrorlist. Кстати, reflector можно запускать через юнит systemd описание тут. А, для получения актуальных зеркал, тут Вы правы - надо этот алиас, наверное, раз в месяц запускать. Здесь, этот алиас, просто привел для примера.

penguin
А инструкция где…
Настройте сами: 1. Подкорректируйте файл .bashrc в Вашей директории. Для контроля Bash History (не сохранять строки начинающиеся с символа <пробел>, не сохранять строки, совпадающие с последней выполненной командой и удалять ВСЕ дубликаты команд с истории) впишите в .bashrc:
export HISTCONTROL=ignoreboth:erasedups ..........................................Там же в .bashrc,чтобы включить shopt впишите:
shopt -s autocd
shopt -s cdspell
shopt -s cmdhist
shopt -s dotglob
shopt -s histappend
shopt -s expand_aliases
shopt -s checkwinsize ..............................................Что означает каждый shopt можно найти в Гугле.
2. Там же в .bashrc пишите алиасы. Например:
alias mirror="sudo reflector –protocol https –latest 50 –number 20 –sort rate –save /etc/pacman.d/mirrorlist"
При команде mirror - этот алиас отсортирует список из 20 зеркал по скорости скачивания. Сколько алиасов можно вписать в .bashrc - не знаю, пока насчитал больше 40. Список алиасов (чтобы не запоминать все) вызывайте командой alias.
