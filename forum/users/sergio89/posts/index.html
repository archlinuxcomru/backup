DNS=192.168.0.10 192.168.1.3 192.168.0.1
При загрузке системы, днс адреса в таком же порядке добавлялись в /run/systemd/resolve/resolv.conf, как указано в конфиге. Тоесть, всегда было:
nameserver=192.168.0.10
nameserver=192.168.1.3
nameserver=192.168.0.1
Сейчас же, после загрузки системы, в /run/systemd/resolve/resolv.conf днс адреса добавляются в случайном порядке, при этом каждый раз по разному.
При systemctl restart systemd-resolved.service он так же добавляет днс в случайном порядке.

Вопрос, как сделать так, чтобы днс адреса добавлялись в том порядке, в котором указаны?

Можно сделать так:
{ rule = { class = "Pcmanfm" },
      properties = { }, callback = function (c) if c.type == "dialog" then awful.client.floating.set(c, true) end end  },
По крайней мере Pcmanfm задает соответствующий тип окну "_NET_WM_WINDOW_TYPE(ATOM) = _NET_WM_WINDOW_TYPE_DIALOG".
С другими приложениями такой способ не пройдёт, они половину модальных окон помечают как "normal/desktop".

reiserfsck восстановил все файлы, по крайней мере бегло просмотрев структуру, вроде все на месте, за исключением корневых названий директорий. Некоторые файлы/директории были перемещены в корень с рандомными именами, но это не страшно.

Natrio
Как правило, после уничтожения ФС восстановить с раздела можно только нефрагментированные файлы, имеющие сигнатуры и внутреннюю структуру, по которой можно определить их границы, например всячесчкие картинки и т.д., для этого есть программа photorec из пакета testdisk.

photorec пробовал, но он восстанавливает без структуризации и дает рандомные имена файлам, при этом все кидает в одну директорию, получиться каша при восстановлении ~500.000 файлов
