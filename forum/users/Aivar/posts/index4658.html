/tmp/i3status.conf
---
general {
output_format = "none"
interval = 1
}

order += "cpu_usage"
order += "cpu_temperature cpu"

cpu_usage {
format = "%usage"
}

cpu_temperature cpu {
path = "/sys/devices/platform/coretemp.0/hwmon/hwmon1/temp2_input"
format = "%degrees°"
}
$ i3status -c /tmp/i3status.conf | osd_cat -c grey -o 7 -d 0 -l 1 -s 1 -A right

есть папка lost+found.
А вот не надо лезть туда немытыми руками пользователя. В директорию lost+found переносятся файлы, которые физически занимают место в файловой системе, но не имеют имени и доступа к себе вследствии програмных или аппаратных сбоев. Это, например, может произойти если удаляется файл, который занят процессом, и в этот момент случается сбой, напр, электропитания. fsck из lost+found способна восстановить утраченные файлы, поэтому эту директорию не рекомендуется удалять, переименовывать и т.д. Удаленный каталог, кстати, создается не по mkdir, а по mklost+found и недоступен пользователю.
